# Webhook 健康檢查紀錄

## 說明

以下為健康檢查中發現的 Webhook 訂閱異常店家，屬於 CRITICAL 等級，需人工介入處理。

---

## 異常清單

### [109] 懷孕飲食分享

| 項目 | 狀態 |
|------|------|
| Webhook 訂閱狀態 | ❌ 未訂閱 |
| DB `WebhookSubscribed` | `false` |
| 處理方式 | 需手動重新執行訂閱 |

**問題描述：**
資料庫中 `WebhookSubscribed` 欄位為 `false`，代表綁定時 Webhook 訂閱未成功（或從未執行）。該粉專目前無法接收任何 Facebook 直播留言事件。

---

### [20] MyDress

| 項目 | 狀態 |
|------|------|
| Webhook 訂閱狀態 | ⚠️ 訂閱異常（欄位不完整）|
| 缺少欄位 | `feed`、`live_videos`、`message_reactions`、`messages`、`messaging_postbacks` |
| 處理方式 | 需人工介入，重新訂閱並指定完整欄位 |

**問題描述：**
Webhook 訂閱存在但訂閱欄位不完整，導致部分直播事件（留言、互動、私訊）無法正常推送到系統。

---

## 處理狀態

目前 QA 與 Prod 環境皆已完成 Webhook 訂閱。上述異常案例若重新出現，需確認：

1. 粉專管理員帳號是否具備三個必要權限（`CREATE_CONTENT`、`MODERATE`、`MESSAGING`）
2. 訂閱 API 呼叫時是否帶入完整的 `subscribed_fields` 清單
3. 訂閱完成後，`WebhookSubscribed` 欄位是否正確更新為 `true`
