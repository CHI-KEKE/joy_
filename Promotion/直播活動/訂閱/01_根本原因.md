# 後台拉不到訊息的根本原因

共整理出三個導致後台無法接收 Facebook 直播留言的根本原因：

---

## 原因 1：粉絲團 Connect 後未執行 Webhook 訂閱

**問題描述：**
粉絲團完成 Connect（OAuth 授權綁定）後，系統未自動呼叫 `POST /{page-id}/subscribed_apps`，導致 Facebook App 無法接收到該粉絲團的 Webhook 事件（留言、互動等）。

**Action Item：**
粉絲團綁定流程完成時，同步執行 Webhook 訂閱（`subscribed_apps`）。

---

## 原因 2：Facebook comment 的 created_time 沒有精確到毫秒

**問題描述：**
Facebook Graph API 回傳的留言 `created_time` 欄位只精確到秒（無毫秒）。在使用分頁機制（pagination）抓取留言時，若同一秒內有多筆留言，以 `created_time` 作為游標會導致部分留言被跳過，造成掉資料。

**Action Item：**
改由 FB Graph API 取得留言並利用 Facebook 原生的分頁機制（cursor-based pagination），而非自行用 `created_time` 做時間切割。

---

## 原因 3：前端合併分頁資料與更新留言，導致 created_time 錯誤

**問題描述：**
前端在組裝留言資料時，將「分頁歷史留言」與「即時更新留言」合併處理，導致拿到錯誤的 `created_time` 去拉歷史留言，造成整段時間區段的留言被遺漏。

**Action Item：**
以分頁留言資料呈現聊天室內容；即時更新留言只做狀態更新（例如：標記已讀、顯示處理結果），不干涉分頁游標邏輯。
