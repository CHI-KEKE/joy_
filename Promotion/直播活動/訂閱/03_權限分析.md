# 必要權限分析

## 三個必要權限

要能完整使用直播功能，粉專管理員帳號必須擁有以下三個 Facebook 頁面權限：

| 權限代碼 | 顯示名稱 | 對應功能 |
|----------|----------|----------|
| `CREATE_CONTENT` | 內容 | Webhook 訂閱的基本前提，發布貼文與管理內容 |
| `MODERATE` | 社群動態 | 直播間發訊息（公開留言回覆）|
| `MESSAGING` | 訊息和來電 | 顧客收到加購私訊（Messenger）|

### 缺少各權限的影響

| 缺少的權限 | 症狀 |
|------------|------|
| `CREATE_CONTENT` | Webhook 訂閱直接失敗，整個功能無法啟用 |
| `MODERATE` | 訂閱成功，但直播間發訊息功能失敗 |
| `MESSAGING` | 訂閱成功，但顧客收不到加購訊息（私訊）|

> 只有 `CREATE_CONTENT` 或 `MODERATE` 其中一個時，可以完成訂閱，但部分功能會無法使用。
> **要完整使用直播功能，三個權限缺一不可。**

## 現況 vs. 目標差距分析

### 現況問題

- 呼叫 `POST /{page-id}/subscribed_apps` 時，若 FB 回傳 error code 200，系統統一拋出 `FACEBOOK_PAGE_INSUFFICIENT_PERMISSION`
- 三個權限（`CREATE_CONTENT` / `MODERATE` / `MESSAGING`）**目前皆未主動檢查**
- 使用者只看到通用錯誤，無法判斷缺少哪個權限，也不知道如何修正

### 目標

在綁定或連結直播時，**主動檢查粉專管理員的權限狀態**，並提供明確提示。

## 改善方向

### 後端

1. `GetPageAccountsAsync` 呼叫時，加入 `tasks` field 以取得每個粉專的權限資訊

   ```
   GET /me/accounts?fields=id,name,tasks
   ```

2. `ShopPageBinding` DynamoDB 資料模型加入 `PageTasks` 欄位，記錄每個粉專的授權任務清單

3. 綁定流程加入權限驗證邏輯：
   - 檢查是否具備 `CREATE_CONTENT`、`MODERATE`、`MESSAGING` 三個任務
   - 若缺少任一權限，回傳具體缺少的權限代碼，而非通用錯誤

### 前端

在粉絲專頁清單 UI 中：
- 若粉專缺少 `MODERATE` 或 `MESSAGING` 權限，顯示對應的提示圖示或警告文字
- 引導使用者至 Facebook 頁面設定補齊所需權限，再重新綁定
