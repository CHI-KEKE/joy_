# 為什麼要做 91APP 直播帶貨

## 91APP 在這中間扮演什麼角色

Facebook 只負責「直播」，不負責「電商流程自動化」。91APP 負責補上這段。

---

## 真正的「直播帶貨」流程

| 步驟 | 執行方 | 說明 |
|------|--------|------|
| 1. 商家在 Facebook 開直播 | Facebook | Facebook 原生功能 |
| 2. 客人留言關鍵字（例如「A1+1」）| 客人 | |
| 3. Facebook 即時通知 91APP | Facebook → 91APP | 需事先授權 Webhook |
| 4. 91APP 解析留言 | **91APP** | |
| 5. 91APP 建立購物車、產生專屬下單連結 | **91APP** | |
| 6. 91APP 透過 Messenger 私訊客人連結 | **91APP** | |
| 7. 客人點連結 → 進商城完成下單、付款 | 客人 + 91APP | |

> 步驟 3～7，Facebook 完全不會幫你做。

---

## 為什麼一定要「綁定粉專」

步驟 3 需要 Facebook **主動推送**留言事件給 91APP。

Facebook 的規則：「你要我把某個粉專的留言事件推給你，你必須先取得該粉專的正式授權。」

這個授權動作就是「**綁定粉專**」。

> 綁定粉專 = 讓 Facebook 願意把「直播現場的即時事件」交給 91APP 處理。

---

## 如果只用 Facebook，商家會卡在哪裡

商家在 Facebook 開直播很容易，但會面臨以下問題：

- 顧客留言「A1+1」後，怎麼自動變成「購物車 + 訂單」？
- 留言爆量時，怎麼做到「即時」抓留言、判斷關鍵字？
- 怎麼自動私訊給顧客「你的加車連結」？
- 怎麼追蹤這場直播帶來多少訂單、轉換率、哪些商品賣得好？

## 閱讀這個系統的建議順序

### Step 1：先理解整條用戶旅程

| 角色 | 旅程 |
|------|------|
| 商家 | 綁粉專 → 建活動 → 設商品 / 關鍵字 → 開播 |
| 顧客 | 留言關鍵字 → 收私訊 → 點連結 → 加車結帳 |

### Step 2：再理解系統與 Facebook 的互動介面

| 介面 | 用途 |
|------|------|
| OAuth 授權 | 取得 Page Access Token |
| Webhook | 即時接收留言事件 |
| Messenger API | 發送購物車私訊 |
| Live API | 連結直播、抓狀態 |

### Step 3：最後才進程式碼

從「粉專綁定模組」開始閱讀是最合理的切入點。
