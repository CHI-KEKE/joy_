# 直播訊息文案運作說明

## 概述

直播訊息文案的流程分為兩個階段：
1. **設定期**：使用者在後台設定訊息模板（含變數佔位符），存入 DynamoDB
2. **發送期**：觸發事件時，後端讀取模板，替換變數後送出

換行排版規則：
- **模板文字之間的換行** → 由前端模板字串控制（使用者可編輯）
- **清單型變數內部各項目的換行** → 由後端組裝時控制

## 一、推薦商品訊息

### 涉及變數
| 變數 | 說明 | 換行規則 |
|---|---|---|
| `{{LIVE_NAME}}` | 直播名稱 | 無強制換行 |
| `{{PRODUCT_INFO_AND_KEYWORDS}}` | 商品名稱 + 各 SKU 規格與關鍵字 | 各項目以 `\n` 分隔，前後無強制換行 |

### 完整鏈路

```
Step 1：使用者在「新增/編輯直播活動」設定「商品推薦訊息」模板
        例如：{{LIVE_NAME}}推薦商品\n{{PRODUCT_INFO_AND_KEYWORDS}}\n下單規則留言「關鍵字+數量」
        ↓
        前端讀取 i18n 預設值（由 messageDefaults.ts getDefaultProductRecommendationText() 提供）
        使用者可修改，按儲存後透過 API 存入 DynamoDB

Step 2：主播在直播間點擊「推薦商品」按鈕
        ↓
        前端發 API：POST /api/live/sessions/{liveSessionId}/products/{productId}/recommend

Step 3：WebAPI 處理
        LiveSessionsController.cs L811  RecommendProduct()
        ↓
        LiveSessionService.cs L897  RecommendProductAsync()
          ├── 驗證直播場次狀態必須為 Live
          ├── 取得商品 SKU 資訊（skuQueryService）
          ├── BuildProductInfoKeywordAsync()  L1721
          │     → 組裝 {{PRODUCT_INFO_AND_KEYWORDS}} 的實際內容
          │     → 格式：商品名稱\nSKU1 - $價格 | 關鍵字：K1\nSKU2 - $價格 | 關鍵字：K2
          │     → 程式碼：string.Join("\n", lines)  （各行以 \n 分隔，前後無強制換行）
          ├── 從 DynamoDB 讀取 messageTemplates.productRecommendation.message 模板
          └── BuildFinalMessage()  L1786
                → Replace("{{LIVE_NAME}}", liveSession.Title)
                → Replace("{{PRODUCT_INFO_AND_KEYWORDS}}", productInfoKeyword)

Step 4：透過 Facebook API 發送公開留言
        platformProvider.PostCommentAsync()
```

### 實際輸出範例

模板：
```
{{LIVE_NAME}}推薦商品\n{{PRODUCT_INFO_AND_KEYWORDS}}\n下單規則留言「關鍵字+數量」
```

後端產生的 `{{PRODUCT_INFO_AND_KEYWORDS}}`：
```
潮流T恤
S - $200 | 關鍵字：S1
M - $220 | 關鍵字：M1
```

最終送出：
```
Amy的直播推薦商品
潮流T恤
S - $200 | 關鍵字：S1
M - $220 | 關鍵字：M1
下單規則留言「關鍵字+數量」
```

## 二、加入購物車訊息

### 涉及變數
| 變數 | 說明 | 換行規則 |
|---|---|---|
| `{{LIVE_NAME}}` | 直播名稱 | 無強制換行 |
| `{{PRODUCT_INFO}}` | 商品名稱 + 購買規格與數量 | 各項目以 `\n` 分隔，前後無強制換行，末尾無多餘空行 |
| `{{ADD_TO_CART_LINK}}` | 加入購物車連結 URL | 無強制換行 |

### 完整鏈路

```
Step 1：使用者在「新增/編輯直播活動」設定「加入購物車訊息」模板
        例如：感謝您於{{LIVE_NAME}}活動購物!\n{{PRODUCT_INFO}}\n商品加入購物車連結:\n{{ADD_TO_CART_LINK}}
        ↓
        前端讀取 i18n 預設值（由 messageDefaults.ts getDefaultAddToCartMessageText() 提供）
        使用者可修改，按儲存後透過 API 存入 DynamoDB

Step 2：粉絲在 Facebook 直播留言區輸入關鍵字下單
        ↓
        Facebook Webhook 觸發，訊息進入 NMQ 佇列

Step 3：Worker 處理留言
        ProcessLiveCommentJob.cs L211  ProcessAsync()
        ↓
        L264  CreateOrderAndSendNotificationAsync()
          ├── 解析留言中的商品關鍵字
          ├── 建立訂單
          ├── CartUrlBuilder.BuildAsync()  → 生成加入購物車連結
          └── PrepareAndSendNotificationAsync()  L1226
                ↓
                GetFormattedNotificationMessageAsync()  L1275
                  ├── MessageTemplateService.GetTemplateAsync()  L54
                  │     → 先查 Redis（key: live_session:{id}:templates，TTL: 180s）
                  │     → miss 時從 DynamoDB 讀取 messageTemplates.cartMessage.message
                  └── NotificationService.FormatNotificationMessageAsync()  L199
                        ├── FormatProductInfoAsync()  L121
                        │     → 組裝 {{PRODUCT_INFO}} 的實際內容
                        │     → 格式：商品名稱\nSKU規格 - $價格 x數量
                        │     → 程式碼：sb.AppendLine(商品名稱)、sb.AppendLine(規格)
                        │     → 末尾用 .TrimEnd() 移除多餘空白
                        ├── Replace("{{LIVE_NAME}}", liveSession.Title)
                        ├── Replace("{{PRODUCT_INFO}}", productInfo)
                        └── Replace("{{ADD_TO_CART_LINK}}", cartUrl)

Step 4：透過 NMQ 發送 Facebook 私訊給粉絲
        SendNotificationTaskAsync()  L1261
```

### 實際輸出範例

模板：
```
感謝您於{{LIVE_NAME}}活動購物!\n{{PRODUCT_INFO}}\n商品加入購物車連結:\n{{ADD_TO_CART_LINK}}
```

後端產生的 `{{PRODUCT_INFO}}`（TrimEnd 後）：
```
潮流T恤
S規格 - $200 x1
```

最終送出：
```
感謝您於Amy的直播活動購物!
潮流T恤
S規格 - $200 x1
商品加入購物車連結:
https://cart.url
```

## 三、抽獎相關訊息

### 3.1 開獎訊息（公開留言）

#### 涉及變數
| 變數 | 說明 | 換行規則 |
|---|---|---|
| `{{ACTIVITY_NAME}}` | 抽獎活動名稱 | 無強制換行 |
| `{{WINNER_LIST}}` | 得獎者名單（頓號串接） | 同一行顯示，無強制換行 |

#### 鏈路
```
主辦者執行開獎
        ↓
LotteryWinnerNotificationService.cs  SendDrawMessageToLiveAsync()
  ├── 從 DynamoDB 讀取抽獎活動的 drawMessage 模板
  ├── 組裝得獎者名單：string.Join("、", winners)
  │     → 得獎者以頓號串接在同一行
  └── Replace("{{ACTIVITY_NAME}}", activityName)
      Replace("{{WINNER_LIST}}", winnerNames)
        ↓
透過 Facebook API 發送公開留言
```

#### 輸出範例
```
🎊夏日抽獎開獎結果公佈！🏆恭喜得獎者：Amy、Bob、Cindy
```

---

### 3.2 中獎私訊（每位得獎者各收一則）

#### 涉及變數
| 變數 | 說明 | 換行規則 |
|---|---|---|
| `{{ACTIVITY_NAME}}` | 抽獎活動名稱 | 無強制換行 |
| `{{WINNER_NAME}}` | 得獎者名稱 | 無強制換行 |
| `{{COUPON_CODE}}` | 優惠券代碼 | 無強制換行 |

#### 鏈路
```
LotteryWinnerNotificationService.cs  SendWinnerPrivateMessagesAsync()
  ├── 從 DynamoDB 讀取 winnerMessage 模板
  └── 逐一替換每位得獎者的變數後送出私訊
```

## 五、換行設計原則

```
┌─────────────────────────────────────────────┐
│  前端模板字串                                │
│  例如：感謝您於{{LIVE_NAME}}活動購物!\n      │
│        {{PRODUCT_INFO}}\n                   │
│        商品加入購物車連結:\n                 │
│        {{ADD_TO_CART_LINK}}                 │
│                                             │
│  ← 前後換行由前端模板的 \n 控制             │
└─────────────────────────────────────────────┘
                      ↓ 存入 DynamoDB
┌─────────────────────────────────────────────┐
│  後端清單型變數內容                          │
│                                             │
│  {{PRODUCT_INFO_AND_KEYWORDS}}              │
│  商品名稱\nSKU1\nSKU2                       │
│                                             │
│  {{PRODUCT_INFO}}                           │
│  商品名稱\nSKU規格 - $價格 x數量（TrimEnd）  │
│                                             │
│  {{WINNER_LIST}}                            │
│  Amy、Bob、Cindy（頓號串接，單行）           │
│                                             │
│  ← 清單內部換行由後端組裝邏輯控制            │
└─────────────────────────────────────────────┘
```

**結論：後端不主動在清單變數前後加換行，排版完全由前端模板的 `\n` 位置決定。**
