# 粉絲頁權限不足問題與改善建議

## 三個必要權限

要能完整使用直播功能，粉專管理員帳號必須擁有以下三個 Facebook 頁面權限：

| 權限代碼 | 顯示名稱 | 缺少時的症狀 |
|----------|----------|-------------|
| `CREATE_CONTENT` | 內容 | Webhook 訂閱失敗，功能完全無法啟用 |
| `MODERATE` | 社群動態 | 直播間發訊息失敗 |
| `MESSAGING` | 訊息和來電 | 顧客收不到加購私訊 |

> 只要具備「內容」或「社群動態」其中一個，即可完成 Webhook 訂閱，但部分功能仍無法正常運作。
> **三個權限缺一不可，才能完整使用直播功能。**

---

## 根本原因

呼叫 FB API 取得粉專列表時，未請求 `tasks` 欄位，導致系統無法判斷授權帳號缺少哪個權限。

授權完成後若有權限缺失，系統不會在綁定時告知，只有在直播實際操作時才會遇到功能異常，體驗極差。

---

## 改善方向

### 後端

1. `GetPageAccountsAsync` 加入 `tasks` field，取得各粉專的授權任務清單

   ```
   GET /me/accounts?fields=id,name,tasks
   ```

2. `ShopPageBinding` 資料模型加入 `PageTasks` 欄位，持久化記錄各粉專的授權狀態

3. 綁定流程加入權限驗證邏輯：若缺少必要權限，回傳具體缺少的權限代碼，而非通用錯誤

### 前端

- 粉絲頁清單 UI 中，對缺少 `MODERATE` / `MESSAGING` 的粉專顯示警告提示
- 引導用戶至 Facebook 頁面設定補齊所需權限，再重新綁定

### 不需改動

- OAuth 授權流程本身
- Webhook 訂閱邏輯
