# 連結 Facebook 直播

## 這個功能的意義

「你先在 Facebook 開好一場直播，系統只是去『連接』那一場已存在的 FB 直播。」

這個功能的本質，是把「系統裡定義的直播活動」對齊到「真正在 FB 發生的那一場直播」，讓後續所有自動化行為（留言監聽、商品推薦、狀態同步）都有一個正確、唯一的對象可以依附。

---

## 商家的操作視角

1. 直播主先在 Facebook 開直播（在粉專直接開）或排程一場直播 👉 **這一步完全不經過系統**
2. 系統裡只是有一個「直播活動」，它本身不會開播，只是一個「設定容器」
   - 裡面放：商品、關鍵字、互動規則
3. 點擊「連接 FB 直播」，系統去 FB 問：「你這個粉專，現在有哪些直播？」，使用者從清單選一場，系統記住這個對應關係
4. **系統直播活動 ⇄ FB 那一場直播**，從此之後：這場 FB Live 的留言才會套用「這個活動」的商品與規則

---

## 前置條件

連結前系統會先檢查是否已綁定 Facebook 粉專：
- ✅ 有 → 開啟彈窗進行選擇
- ❌ 無 → 顯示警告 Dialog，引導先完成粉專綁定

---

## 操作流程

```
1. 使用者點擊「連接FB直播」
   ↓
2. 前端檢查是否有綁定粉專
   ├─ ✅ 有 → 開啟彈窗
   └─ ❌ 無 → 顯示警告 Dialog，引導去綁定粉專
   ↓
3. 步驟 2-1：選擇粉專（下拉選單）
   ↓ 觸發 API
   GET /api/live/sessions/{id}/facebook-pages/{pageId}/live-videos
   ↓
4. 步驟 2-2：顯示直播列表（LIVE + SCHEDULED）
   每個項目包含：標題、狀態徽章、開始時間、外部連結、已連結提示
   ↓
5. 選擇直播 + 點擊「連結」
   ↓ 觸發 API
   POST /api/live/sessions/{id}/link-facebook
   {
     facebookPageId: "123",
     facebookLiveVideoId: "456"
   }
   ↓
6. 後端處理：
   a. 驗證場次狀態（未連結、未結束）
   b. 驗證粉專綁定（取得 Token）
   c. 呼叫 Facebook Graph API 取得直播資訊
   d. 判斷 PostId（用於後續留言）
   e. 組織連結資料
   f. 儲存到 DynamoDB
   ↓
7. 前端接收回應：顯示成功訊息、暫存資料、重新載入 session、關閉彈窗
   ↓
8. 顯示直播畫面（使用 Facebook Embed HTML）
   ↓
9. Webhook 自動同步狀態：
   Facebook 直播開始/結束
   → Facebook 推送 Webhook
   → 後端更新系統狀態
   → 前端自動重新載入（透過 SWR）
```

### 步驟 3：選擇粉專（前端）

- 位置：`VideoPlayerArea.tsx:127`
- 顯示粉專下拉選單，使用者選擇後自動觸發 `useFacebookLiveStreams` Hook
- 呼叫後端 API：`GET /api/live/sessions/{liveSessionId}/facebook-pages/{pageId}/live-videos`

每個直播項目包含：
- ✅ 標題與描述
- ✅ 狀態徽章（直播中 / 已排程）
- ✅ 開始時間
- ✅ 外部連結圖示（開啟 Facebook 直播頁面）
- ⚠️ 已連結提示（如果該直播已被其他場次使用）

### PostId 判斷邏輯

```csharp
private string GetCommentTargetId(FacebookVideoEntity videoEntity)
{
    // 1. PostId 存在且不是 "0" → 傳統貼文，使用 PostId
    // 2. 否則 → 原生影片模式，使用 VideoId
    if (!string.IsNullOrEmpty(videoEntity.PostId) && videoEntity.PostId != "0")
        return videoEntity.PostId;
    
    return videoEntity.Id; // Fallback 到 Video ID
}
```

### 步驟 4：轉換直播開始時間為 UTC

```csharp
DateTime? liveStartTime = null;
```

## Webhook 自動同步直播狀態

```csharp
public async Task HandleLiveVideoStatusChangeAsync(string videoId, string status, ...)
{
    // 1. 查詢哪些 LiveSession 連結了這個 Facebook 直播
    var liveSessions = (await liveSessionRepository.GetActiveLiveSessionsAsync())
        .Where(session => session.PlatformLiveId == videoId)
        .ToList();
    
    if (liveSessions.Count == 0) return; // 找不到，忽略
    
    // 2. 轉換狀態
    var targetStatus = ConvertFacebookStatusToLiveSessionStatus(status);
    var shouldClearLink = (status == "VOD" || status == "UNPUBLISHED");
    
    // 3. 批次更新
    foreach (var session in liveSessions)
    {
        if (session.Status != targetStatus)
        {
            session.Status = targetStatus;
            if (targetStatus == "Ended") session.EndTime = DateTime.UtcNow;
            await liveSessionRepository.UpdateLiveSessionAsync(session);
        }
        
        if (shouldClearLink)
            await liveSessionRepository.ClearFacebookLiveLinkAsync(session.LiveSessionId);
    }
}
```

### Facebook 狀態對應表

| Facebook 狀態 | 系統狀態 | 說明 |
|--------------|---------|------|
| `LIVE` | `Live` | 直播中 |
| `VOD` | `Ended` | 直播已結束（變成重播）|
| `SCHEDULED_LIVE` | `Scheduled` | 已排程 |
| `SCHEDULED_UNPUBLISHED` | `Scheduled` | 已排程（未發布）|
| `UNPUBLISHED` | `Ended` | 已下架 |
