# 啟用粉專綁定與管理

## 兩階段綁定說明

OAuth 完成後，粉專資料存入 DynamoDB 但 `IsEnabled = false`（尚未啟用）。  
使用者需再手動勾選並點擊「連結」，系統才將 `IsEnabled` 設為 `true`，粉專才正式可用。

```
OAuth 完成 → IsEnabled = false（資料存入，未啟用）
使用者勾選粉專 + 點「連結」→ IsEnabled = true（正式啟用）
```

## 步驟 10：使用者啟用粉專綁定

前端開啟粉專選擇彈窗（`FanPageSelectionModal`），顯示可用粉專清單（已排除已綁定的）。

使用者勾選粉專後點「確認」，前端呼叫：

```
POST /api/shop-page-bindings
{
  "shopId": "123",
  "platformPageIds": ["123456789", "987654321"],
  "platform": "Facebook"
}
```

後端處理（`ShopPageBindingService.CreateBindingAsync`）：

```csharp
[HttpPost]
public async Task<ActionResult> CreateBinding([FromBody] CreateShopPageBindingRequest request)
{
    var bindings = await this.shopPageBindingService.CreateBindingAsync(request);   
    return this.CreatedAtAction(..., bindings);
}

public async Task<IReadOnlyCollection<ShopPageBinding>> CreateBindingAsync(...)
{
    foreach (var platformPageId in platformPageIds)
    {
        //// 1. 從 DynamoDB 讀取已存在的綁定
        var entity = await this.repository.GetByIdAsync(shopId, platform, platformPageId);
        
        //// 2. 解密 Token
        this.Decrypt(entity);
        
        //// 3. 啟用綁定
        entity.IsEnabled = true;  // ← 這裡！
        
        //// 4. 加密後更新到 DynamoDB
        this.Encrypt(entity);
        await this.repository.UpdateAsync(entity, userId);
        this.Decrypt(entity);
    }
}
```

## 完整流程圖（OAuth → 啟用）

```
1. 使用者進入「直播串流設定」頁面 (/stream-settings)
   ↓
2. 使用者點擊「帳號登入」或「+ 新增粉絲專頁」按鈕
   ↓
3. 前端呼叫後端 API:
   GET /api/facebook/oauth/authorize?shopId={shopId}
   ↓
4. 後端 (FacebookOAuthController.Authorize):
   - 產生加密的 State (含 shopId, userId, timestamp, nonce)
   - 組合 Facebook OAuth URL (包含 client_id, redirect_uri, scope)
   - 302 重導向至 Facebook
   ↓
5. Facebook OAuth Dialog (在新視窗開啟):
   - 使用者登入 Facebook 帳號
   - Facebook 顯示授權確認頁面
   - 使用者選擇要授權的粉絲專頁
   - 使用者點擊「確認」
   ↓
6. Facebook 回調系統:
   GET /api/facebook/oauth/callback?code={授權碼}&state={State}
   ↓
7. 後端 (FacebookOAuthController.Callback):
   - 驗證 State 參數（防 CSRF）
   - 用授權碼交換 User Access Token
   - 呼叫 Facebook Graph API: GET /me/accounts
   - 取得粉專列表 + Page Access Token
   ↓
8. 後端 (SaveBindingsAsync):
   - 遍歷每個粉專，建立 ShopPageBinding 實體
   - 儲存資料：
     • ShopId (Partition Key)
     • Platform + PlatformPageId (Sort Key)
     • PageName, PageAccessToken (加密), PicUrl
     • BindingStatus = "Active"
     • IsEnabled = false（預設關閉）
     • WebhookSubscribed = false
   - ShopPageBindingService.AddBindingAsync() → 加密 → DynamoDB
   ↓
9. 後端導回前端回調頁面:
   Redirect: /oauth/facebook/callback?oauth=success&shopId={shopId}
   ↓
10. 前端 (FacebookCallbackPage):
    - 顯示「授權成功」訊息
    - 通知主視窗 (postMessage)
    - 1.5 秒後關閉彈窗
    ↓
11. 主視窗 (StreamSettingsPage):
    - 接收 postMessage
    - 顯示 Toast「帳號登入成功」
    - 刷新粉專列表 (refreshBindings, refreshTokenStatus)
    ↓
12. 使用者點擊「+ 新增粉絲專頁」:
    - 開啟粉專選擇彈窗 (FanPageSelectionModal)
    - 顯示可用粉專列表（已排除已綁定的）
    ↓
13. 使用者勾選粉專，點擊「確認」:
    - 前端呼叫 POST /api/shop-page-bindings
    - Body: { shopId, platformPageIds: [...], platform: "Facebook" }
    ↓
14. 後端 (ShopPageBindingsController.CreateBinding):
    - ShopPageBindingService.CreateBindingAsync()
    - 讀取 DynamoDB 中的綁定記錄
    - 將 IsEnabled 設為 true ← 啟用綁定
    - 更新到 DynamoDB
    ↓
15. 前端顯示「連結成功」Toast
    - 刷新綁定列表
    - 粉專卡片顯示在頁面上
    ↓
16. 完成！粉專已綁定並啟用
    DynamoDB 中的資料:
    • BindingStatus: "Active"
    • IsEnabled: true
    此時可在直播間連接 Facebook 直播使用該粉專
```

## 解除綁定 / 重新連結

- **解除綁定**：後端標記 `BindingStatus = "Disconnected"`（不是硬刪）
- **重新連結**：重新走一次 Facebook OAuth 流程

---

## 安全性設計

| 機制 | 說明 |
|------|------|
| CSRF 防護 | State 參數包含 HMAC 簽章 |
| 重放攻擊防護 | State 有 10 分鐘時效 |
| Token 安全 | PageAccessToken / UserAccessToken 加密後才存入 DynamoDB |

## OAuth Scope 說明

| Scope | 用途 |
|-------|------|
| `pages_manage_posts` | 發佈留言、推薦商品 |
| `pages_read_engagement` | 讀取留言、互動資料 |
| `pages_show_list` | 取得粉專列表 |
