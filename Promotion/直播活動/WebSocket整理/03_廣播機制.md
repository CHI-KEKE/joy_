# WebSocket 廣播機制

## 3️⃣ 推送錯誤處理

#### 推送失敗

- 記錄失敗 connectionId，不影響其他連線的推送繼續進行
- 外層 try/catch 保護整個廣播流程，不影響 webhook → NMQ 的後續處理

#### GoneException 案例

AWS API Gateway 在連線已斷但 $disconnect 未觸發時，呼叫 `PostToConnectionAsync` 會拋出 `GoneException`。

```
websocket: connection : ZviBtfe6yQ0CHjw=
處理 WebSocket 連線中斷失敗
完成發送訊息到 ConnectionId
發送訊息到 WebSocket 發生錯誤
```

處理邏輯：捕捉 GoneException 後，將該 connectionId 加入失敗清單，廣播流程繼續推送其餘連線。廣播完成後統一呼叫 Lazy Cleanup 清除殭屍連線（見 `02_連線管理.md` 第 5 節）。
