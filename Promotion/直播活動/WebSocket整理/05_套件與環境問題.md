# WebSocket 套件與環境問題

## AWSSDK 版本衝突問題

### 問題摘要

引入 `AWSSDK.ApiGatewayManagementApi` 套件後，導致既有的 `AWSSDK.DynamoDBv2` 套件出現執行期例外。

### 錯誤訊息

```bash
InnerException:
"Method 'get_ServiceFullName' in type 'Amazon.DynamoDBv2.Internal.AmazonDynamoDBMetadata'
from assembly 'AWSSDK.DynamoDBv2, Version=4.0.0.0'
does not have an implementation."

主例外:
"The type initializer for 'Amazon.DynamoDBv2.AmazonDynamoDBClient' threw an exception."
```

### 發生錯誤的位置與順序

1. `Nine1.Livebuy.DA.Repositories.DynamoDB.LiveSessionRepository.cs: line 505`
   → 呼叫 `AmazonDynamoDBClient.ScanAsync()` 查詢直播資料

2. `Nine1.Livebuy.BL.Services.FacebookIntegration.FacebookWebhookService.cs: line 142`
   → 呼叫上層的 `LiveSessionRepository.GetActiveLiveSessionsAsync()`，在處理 Facebook webhook 時發生

### 原因分析

- 使用的 `AWSSDK.DynamoDBv2` 版本（Version=4.0.0.0）不完整或出現破壞性變更
- `AmazonDynamoDBMetadata` 中的 `get_ServiceFullName` 方法在反射或執行時期找不到對應實作
- 可能原因：
  - SDK 更新後某些函式接口改變但 DLL 沒有同步更新
  - `AWSSDK.Core` 未一併升級，導致相依性不一致

### 解法

> **新套件版本不要裝太新！**

`AWSSDK.ApiGatewayManagementApi` 與 `AWSSDK.DynamoDBv2` 需確保版本相容。
升級任一 AWS SDK 套件時，務必同步確認 `AWSSDK.Core` 版本一致。
