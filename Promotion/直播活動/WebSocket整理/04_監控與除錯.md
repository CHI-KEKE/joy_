# WebSocket 監控與除錯

## Loki 查詢語法

### 基本服務設定

```bash
# QA 環境
prod : {service="prod-91app-live"}
qa   : {service="hk-qa-91app-live"}

# group
91app-live.group1
```

監控 Dashboard：
`https://monitoring-dashboard.91app.io/explore`

### 用 ConnectionId 追蹤單一連線

```bash
{service="hk-qa-91app-live"}
|json
|= `ZvjyHdMGSQ0CHgA=`   # ← 替換為目標 connectionId
|json
| line_format "{{._msg}}"
```

### WebSocket 關鍵字查詢

用於排查推送失敗、連線中斷問題：

```bash
## 關鍵字說明：
## 發送訊息到 WebSocket 發生錯誤  ← GoneException / 推送失敗
## 處理 WebSocket 連線中斷失敗    ← $disconnect 處理異常
## 完成發送訊息到 ConnectionId    ← 正常推送成功確認

{service="hk-qa-91app-live"}
|json
|= `完成發送訊息到 ConnectionId`
|json
| line_format "{{._msg}}"
```

---

### 用 RequestId 追蹤單次請求

```bash
{service="hk-qa-91app-live"} | json | container = "nine1-livebuy-web-api"
|~ `.*.*`
|~ "0HNJQIKGKLROT:0000001D"   # ← 替換為目標 requestId
| Job =~ `.*.*`
| _msg =~ `.*.*`
| line_format "{{._msg}}"
```

---

### 查 WebAPI log

```bash
{container="nine1-livebuy-web-api",service="prod-91app-live"}
|json
|= `live/comments/sync`
|json
| line_format "{{._msg}}"
```

## GoneException 排查流程

1. 用關鍵字 `發送訊息到 WebSocket 發生錯誤` 找到錯誤 log
2. 從 log 取出 `connectionId`（例如 `ZviBtfe6yQ0CHjw=`）
3. 用上方「ConnectionId 追蹤」語法查看該連線的完整生命週期
4. 確認是否有對應的 `處理 WebSocket 連線中斷失敗`（$disconnect 有無正常觸發）
5. 若 $disconnect 未觸發，屬於正常的殭屍連線，會被 Lazy Cleanup 清除（見 `02_連線管理.md` 第 5 節）
