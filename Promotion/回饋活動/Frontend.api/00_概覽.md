# 回饋活動 — 概覽

## 業務定義

「回饋活動」在程式碼中對應以下三種 `PromotionEngineTypeDefEnum`：

| 類型 | 說明 |
|------|------|
| `RewardReachPriceWithPoint2` | 滿額固定給點（V2） |
| `RewardReachPriceWithRatePoint2` | 滿額依比例給點（V2） |
| `RewardReachPriceWithCoupon` | 滿額給券 |

> ⚠️ `RewardReachPriceWithPoint`（V1）和 `RewardReachPriceWithRatePoint`（V1）是「給點活動」，**不屬於**回饋活動。

---

## 執行位置

回饋活動定義在 **BottomGroup**（Priority = 70），在所有折扣活動算完之後才執行：

```
ProcessRepository.cs (Line 83-85)
├── RewardReachPriceWithPoint2     Priority 90000
├── RewardReachPriceWithRatePoint2 Priority 90000
└── RewardReachPriceWithCoupon     Priority 90000
```

**特性：**
- 不產生折扣金額（Amount = 0）
- 只產生 `PromotionRecord`，記錄活動是否被觸發
- 計算結果為「達成」或「未達成（含 Lack 原因）」

## 給點 vs 給券核心差異

| 特性 | Point2（固定給點） | RatePoint2（比例給點） | Coupon（給券） |
|------|:-:|:-:|:-:|
| 給點公式 | 固定 Point | Rate × 金額，上限 MaxRewardPoint | EcouponId + Qty |
| 給點效期（PointUntil） | ✅ | ✅ | ❌（由券系統管） |
| GetPromotionEngineRule | ✅ 完整實作 | ✅ 完整實作 | ❌ return string.Empty（TODO） |
| SetPromotionEngineForCart | ❌ 空實作 | ❌ 空實作 | ❌ 暫時不實作 |
| SetCustomField1（IsDisplayInShoppingCart） | ✅ | ✅ | ✅ |
| SetCustomField2（Quota） | ✅ | ✅ | ✅ |
