# 回饋活動 — 給點 vs 給券差異

## 三種類型對照

| 特性 | Point2（固定給點） | RatePoint2（比例給點） | Coupon（給券） |
|------|:-:|:-:|:-:|
| Rule Service 類別 | `RewardReachPriceWithPoint2RuleService` | `RewardReachPriceWithRatePoint2RuleService` | `RewardReachPriceWithCouponRuleService` |
| 引擎 Rule 類別 | `RewardReachPriceWithPoint2` | `RewardReachPriceWithRatePoint2` | `RewardReachPriceWithCoupon` |

---

## 條件門檻計算邏輯

### Point2（固定給點）
```
滿 [Condition] 元 → 給 [Point] 點
```

Rule JSON `Thresholds`:
```json
{
  "AllUserScope": {
    "ReachPricePointPairs": [{ "ReachPrice": 1000, "Point": 50 }]
  }
}
```
可依會員等級（`CrmShopMemberCard:xx`）設不同給點：
```json
{
  "CrmShopMemberCard:3": { "ReachPricePointPairs": [{ "ReachPrice": 1000, "Point": 100 }] },
  "AllUserScope":        { "ReachPricePointPairs": [{ "ReachPrice": 1000, "Point": 50 }] }
}
```

### RatePoint2（比例給點）
```
滿 [Condition] 元 → 給消費金額 × [Rate]% 點，上限 [MaxRewardPoint] 點
```

Rule JSON `Thresholds`:
```json
{
  "AllUserScope": {
    "ReachPriceRatePointPairs": [{ "ReachPrice": 500, "Rate": 0.05 }],
    "MaximumPoints": 200
  }
}
```

額外欄位：`PointCalculateType`（`RewardPointCalculateType` enum），決定點數計算方式。

### Coupon（給券）
```
滿 [Condition] 元 → 給 [Qty] 張 [EcouponId] 號優惠券
```

Rule JSON `Thresholds`:
```json
{
  "AllUserScope": {
    "ReachPriceCouponPairs": [{
      "ReachPrice": 800,
      "Coupons": [{ "Id": "12345", "Qty": 1 }]
    }]
  }
}
```

---

## 給點效期（PointUntil）

Point2 和 RatePoint2 有效期設定，Coupon 無此設定（由券系統管理）：

| UntilType | 說明 | 額外欄位 |
|-----------|------|---------|
| `AfterDays` | 發點後 N 天到期 | `AfterDays` |
| `UntilTheEndOfYear` | 到某年某月底 | `UntilYearOffset`, `UntilYearMonth` |
| `FixedDate` | 固定到期日 | `FixedDate` |
| `SameWithMemberAccountLevel` | 同會員帳號等級效期 | — |

---

## GetPromotionEngineRule 實作狀態

| 類型 | 狀態 |
|------|------|
| Point2 | ✅ 完整實作，組裝 Thresholds + PointUntil |
| RatePoint2 | ✅ 完整實作，組裝 Thresholds + PointUntil + PointCalculateType |
| Coupon | ❌ `return string.Empty;`（TODO 未實作） |

> Coupon 的 `GetPromotionEngineRule` 直接回傳空字串，表示**給券活動目前無法由後台重新組裝 Rule JSON**，依賴既有存入 DB 的 Rule。

## SetPromotionEngineForCart 實作狀態

所有三種類型 **皆為空實作或暫時不實作**：

```csharp
// Point2 / RatePoint2
public override void SetPromotionEngineForCart(string extendRule, PromotionEngineForCartResponseEntity cartPromotion)
{
    // 空
}

// Coupon
public override void SetPromotionEngineForCart(string extendRule, PromotionEngineForCartResponseEntity cartPromotion)
{
    //// 暫時不實作
}
```

## 購物車顯示設定（SetCustomField）

三種類型皆實作 `SetCustomField1ForCart` 和 `SetCustomField2ForCart`：

```csharp
// SetCustomField1：IsDisplayInShoppingCart
public override void SetCustomField1ForCart(string customFiled, PromotionEngineForCartResponseEntity cartPromotion)
{
    bool.TryParse(customFiled.Split(":")[1], out var isDisplay);
    cartPromotion.IsDisplayInShoppingCart = isDisplay;
}

// SetCustomField2：Quota（張數/次數限制）
public override void SetCustomField2ForCart(string customFiled, PromotionEngineForCartResponseEntity cartPromotion)
{
    int.TryParse(customFiled.Split(":")[1], out var quota);
    cartPromotion.Quota = quota;
}
```

`IsDisplayInShoppingCart`（預設 `true`）：控制是否在購物車顯示回饋活動資訊。
`Quota`：此活動的張數或使用次數限制。
