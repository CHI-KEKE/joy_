# 回饋活動 — 情境：當月壽星

## 前提

管理後台設定一個「當月壽星限定」的回饋活動：
- `TargetMemberTypeDef = MemberCollection`（或有特殊欄位）
- Rule JSON 中 `IsBirthdayMonthEnabled = true`
- `MatchedUserScopes` 實際在執行期動態填入（不是後台設定時就寫死）

---

## 情境 A：使用者是當月壽星 ✅

**假設**：今天是 3 月，使用者生日是 3 月 15 日

**執行步驟：**

### Step 1：`GetRuleInfoListAsync` 掃描活動規則

```csharp
// PromotionRuleRepository.cs
var isBirthdayMonthEnabled = this.IsPromotionBirthdayMonthEnabled(ruleList);
// → 發現至少一個活動的 Rule JSON 中 IsBirthdayMonthEnabled = true
// → isBirthdayMonthEnabled = true
```

### Step 2：取訂單時間月份

```csharp
// context.Cart.Now 是購物車建立時間（非 DateTime.Now）
var month = context.Cart.Now.Month;  // = 3
```

### Step 3：查 DynamoDB 取當月 CollectionId

```csharp
// MemberCollectionService.GetBirthdayCollectionIdByMonthAsync()
// Key: System_BirthdayMonth_{shopId}
// DynamoDB 存有: { "3": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" }
var birthdayCollectionId = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx";
```

### Step 4：打 Member Collection API 比對

```csharp
// SetMemberCollectionToUserTagAsync()
// → PostMemberCollectionMatchAsync(shopId, memberId, [birthdayCollectionId])
// → API 回傳：使用者符合此 CollectionId（生日在 3 月）
```

### Step 5：注入 Tag

```csharp
// 先把 birthdayCollectionId 加到 User.Tags
// 然後替換成 "CurrentBirthdayMonth"（語意化）
context.Calculate.User.Tags = context.Calculate.User.Tags
    .Select(tag => tag == birthdayCollectionId ? "CurrentBirthdayMonth" : tag)
    .ToArray();
// User.Tags 現在包含 "CurrentBirthdayMonth"
```

### Step 6：引擎計算

Rule JSON 的 `MatchedUserScopes`:
```json
{ "UserScopeType": "TagUserScope", "Tag": "CurrentBirthdayMonth" }
```

`User.Tags` 包含 `"CurrentBirthdayMonth"` → **UserScope match ✅**

接著比對其他 Scope（商品、通路等）全過 → `IsMatch = true`

---

## 情境 B：使用者不是當月壽星 ❌

### Step 1-3：同上，取得 `birthdayCollectionId`

### Step 4：打 Member Collection API 比對

→ API 回傳：使用者**不符合**此 CollectionId（生日不在 3 月）

### Step 5：Tag 不注入

User.Tags 中不包含 `birthdayCollectionId`，也不會有 `"CurrentBirthdayMonth"`

### Step 6：引擎計算

Rule 要求 `Tag = "CurrentBirthdayMonth"`，User.Tags 沒有 → **UserScope not match ❌**

→ `IsMatch = false`，`LackOfUserScope` 有值

---

## 情境 C：DynamoDB 沒有當月 CollectionId（未設定）

```csharp
var birthdayCollectionId = await this._memberCollectionService.GetBirthdayCollectionIdByMonthAsync(...);
// → return null 或 empty string
if (string.IsNullOrEmpty(birthdayCollectionId) == false)
{
    promotionMemberCollectionList.Add(birthdayCollectionId);
}
// → 不加入，正常走一般 MemberCollection 流程
```

→ 壽星 Tag 不注入，活動不 match（視活動其他設定而定）

---

## 關鍵：Request 不帶壽星 Tag

> 外部（Cart Service / Shopping）在打 calculate request 時，**不需要**也**不應該**自己決定是否帶壽星 tag。
> 所有壽星判斷都在 Promotion Frontend 內部動態完成：
> 訂單時間 → DynamoDB 月份 CollectionId → Member Collection API → 動態注入 Tag
