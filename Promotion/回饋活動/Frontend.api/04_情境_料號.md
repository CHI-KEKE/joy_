# 回饋活動 — 情境：料號（OuterId）圈選

## 前提

管理後台設定一個「指定料號 SKU 才能觸發」的回饋活動：
- `TargetTypeDef = Tag`（活動的商品圈選類型）
- 活動設定中有 `IncludedProductSkuOuterIdTags`（料號圈選設定）
- Rule JSON 的 `IncludedProductScopes` 中存的是 **PromotionTagId**（非 OuterId 本身）

---

## 兩條料號比對路徑

| 路徑 | 使用場景 | 資料來源 |
|------|---------|---------|
| `PromotionTagOuterIdRepository` | 一般訂單計算（BottomProcessGroup） | DB (Redis 快取) |
| `S3OuterIdRepository` | 菜籃計算（CustomRuleBasketGroup） | S3 JSON 檔案 |

兩者邏輯相同，差異只在資料取得方式。

---

## 情境 A：商品在料號圈選範圍內 ✅

**假設**：
- 回饋活動設定了料號 `"SKU-ABC-001"` 在圈選範圍內
- 對應 `PromotionTagId = "88888"`
- 使用者購物車中有 OuterId = `"SKU-ABC-001"` 的商品

**執行步驟：**

### Step 1：`GetOuterIdPromotionListAsync` — 取料號圈選活動 Id

```csharp
// PromotionRuleRepository.cs
// 只查三種回饋活動類型（RewardReachPriceWithPoint2 / RatePoint2 / Coupon）
var outerIdPromotionIds = await GetOuterIdPromotionListAsync(shopId);
// → 取得有設定料號圈選的回饋活動 Id 清單
```

### Step 2：`GetOuterIdTagsAsync` — 比對購物車商品 OuterId

```csharp
// PromotionTagOuterIdRepository.cs
// 查 DB：活動 Id → PromotionTagId → 料號清單
// (Redis 快取，Key: PromotionEngine_OuterId_Shop-xxx:{shopId})
var outerIdDictionary = await GetOuterIdDictionaryAsync(shopId, engineIds);
// outerIdDictionary[PromotionTagId] = ["SKU-ABC-001", "SKU-ABC-002", ...]
```

購物車商品 `OuterId = "SKU-ABC-001"` 在 `PromotionTagId = "88888"` 的清單中 → **符合**

### Step 3：`UpdateSalepageTagByOuterId` — 貼 Tag

```csharp
// BaseProcessGroup.cs
context.SalepageSkuItemList
    .Where(i => i.OuterId == sku.OuterId)
    .ForEach(item => item.Tags.Add("88888"));  // 貼上 PromotionTagId
```

### Step 4：引擎計算

Rule JSON 的 `IncludedProductScopes`:
```json
{ "ProductScopeType": "TagProductScope", "Tag": "88888" }
```

商品 Tags 包含 `"88888"` → **ProductScope match ✅**

其他維度通過 → `IsMatch = true`，回饋觸發

---

## 情境 B：商品不在料號圈選範圍 ❌

購物車商品 `OuterId = "SKU-XYZ-999"`，不在任何 PromotionTagId 的料號清單中

→ `UpdateSalepageTagByOuterId` 不貼 Tag

→ 引擎比對：商品 Tags 沒有 `"88888"` → **ProductScope not match ❌**

→ `IsMatch = false`，`LackOfProduct` 有值

---

## 情境 C：菜籃計算（S3 路徑）

菜籃計算走 `CustomRuleBasketGroup`，料號資料從 S3 取：

```csharp
// S3OuterIdRepository.GetOuterIdTagsAsync()
// 1. 取活動清單（只有三種回饋活動類型）
// 2. 查 DB 取 S3 Key（PromotionEngineRuleRecord_S3Key）
// 3. DownloadFromS3Async() 下載 JSON
// 4. 解析 JSON 中的 OuterId → TagId 對應
// 5. 比對購物車商品的 OuterId
// 6. 符合 → 回傳 SaleProductSkuResponseEntity（含 Tags）
```

後續 `UpdateSalepageTagByOuterId` 邏輯與 DB 路徑相同。

---
