# 回饋活動 — 情境：通路（Channel）與門市（Location）

---

# 一、通路（Channel）

## 活動設定

管理後台設定限定通路「Web + AppIOS + AppAndroid」的回饋活動。

`GetRuleObject` 時寫入 Rule JSON：
```csharp
result.MatchedSalesChannels = (int)SalesChannelAdapter.ToSalesChannelEnum(entity.TargetPlatformTypeList);
// 例如 Web | AppIOS | AppAndroid = 7
```

## 情境 A：線上訂單（Web 下單） ✅

**Request：** `Channel = SalesChannelEnum.Web`

**`CreateShoppingCartContext`：**
```csharp
var shoppingCartContext = new ShoppingCartContext(
    ...,
    context.Calculate.Channel,   // = Web
    ...);
```

**引擎計算：**  
Rule `MatchedSalesChannels` = 7（Web | AppIOS | AppAndroid），Cart Channel = Web → **通路 match ✅**

---

## 情境 B：InStore 線下刷機下單 ❌

**Request：** `Channel = SalesChannelEnum.InStore`

**引擎計算：**  
Rule `MatchedSalesChannels` 不包含 InStore flag → **通路 not match ❌**

→ `IsMatch = false`，`LackSalesChannel` 有值

---

## 情境 C：Basket 菜籃計算的通路行為

`PromotionBasketGroup.ModifyRuleByCustomMethodAsync`：

```csharp
foreach (var rules in promotionCommonRuleBase)
{
    rules.PaymentTypes = null;         // ← 清空（無法得知金流）
    rules.ShippingTypes = null;        // ← 清空（無法得知物流）
    rules.IncludeRegionScopes = null;  // ← 清空（無法得知地區）
    // MatchedSalesChannels 不清空 ← 通路維持比對
}
```

> 菜籃計算刻意**保留通路限制**，確保不同通路顯示的活動是正確的。

# 二、門市（Location）

## 活動設定

管理後台設定「限定 A、B 兩間門市」的回饋活動：

Rule JSON 的 `IncludedLocationScopes`：
```json
[
  { "LocationScopeType": "TagLocationScope", "Tag": "TagId_X" },
  { "LocationScopeType": "TagLocationScope", "Tag": "TagId_Y" }
]
```

S3 上存有 LocationId → TagId 的對應表：
```json
{
  "Locations": [
    { "LocationId": "1001", "TagId": "TagId_X" },
    { "LocationId": "1002", "TagId": "TagId_Y" }
  ]
}
```

---

## 情境 A：線上訂單，門市永遠通過 ✅

**Request：** `Channel = Web`，`Shipping.LocationId = 0`

**`CreateShoppingCartContext`：**
```csharp
var isLocationAlwaysMatch = OnlineSalesChannels.Contains(context.Calculate.Channel);
// OnlineSalesChannels = [Web, AppIOS, AppAndroid]
// → isLocationAlwaysMatch = true

var locationContext = new LocationContext(
    locationId: 0,
    locationTags: [],
    isAlwaysMatch: true);   // ← 引擎跳過門市比對
```

**結果：** 無論活動設定哪幾間門市，線上訂單的門市 Scope **永遠通過**。

---

## 情境 B：InStore 線下訂單，顧客在門市 1001（一般訂單計算） ⚠️

**Request：** `Channel = InStore`，`Shipping.LocationId = 1001`

**`CalculateService.CreateProcessContextAsync` 初始化：**
```csharp
shipping = new ShippingProcessEntity
{
    LocationId = 1001,
    LocationTags = []   // ← 初始為空
};
```

**`BottomProcessGroup.CalculateAsync` 執行 `UpdateLocationTagsAsync`：**
```csharp
// BaseProcessGroup 的預設實作 → 空實作，不填充 LocationTags
protected virtual async Task UpdateLocationTagsAsync(GroupContext context) { }
```

**`CreateShoppingCartContext`：**
```csharp
var isLocationAlwaysMatch = OnlineSalesChannels.Contains(InStore);
// → false，需要比對

var locationContext = new LocationContext(
    locationId: 1001,
    locationTags: [],    // ← 空！
    isAlwaysMatch: false);
```

**引擎比對：** Rule 要求 `Tag = "TagId_X"`，但 `LocationTags` 是空的 → **not match ❌**

> ⚠️ **已知缺口：** `BottomProcessGroup`（一般訂單計算）及 `PromotionBasketGroup`（回饋活動菜籃）都沒有 override `UpdateLocationTagsAsync`，線下訂單的門市維度實際上**無法正確比對**，門市限制不生效。

---

## 情境 C：CustomRuleBasketGroup（點數活動菜籃），門市正確實作 ✅

`CustomRuleBasketGroup` override 了 `UpdateLocationTagsAsync`：

```csharp
protected override async Task UpdateLocationTagsAsync(GroupContext context)
{
    var defaultLocationTags = typeof(AllUserScope).Name;  // "AllUserScope"
    var locationTags = await this._s3LocationRepository.GetLocationTagsAsync(context);
    locationTags.Add(defaultLocationTags);
    context.Shipping.LocationTags = locationTags.ToArray();
}
```

**`S3LocationRepository.GetLocationTagsAsync` 流程：**

1. 確認 `context.Shipping.LocationId != 0`（否則回傳空）
2. 只針對三種回饋活動類型（Point2 / RatePoint2 / Coupon）查詢活動
3. 查 DB 取 `PromotionEngineRuleRecord_S3Key`
4. `DownloadFromS3Async()` 下載 JSON
5. 遍歷 `JSON["Locations"]`，比對 `LocationId == 1001` → 找到 `TagId_X`
6. 回傳 `["TagId_X"]`

填充後：`LocationTags = ["TagId_X", "AllUserScope"]`

**引擎比對：** Rule 要求 `Tag = "TagId_X"`，`LocationTags` 包含 → **match ✅**

---

## 門市維度實作現況彙整

| Group | 通路 | 門市（線上） | 門市（線下） |
|-------|------|------------|------------|
| BottomProcessGroup（一般訂單） | ✅ 引擎比對 | ✅ isAlwaysMatch = true | ⚠️ LocationTags 空，不生效 |
| PromotionBasketGroup（回饋活動菜籃） | ✅ 不清空，引擎比對 | ✅ isAlwaysMatch = true | ⚠️ LocationTags 空，不生效 |
| CustomRuleBasketGroup（點數活動菜籃） | ✅ | ✅ | ✅ S3 取 TagId，正確比對 |
