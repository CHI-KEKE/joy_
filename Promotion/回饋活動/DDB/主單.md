## 📅 時間

| 欄位名稱 | 說明 |
|---------|------|
| `BookingTimeUTC` | **指定日期模式**：日期 0:00<br>**+ N 天模式**：orderTime + rewardDays 0:00 |
| `OrderTimeUTC` | **線上訂單**：`OrderCreated Event 觸發時間`<br>**線下訂單**：`CrmSalesOrderTradesOrderFinishDateTime` |
| `RewardTimeUTC` | 實際回饋時間 (指定日期) |
| `RewardDays` | 回饋延遲天數 (N 天) |
| `ValidDateTimeUTC` | 預計點數生效時間 (可能會被更新) |
| `ExpireDateTimeUTC` | 點數效期<br>⚠️ 優惠券到期日以優惠券設定為主 |
| `UpdateTimeUTC` | 資料表更新時間 |

## 回饋狀態 (`RewardStatus`)

| 狀態值 | 說明 |
|--------|------|
| `WaitToReward` | 等待回饋 |
| `Reward` | 已回饋 |
| `Unmatch` | 不符合條件 |
| `Cancel` | 已取消 |
| `Occupy` | 已佔用 |
| `MatchWithoutQuota` | 符合但無配額 |

## 其他狀態欄位

- `RewardType` - 回饋類型
- `UpdateReason` - 更新原因

## 回饋歷史記錄 (`RewardHistory`)

包含以下欄位的歷史異動記錄：

- `RewardStatus` - 回饋狀態
- `RewardTime` - 回饋時間
- `TotalLoyaltyPoint` - 總點數
- `GivingPoints` - 發放點數
- `RecyclePoints` - 回收點數
- `UpdateUser` - 更新使用者
- `UpdateReason` - 更新原因
- `UpdateTime` - 更新時間

## 🛒 訂單

### RewardSalesOrderSlaveStatus

訂單回饋狀態 (依規則制定)：

| 狀態值 | 說明 | 適用場景 |
|--------|------|----------|
| `Finish` | 訂單完成給點 | 線下訂單都是訂單完成 |
| `WaitingToShipping` | 訂單成立給點 | 線上訂單立即給點 (不會 + N 天) |

### TradesOrderGroupCode

- **線下訂單**：`CrmSalesOrder:{order.CrmSalesOrderId}`
- **線上訂單**：正常的訂單群組代碼
