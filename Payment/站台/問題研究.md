## 原因研究

這不是單純的「連不上資料庫」，而是因為：

你的 TransactionScope 交易物件在執行中被「異常中斷」或「已失效」，
但 Entity Framework 嘗試在這個失效的交易裡開啟連線或進行 enlist 時，被拒絕。

換句話說：

這筆交易的狀態（Transaction State）已經是「中止（Aborted）」或「無效（Invalid）」，
但 EF 還想往裡面註冊一個新連線（Enlist），
結果就被 System.Transactions 拒絕，拋出這個錯誤。

TransactionScope 在尚未完成時出現例外、逾時、或被 Dispose
導致交易狀態變成「Aborted」，
而你後面的 context.SaveChanges() 嘗試打開連線時，就發生這個錯誤。

### TransactionScope 逾時（預設 Timeout 是 1 分鐘）
預設 timeout 是 1 分鐘，太長時如果連線被卡、會導致 scope 被自動 Abort。
### query 是 null → 沒呼叫 Complete() 就離開 scope

### 任何例外（包含 DbUpdateConcurrencyException、SqlException）拋出後沒有被處理

Transaction 升級成 DTC；

### 解法 2：不用 TransactionScope，改用 EF Transaction

{

                 where ...
                 select p).ToArray();

    // update logic
    context.SaveChanges();
    tx.Commit();
}

System.Transactions.TransactionException: The operation is not valid for the state of the transaction.
在 .NET 的 System.Transactions 裡，這句訊息代表：

TransactionScope 內部的交易（Transaction）狀態不再是 Active，而是：

Aborted（已中止）

Disposed（已被釋放）

Timeout（已過期）

而這種情況下，錯誤通常出現在：

SqlInternalConnection.EnlistNonNull(Transaction tx)


也就是「EF 嘗試讓 SQL 連線加入交易」的階段。
👉 換句話說，連線還沒真正開起來、SQL 還沒送出，交易就壞了。


💡 它是「交易初始化」失敗，不是「交易執行中」失敗。

也就是：

沒有任何資料寫入；

沒有送出 SQL；

EF 只是要打開連線 → 發現這個 TransactionScope 狀態異常；

於是拋出這個錯誤。

int retryCount = 0;
const int maxRetry = 3;

while (true)
{
    try
    {
        UpdateInternal(updateParam); // 你原本的交易邏輯
        break;
    }
    catch (TransactionException ex)
    {
        // ✅ 只針對這句訊息重試是安全的
        if (ex.Message.Contains("not valid for the state of the transaction") && retryCount < maxRetry)
        {
            retryCount++;
            _logger.Warn(ex, $"Transaction state invalid，第 {retryCount} 次重試");
            Thread.Sleep(200 * retryCount);
            continue;
        }

        throw;
    }
}



🚫 相反地，以下錯誤就「不安全重試」
錯誤訊息	意思	為什麼不安全
Timeout expired (SqlException -2)	查詢執行太久	SQL 可能部分執行成功
The transaction has aborted.	交易在執行中被中止	有可能部分更新已寫入
MSDTC on server is unavailable	分散式交易協調器失效	交易可能不確定是否提交
Communication with the underlying transaction manager has failed	網路中斷	不確定交易提交狀態


原因是：

之前查詢 .ToArray() 時 EF 開過一次連線；

查詢完後 EF 自動關閉連線；

SaveChanges() 時重新打開連線 → 嘗試重新加入同一個 TransactionScope；

但這個 TransactionScope 已經 timeout 或被 Aborted；

所以 TransactionManager 拒絕 Enlist；

丟出「The operation is not valid for the state of the transaction」。

💡 也就是「交易還沒開始更新、但已經壞掉」→ 重試安全。
