# Stripe 信用卡付款 - 付款情境分析

## 情境一：單純 Pay

### 觸發條件

| ExtendInfo 欄位 | 值 |
|----------------|---|
| `payment_method` | 空（未提供） |
| `customer_id` | 空（未提供） |
| `is_reuse_payment_method` | `false`（預設） |

### API 呼叫序列

#### DirectCharge 流程

```
1. POST /v1/payment_methods
   Header: Stripe-Account: {sub_account}
   Body:
     type=card
     card[number]={卡號}
     card[exp_year]={到期年}
     card[exp_month]={到期月}
     card[cvc]={CVV}
     metadata[request_id]={request_id}
     metadata[tg_code]={tg_code}

2. POST /v1/payment_intents
   Header: Stripe-Account: {sub_account}
   Body:
     payment_method={上一步 payment_method.id}
     amount={金額 * 100，取整數}
     currency={幣別}
     confirmation_method=automatic
     confirm=true
     application_fee_amount={手續費 * 100}
     return_url={return_url}
     metadata[request_id]={request_id}
     metadata[tg_code]={tg_code}
```

#### DestinationCharge 流程

```
1. POST /v1/payment_methods
   （不帶 Stripe-Account Header，使用主帳號）
   Body: 同上

2. GET /v1/accounts/{sub_account}
   （取得 settings.payments.statement_descriptor，供銀行簡訊顯示）

3. POST /v1/payment_intents
   （不帶 Stripe-Account Header）
   Body:
     transfer_data[destination]={sub_account}
     on_behalf_of={sub_account}
     payment_method={上一步 payment_method.id}
     amount={金額 * 100}
     currency={幣別}
     confirmation_method=automatic
     confirm=true
     application_fee_amount={手續費 * 100}
     return_url={return_url}
     statement_descriptor={取自帳號設定，若有的話}
     metadata[request_id]={request_id}
     metadata[tg_code]={tg_code}
```

### 回應情境

| Stripe PaymentIntent status | ReturnCode | 說明 |
|----------------------------|-----------|------|
| `succeeded` | `0000` | 直接付款成功 |
| `requires_action` + next_action | `2003` | 需要 3D 驗證，回傳 redirect URL |
| Stripe API 拋出 ApiException | `3000` | 付款失敗（卡被拒、卡號錯誤等） |
| 其他未預期例外 | HTTP 500 | 系統錯誤，logger 記錄後 throw |

## 情境二：記住信用卡（首次綁卡）

### 觸發條件

| ExtendInfo 欄位 | 值 |
|----------------|---|
| `is_reuse_payment_method` | `true` |
| `payment_method` | 空（新卡，無 Token） |
| `customer_id` | 空 |
| `shop_id` | 必填（用於識別 Customer） |
| `member_id` | 必填（用於識別 Customer） |

### API 呼叫序列

```
1. GET /v1/customers/search?query=name:"{shop_id}_{member_id}"
   Header: Stripe-Version: 2020-08-27
   （DirectCharge 帶 Stripe-Account；DestinationCharge 不帶）
   → 查詢此會員是否已有 Stripe Customer 記錄

2. POST /v1/payment_methods
   （同情境一，帶明碼卡號）

3. POST /v1/payment_intents
   Body 額外加入:
     setup_future_usage=off_session      ← 固定帶，通知 Stripe 之後要 off-session 扣款
     customer={customer_id}              ← 若 ExtendInfo.CustomerId 有值才帶
     off_session=true                    ← 僅當 ExtendInfo.OffSession = true 時帶入

   付款完成後依據 PaymentIntent status 決定是否綁卡：
```

```
PaymentIntent.status
   ├─ succeeded 或 requires_action → 進行綁定
   │       │
   │       ├─ customers.data.Count = 1（Customer 已存在）
   │       │     4a. POST /v1/payment_methods/{payment_method_id}/attach
   │       │         Body: customer={existing_customer_id}
   │       │         （DirectCharge 帶 Stripe-Account）
   │       │
   │       └─ customers.data.Count = 0（Customer 不存在）
   │             4b. POST /v1/customers
   │                 Body:
   │                   name={shop_id}_{member_id}
   │                   payment_method={payment_method_id}
   │                 （DirectCharge 帶 Stripe-Account）
   │
   └─ 其他狀態（付款失敗） → 不綁定，直接回傳
```

### 回應情境

| PaymentIntent status | 綁卡動作 | ReturnCode | 說明 |
|---------------------|---------|-----------|------|
| `succeeded` | attach 或 create customer | `0000` | 付款成功並完成綁卡 |
| `requires_action` | attach 或 create customer | `2003` | 需要 3D 驗證，已完成綁卡準備 |
| 其他失敗狀態 | 不綁卡 | `0000`（帶 PaymentIntent） | 付款未成功，不記卡 |
| Stripe API 拋出 ApiException | 不綁卡 | `3000` | 付款失敗 |

## 情境三：舊卡復用

### 觸發條件

| ExtendInfo 欄位 | 值 |
|----------------|---|
| `payment_method` | 必填（已綁定的 PaymentMethod Token） |
| `customer_id` | 必填（已建立的 Customer ID） |
| `off_session` | `true` = 靜默扣款；`false` = 可能觸發 3D |

### API 呼叫序列

```
1. GET /v1/customers/search?query=name:"{shop_id}_{member_id}"
   Header: Stripe-Version: 2020-08-27
   → 驗證傳入的 customer_id 是否在 search 結果中（防止非法使用）

   驗證通過（customer_id 在 data 中）→ 繼續
   驗證失敗（customer_id 不在 data 中）→ throw NotSupportedException → HTTP 500

2. POST /v1/payment_intents
   Body:
     payment_method={request.ExtendInfo.PaymentMethod}
     amount={金額 * 100}
     currency={幣別}
     confirmation_method=automatic
     confirm=true
     application_fee_amount={手續費 * 100}
     return_url={return_url}
     customer={customer_id}              ← 若有值才帶
     off_session=true                    ← 僅當 ExtendInfo.OffSession = true 時帶入
     （DestinationCharge 額外帶 transfer_data[destination]、on_behalf_of、statement_descriptor）
```

### 回應情境

| PaymentIntent status | ReturnCode | 說明 |
|---------------------|-----------|------|
| `succeeded` | `0000` | 舊卡直接付款成功 |
| `requires_action` + next_action | `2003` | 需要 3D 驗證 |
| `customer_id` 驗證失敗 | HTTP 500 | 非法 Customer，throw NotSupportedException |
| Stripe API 拋出 ApiException | `3000` | 付款失敗（Token 失效、卡被停用等） |

---

## 三種情境 API 呼叫數量對照

| 情境 | DirectCharge API 數 | DestinationCharge API 數 |
|------|--------------------|-----------------------|
| 單純 Pay | 2 | 3（多一個 GET /v1/accounts） |
| 記住信用卡（無 Customer） | 4 | 5 |
| 記住信用卡（有 Customer） | 4 | 5 |
| 舊卡復用 | 2 | 3 |
