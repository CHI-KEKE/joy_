# Stripe 信用卡付款 - 架構總覽

## 系統架構層次

```
呼叫端 (Commerce / 外部系統)
        │
        ▼
MiddlewareController (Web API 入口)
  POST /api/v1/Pay/{payMethod}_{payChannel}/{tgCode}
  例如: POST /api/v1/Pay/CreditCardOnce_Stripe/{tgCode}
        │
        ▼
IPluginDispatcher
        │
        ▼
StripePlugin.Pay()
        │
        ├─────────────────────────────────────┐
        ▼                                     ▼
DirectChargePaymentFlowStrategy    DestinationChargePaymentFlowStrategy
（子帳號直接扣款）                    （主帳號扣款，轉帳給子帳號）
```

---

## 核心元件

| 元件 | 說明 |
|------|------|
| `MiddlewareController` | Web API 入口，接收付款請求並轉發給 Dispatcher |
| `IPluginDispatcher` | 根據 payChannel 選擇對應的 Plugin |
| `StripePlugin` | Stripe 付款主邏輯，決定走哪條流程 |
| `DirectChargePaymentFlowStrategy` | DirectCharge 付款策略 |
| `DestinationChargePaymentFlowStrategy` | DestinationCharge 付款策略（含取帳號資訊） |
| `RememberPaymentMethodProcess` | 記住信用卡流程（首次綁卡） |
| `ReusePaymentMethodPaymentIntentProcess` | 舊卡復用流程 |
| `IStripeHttpClient` | Refit HTTP Client，封裝所有對 Stripe API 的呼叫 |

---

## 兩種 Payment Flow 差異

| 比較項目 | DirectCharge | DestinationCharge |
|---------|-------------|-------------------|
| PaymentMethod 建立 | 在**子帳號**下建立（帶 `Stripe-Account` Header） | 在**主帳號**下建立（不帶 Header） |
| PaymentIntent 建立 | 在**子帳號**下建立 | 在**主帳號**下建立，資金轉給子帳號 |
| 額外 API 呼叫 | 無 | `GET /v1/accounts/{id}` 取得子帳號描述文字 |
| 手續費方式 | `application_fee_amount` | `application_fee_amount` + `transfer_data[destination]` |
| 子帳號帳款顯示 | 子帳號對帳單直接顯示 | 透過 `statement_descriptor` 設定銀行顯示文字 |

---

## 三種付款情境判斷邏輯

`StripePlugin.Pay()` 依據 Request 的 ExtendInfo 欄位決定走哪條路：

```csharp
// 情境三：舊卡復用
if (有 payment_method && 有 customer_id)
    → ReusePaymentMethodPaymentIntentProcess

// 情境二：記住信用卡（首次綁卡）
else if (is_reuse_payment_method == true)
    → RememberPaymentMethodProcess

// 情境一：單純 Pay
else
    → strategy.Pay()
```

## 所有 Stripe API 彙總

| API | Method | 說明 | 使用情境 |
|-----|--------|------|---------|
| `/v1/payment_methods` | POST | 建立 PaymentMethod（帶卡號） | 單純Pay、記住信用卡 |
| `/v1/payment_intents` | POST | 建立並確認 PaymentIntent | 所有情境 |
| `/v1/payment_intents/{id}` | GET | 查詢 PaymentIntent 狀態 | QueryPayment |
| `/v1/payment_intents/{id}/cancel` | POST | 取消 PaymentIntent | Cancel |
| `/v1/accounts/{id}` | GET | 取得子帳號資訊 | DestinationCharge 流程 |
| `/v1/customers/search` | GET | 查詢 Customer | 記住信用卡、舊卡復用 |
| `/v1/customers` | POST | 建立新 Customer | 記住信用卡（無 Customer 時） |
| `/v1/payment_methods/{id}/attach` | POST | 綁定 PaymentMethod 至 Customer | 記住信用卡（已有 Customer） |
| `/v1/charges/{id}` | GET | 取得 Charge 資訊 | 退款流程 |
| `/v1/refunds` | POST | 建立退款 | 退款流程 |
| `/v1/application_fees/{id}/refunds` | POST | 退還手續費 | 退款流程 |
| `/v1/transfers/{id}/reversals` | POST | 反轉 Transfer（退款） | DestinationCharge 退款 |
