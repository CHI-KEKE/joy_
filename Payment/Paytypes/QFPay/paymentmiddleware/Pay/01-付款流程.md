# QFPay — Pay 付款流程分析

## 核心架構：URL Redirect（簽名跳轉）

與 Cybersource 類似，QFPay 的 Pay **不直接呼叫 API**。  
PMW 負責組建帶有 SHA256 簽名的付款頁 URL，回傳 `Redirect` action 讓前台導向。

```
前台 → POST /Pay → PMW → 組建 QFPay 付款頁 URL（含簽名）
                         ↓
                   回傳 Redirect Action（WebPaymentUrl + AppPaymentUrl）
                         ↓
                   前台導向 QFPay 付款頁
                         ↓
                   用戶完成付款（QFPay 頁面處理）
                         ↓
         成功 → ReturnUrl  /  失敗 → FailedUrl（同時非同步通知 NotifyUrl）
```

---

## 環境切換：IsUsingLiveTest

QFPay 支援兩套環境，由 `ExtendInfo.IsUsingLiveTest` 決定使用哪個 BaseUrl：

| IsUsingLiveTest | 使用的 URL |
|----------------|-----------|
| `false`（預設） | `_N1CONFIG:Plugins:QFPay:HttpClient:BaseUrl` |
| `true` | `_N1CONFIG:Plugins:QFPay:HttpClient:LivingTestBaseUrl` |

> 此設計讓同一套 Production 環境可以對 QFPay 的 Live Test 環境進行測試。

## 簽名演算法（全功能通用）

**所有 QFPay 的請求（Pay / Query / Refund / RefundQuery / Cancel）皆使用相同簽名邏輯：**

```
Step 1：取出所有 payload 欄位
Step 2：過濾掉空值欄位
Step 3：依欄位名稱字母排序（A → Z）
Step 4：組成 key=value 格式，以 & 連接
        → payloadString = "amount=1000&currency=HKD&out_trade_no=TG123..."
Step 5：簽名 = SHA256(payloadString + apiKey) → 小寫 hex 字串
```

```csharp
// 偽代碼
var sortedFields = payload.OrderBy(x => x.Key).Where(x => !string.IsNullOrEmpty(x.Value));
var payloadString = string.Join("&", sortedFields.Select(x => $"{x.Key}={x.Value}"));
var signature = SHA256(payloadString + apiKey).ToLower();
```

> `apiKey` 來自 request header `X-Api-Key`，不包含在 payload 中，只用於簽名計算尾端。

### Pay 的特殊差異：URL Encode

Pay 有兩種 payload string：
| 用途 | URL Encode |
|------|-----------|
| 簽名計算 | ❌ 不做 URL Encode |
| 組 URL query string | ✅ 做 URL Encode |

```
最終付款 URL = {payUrl}{urlEncodedPayload}&sign={signature}
```

## Request Header（所有 REST API 通用）

```
Content-Type: application/x-www-form-urlencoded
X-QF-APPCODE: {apiCode}         ← 來自 header X-Api-Code
X-QF-SIGN: {sha256_signature}
X-QF-SIGNTYPE: SHA256
```

## Pay Payload 欄位對照

| QFPay 欄位 | 說明 | 來源 | 必填 |
|-----------|------|------|------|
| `appcode` | API 憑證 | header `X-Api-Code` | ✅ |
| `sign_type` | 簽名類型，固定 `"sha256"` | 硬碼 | ✅ |
| `paysource` | 付款來源，固定 `"91App_checkout"` | 硬碼 | ✅ |
| `txamt` | 付款金額（× 100 最小單位） | `request.Amount * 100` | ✅ |
| `txcurrcd` | 幣別（如 `HKD`） | `request.Currency` | ✅ |
| `out_trade_no` | 外部訂單號（TG Code） | `request.TradesOrderGroupCode` | ✅ |
| `txdtm` | 訂單時間（`yyyy-MM-dd hh:mm:ss`） | `ExtendInfo.OrderDateTime` | ✅ |
| `return_url` | 付款成功導向 URL | `ExtendInfo.ReturnUrl` | ✅ |
| `failed_url` | 付款失敗導向 URL | `ExtendInfo.FailedUrl` | ✅ |
| `notify_url` | 非同步通知 URL | `ExtendInfo.NotifyUrl` | ✅ |
| `goods_name` | 商品名稱（最大 64 字，不含特殊字元） | `ExtendInfo.GoodsName` | ✅ |
| `mchntid` | 代理商商戶識別碼（最大 16 字元） | `ExtendInfo.Mchntid` | ❌ |
| `txzone` | 時區（預設 `+0800`） | `ExtendInfo.TimeZone` | ❌ |
| `lang` | UI 語言（`zh-hk`/`zh-cn`/`en`） | `ExtendInfo.Lang` | ❌ |
| `expired_time` | QR Code 到期時間（分鐘，5~120） | `ExtendInfo.ExpiredTime` | ❌ |
| `checkout_expired_time` | 客戶端結帳到期時間（毫秒） | `ExtendInfo.CheckoutExpiredTime` | ❌ |

### ⚠️ Dead Fields（`QFPayCreatePaymentRequestEntity` 有但從未被賦值）

| 欄位 | 說明 |
|------|------|
| `limit_pay` | 禁止信用卡旗標（僅 WeChat Pay 支援），Plugin 從未設定 |
| `udid` | 裝置 ID，Plugin 從未設定（Cancel/Refund 有，Pay 沒有） |

## Pay 回傳結構

```json
{
  "ReturnCode": "2003",
  "ReturnMessage": "付款頁面成功產出",
  "TransactionId": "",
  "Action": {
    "Action": "Redirect",
    "WebPaymentUrl": "https://o2-hk.qfapi.com/...?appcode=xxx&...&sign=abc123",
    "AppPaymentUrl": "https://o2-hk.qfapi.com/...?appcode=xxx&...&sign=abc123",
    "RedirectHttpMethod": "POST"
  },
  "ExtendInfo": {
    "paymantPageUrl": "https://o2-hk.qfapi.com/..."
  }
}
```

> `WebPaymentUrl` 與 `AppPaymentUrl` 帶有相同 URL（QFPay 付款頁支援 Web 及 App 共用同一入口）。
