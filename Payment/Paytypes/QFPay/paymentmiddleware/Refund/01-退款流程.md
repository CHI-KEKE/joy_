# QFPay — Refund 退款流程分析

## 流程概覽

```
PMW → POST /trade/v1/refund
       ↓
   QFPay 回傳退款結果（ResponseCode）
       ↓
   對應 ReturnCode（詳見 02-ReturnCode對照.md）
```

---

## API 呼叫

```
POST /trade/v1/refund
Content-Type: application/x-www-form-urlencoded
X-QF-APPCODE: {apiCode}
X-QF-SIGN: {sha256_signature}
X-QF-SIGNTYPE: SHA256
```

Request Body（form-urlencoded，排序後簽名）：

| 欄位（QFPay） | 說明 | 來源 | 必填 |
|-------------|------|------|------|
| `syssn` | 原始付款 QFPay 交易編號 | `request.TransactionId` | ✅ |
| `out_trade_no` | 退款訂單號（TS Code） | `ExtendInfo.OrderCode` | ✅ |
| `txamt` | 退款金額（× 100 最小單位） | `request.Amount * 100` | ✅ |
| `txdtm` | 退款請求時間（`yyyy-MM-dd hh:mm:ss`） | `ExtendInfo.RequestDatetime` | ✅ |
| `mchid` | 商戶編號 | ⚠️ **固定為 null** | ❌ |
| `txzone` | 時區 | `ExtendInfo.TimeZone` | ❌ |
| `udid` | 裝置 UUID | `ExtendInfo.DeviceUuid` | ❌ |

---

## ResponseCode 對應 ReturnCode

詳細對應邏輯請見 [02-ReturnCode對照.md](./02-ReturnCode對照.md)

**快速摘要：**

| QFPay ResponseCode | ReturnCode |
|-------------------|-----------|
| `0000` | Success |
| `1124` / `1269` / `1265` / `1145` / `1143` | RefundPending |
| `1265`（描述含「超出可退款時限」） | **Failed**（覆蓋 RefundPending） |
| `1250` / `1297` | RefundPeriodExceeded |
| `1181` | RefundFailed |
| 其他 | Failed |

---

## 退款回傳結構

| PMW 欄位 | 來源 |
|---------|------|
| `TransactionId` | `apiResponse.RefundTransactionId`（新建的退款 syssn） |
| `ReturnCode` | 依上方 ResponseCode 對應 |
| `ReturnMessage` | `{apiResponse.ResponseMessage} - {apiResponse.ResponseDescription}` |
| `ExtendInfo` | 完整 `QFPayRefundResponseEntity` |

## 特殊設計說明

### 1265 雙重含義

`1265` 正常代表「結帳期間（23:30~00:30）禁止退款」，應視為 RefundPending 等待重試。  
但 FPS（轉數快）超過退款期限也回 `1265`，描述為「該筆訂單被禁止退款，已經超出可退款時限」。  
此情況不應重試，直接視為 **Failed**：

```csharp
if (apiResponse.ResponseDescription.Contains("該筆訂單被禁止退款，已經超出可退款時限"))
{
    returnCode = ReturnCodes.Failed;
}
```

> FPS 僅允許退款 29 天以內的交易。

### 金額轉換

與 Pay 相同，退款金額需乘以 100：
```csharp
Amount = (int)(request.Amount * 100)
```

### 無例外處理

Refund 不包含 `try-catch`，若 API 呼叫失敗（如網路錯誤），例外會直接往上拋。
