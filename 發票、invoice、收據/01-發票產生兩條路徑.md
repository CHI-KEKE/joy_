# 發票產生兩條路徑

---

## 路徑一：一般付款流程（主流程）

**觸發時機**：ThirdPartyProcess / CreditCard / ApplePay / GooglePay 等付款成功後  
**核心 Processor**：`CreateGlobalInvoiceProcessor`  
**位置**：`WebStore/Frontend/BLV2/PayProcesses/Processors/CreateGlobalInvoiceProcessor.cs`

```
付款成功
    ↓
CreateGlobalInvoiceProcessor.Execute()
    │
    ├─ Guard 1：AppSetting["Global.Invoice.Enable"] == false → return（不產生）
    │
    ├─ Guard 2：SettingHelper.DefaultCountry == "TW"
    │           AND 是 MobileWallet 付款（GooglePay/ApplePay）
    │           → return（TW 行動錢包不由此路徑產生）
    │
    └─ 推送 NMQ Task：GenerateMyInvoiceTask
         （延遲秒數由 AppSetting["Global.Invoice.Generate.Delay.Seconds"] 控制）
```

**Processor 在付款流程中的位置（ThirdPartyProcess）**：
```
PayProcessProcessorModule.cs L388 附近：
...
→ PaymentMiddlewareRequestProcessor
→ [付款 API 呼叫]
→ CreateTradesOrderProcessor      ← IsIssuedInvoice 寫入 DB
→ ...
→ CreateGlobalInvoiceProcessor    ← 推送 NMQ
→ ...
```

---

## 路徑二：GlobalPay Callback

**觸發時機**：GlobalPay（大馬原生金流）付款回調成功時  
**核心 Processor**：`GenerateInvoiceProcessor`  
**位置**：`WebStore/Frontend/BLV2/PayProcesses/Processors/GlobalPays/GenerateInvoiceProcessor.cs`

```
GlobalPay 付款回調
    ↓
GenerateInvoiceProcessor.Execute()
    │
    ├─ 狀態為 TradesOrderThirdPartyPaymentStatusEnum.Success 才執行
    │
    └─ 推送 NMQ Task：GenerateMyInvoiceTask
```

**Processor 在付款流程中的位置（GlobalPayFinishProcess）**：
```
PayProcessProcessorModule.cs L1019 附近：
...
→ GenerateInvoiceProcessor    ← 推送 NMQ（原始設計為大馬發票用途）
→ ...
```

---

## 兩條路徑比較

| 項目 | 路徑一（一般付款） | 路徑二（GlobalPay Callback） |
|---|---|---|
| 觸發點 | 付款成功後同步流程 | GlobalPay 非同步 Callback |
| Guard 條件 | `Global.Invoice.Enable` + TW/MobileWallet 排除 | 僅限 `Success` 狀態 |
| 推送的 NMQ Task | `GenerateMyInvoiceTask` | `GenerateMyInvoiceTask` |
| 設計目的 | 通用（TW + MY） | 大馬在地化發票 |

---

## NMQ Consumer 端（GenerateMyInvoiceTask handler）

- handler 位於 NMQ 消費端專案（非本 repo）
- 執行 `TradesOrderService.ProcessMyInvoice()`
- 該方法**只有在訂單擴充資訊含 `DataSetTypeEnum.MalaysiaInvoice` 時才觸發**
- → SG 訂單若無此 DataSetType，handler 會提前 return，不實際產生發票
