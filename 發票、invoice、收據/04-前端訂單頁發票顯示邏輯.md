# 前端訂單頁發票顯示邏輯

---

## 核心原則

> **訂單明細/列表的發票顯示，完全依賴 DB 欄位 `TradesOrderSlave_IsIssuedInvoice`，無任何即時計算。**

欄位在建立訂單時（`CreateTradesOrderProcessor` L480）寫入，之後不再異動。

---

## 訂單明細頁（ArrangeInvoiceInfo）

**位置**：`MemberTradesOrderService.cs` L2796  
**呼叫鏈**：`GetMemberTradesOrderDetail()` → `ArrangeInvoiceInfo()`

### ArrangeShowInvoiceStatus 邏輯

**位置**：`MemberTradesOrderService.cs` L2773

```csharp
// 發票顯示狀態直接讀 DB
ts.ShowInvoiceStatus = ts.IsIssuedInvoice;

// 是否顯示三聯式電子發票（有 PDF 可下載）
ts.IsShowThreeCopiesElectronicInvoice =
    發票未作廢(IsVoid == false)
    AND InvoiceSendInfoId 不為空（代表有發票 PDF 產生）
```

### 顯示欄位整理

| 前端欄位 | 資料來源 | 說明 |
|---|---|---|
| `ShowInvoiceStatus` | `TradesOrderSlave_IsIssuedInvoice`（DB） | 是否顯示發票區塊 |
| `IsShowThreeCopiesElectronicInvoice` | `IsVoid` + `InvoiceSendInfoId` | 是否有 PDF 可下載（三聯式） |
| `InvoiceNumber` | DB 欄位 | 發票號碼 |
| `InvoiceDate` | DB 欄位 | 開立日期 |

---

## 訂單列表頁

訂單列表同樣直接讀 DB，不另行計算。  
若 `IsIssuedInvoice = false`，該筆訂單不顯示任何發票資訊。

---

## 發票資訊完整不顯示的條件

以下任一條件成立，訂單頁就不會出現發票相關資訊：

1. `TradesOrderSlave_IsIssuedInvoice = false`（建立訂單時決定）
2. 付款方式在黑名單（PoyaPay、Wallet、RazerPay [GlobalPay 封裝版]）
3. `InvoiceIssueTypeDef` 為 `NotIssue` / `NoInvoiceRequired`
4. `InvoiceIssueTypeDef` 為 `OverseaNotIssue` 且 `countryCode != "TW"`

---

## 三聯式發票（TW 專用）的特殊欄位

| 欄位 | 說明 |
|---|---|
| `IsShowThreeCopiesElectronicInvoice` | 有 PDF 且未作廢才顯示下載按鈕 |
| `InvoiceSendInfoId` | 發票 PDF 寄送資訊 ID，有值才代表 PDF 已產生 |
| `IsVoid` | 是否已作廢（作廢後不顯示下載） |
