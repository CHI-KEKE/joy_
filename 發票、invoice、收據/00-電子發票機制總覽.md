# 電子發票機制總覽

> 專案：`nineyi.webstore.mobilewebmall`  
> 分析時間：2026-04

---

## 一、整體架構

```
結帳流程（建立訂單）
│
├─ ArrangeDataProcessor        → 決定是否蒐集發票資訊（前端表單顯示）
├─ CreateTradesOrderProcessor  → 寫入 IsIssuedInvoice = true/false 到 DB
│
└─ 付款成功後
     ├─ [一般付款路徑] CreateGlobalInvoiceProcessor → 推送 GenerateMyInvoiceTask
     └─ [GlobalPay Callback] GenerateInvoiceProcessor → 推送 GenerateMyInvoiceTask
```

---

## 二、關鍵決策點

| 決策 | 位置 | 說明 |
|---|---|---|
| 是否顯示發票填寫區（結帳頁） | `InvoiceService.IsShowInvoiceInformation()` | 依 countryCode + InvoiceIssueTypeDef |
| 是否蒐集三聯式資訊 | `InvoiceService.IsCollectThreeCopiesInvoice()` | 只有 TW 才蒐集 |
| 是否代開發票（寫 DB） | `InvoiceService.IsIssueInvoice()` | 核心判斷，見文件 02 |
| 前端訂單頁是否顯示發票 | `MemberTradesOrderService.ArrangeShowInvoiceStatus()` | 直接讀 DB 欄位 |
| 是否推送 NMQ 產生任務 | `CreateGlobalInvoiceProcessor` | 依 AppSetting 開關 |

---

## 三、重要 AppSetting

| Key | 說明 |
|---|---|
| `Global.Invoice.Enable` | 全站是否啟用發票產生機制（`true`/`false`） |
| `Global.Invoice.Generate.Delay.Seconds` | 推送 `GenerateMyInvoiceTask` 的延遲秒數 |
