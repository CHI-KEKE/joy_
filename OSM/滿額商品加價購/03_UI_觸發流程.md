# UI 觸發流程

## 一、新增促購活動頁面的分類架構

新增促購活動頁面（`selectActivityForm`）採用**分類群組**的方式呈現各種促購類型：

```
新增促購活動頁面
  ├── 【購物車活動】分類        ← CartPromotionViewModel（全新）
  │     └── 「滿額加價購」按鈕  ← EnableCartReachPriceExtraPurchase = true 才顯示
  │
  ├── 【單品特價】分類
  │     └── ...
  │
  ├── 【現折活動】分類
  │     └── ...
  │
  ├── 【湊件優惠】分類
  │     └── ...
  │
  ├── 【指定付款優惠】分類
  │     └── ...
  │
  ├── 【登記活動】分類
  │     └── ...
  │
  └── 【商品加價購】分類        ← 獨立分類，與購物車活動完全分開
        └── 「商品加價購」按鈕  ← EnableAddOns = true 才顯示
```

---

## 二、滿額加價購的觸發步驟

### Step 1：使用者點擊「滿額加價購」按鈕

```typescript
// TypeDef 值傳入
TypeDef: this.PromotionTypeEnum[this.PromotionTypeEnum.CartReachPriceExtraPurchase]
// → "CartReachPriceExtraPurchase"
```

### Step 2：跳轉至促購新增/編輯表單

前端傳入 TypeDef = `CartReachPriceExtraPurchase`，表單根據此 TypeDef 渲染對應欄位。

### Step 3：表單欄位（differFieldSetting.config.ts 設定）

| 欄位 | 顯示 | 說明 |
|------|------|------|
| `description` | ✅ | 活動說明 |
| `image` | ✅ | 活動圖片 |
| `displayOrder` | ✅ | 排序設定 |
| `memberCollection` | ✅ | 會員集合 |
| `displaySetting` | ✅ | 顯示設定 |
| `labelForMemberCollection` | ✅ | 會員集合標籤 |
| `channelType` | ✅ | 通路類型 |
| `cartExtraPurchaseRuleCondition` | ✅ | **購物車滿額門檻條件（核心）** |
| `isCartExtraPurchaseThresholdEligible` | ✅ | 是否列入購物車折扣門檻計算 |
| `supportMultiLevel` | ✅ | **支援多階層（可設定多個金額門檻）** |
| `ruleCondition` | ❌ | 傳統門檻條件（隱藏） |
| `memberTier` | ❌ | 會員等級（不支援） |
| `promotionTargetRegionTypeDef` | ❌ | 地區限定（不支援） |
| `eCoupon` | ❌ | 電子票券（不支援） |
| `promotionCycleSetting` | ❌ | 週期設定（不支援） |
| `giftAccumulateSetting` | ❌ | 贈品累計設定（不支援） |

### Step 4：使用者填入條件並儲存

滿額門檻元件（`cartReachPriceExtraPurchase.tsx`）顯示：

```
消費滿 NT$ [金額輸入]，即可加購商品
```

可透過「增加階層」按鈕新增多個金額門檻（最多 N 個）。

### Step 5：後端建立活動規則

```
POST /Api/PromotionEngine/Create
  Body: { TypeDef: "CartReachPriceExtraPurchase", ExtraPurchaseRuleList: [{ReachPrice: 500, ...}] }
  ↓
CartReachPriceExtraPurchaseRuleService.GetPromotionEngineRule()
  ↓
產生 CartReachPriceExtraPurchase Rule JSON → 傳至 N1PromotionService 儲存
```
