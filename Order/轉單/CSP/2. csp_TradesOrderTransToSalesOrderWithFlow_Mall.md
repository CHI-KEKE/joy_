## Step 5️⃣：子單 SalesOrderSlave（全場最複雜）

寫入

SalesOrderSlave

這一段做了三件「非常核心」的事


#### 🔹 1. 決定子單狀態（StatusDef）

這一段是整支 SP 的靈魂：

信用卡 / ApplePay / GooglePay
 ├─ AuthSuccess → WaitingToCreditCheck
 ├─ AuthExpired → Cancel
 └─ Others → Fail


ATM / CustomOffline → WaitingToPay
第三方支付 → WaitingThirdPartyPaymentCheck
其他 → WaitingToShipping

⚠️ HK 市場對 ApplePay / GooglePay 有特殊放行


#### 🔹 2. 是否 Close

Cancel / Fail → IsClosed = 1

正常流程 → IsClosed = 0


#### 🔹 3. 預計出貨日 ExpectShippingDate

超級長的 CASE：

冷藏 / 冷凍 + Family

預購（指定日 / 指定區間 / 指定工作天）

定期購

客製商品

fallback：cfn_GetDateExcludeHolidays(+2)

👉 這一段 直接影響 SLA、出貨、客服
