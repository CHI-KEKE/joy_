# 01 — 識別機制：如何區分加購品的加入來源

## 問題核心

同一個商品頁（SalePage）可能以兩種方式進入購物車：

| 加入方式 | 應有交期 | 識別方法 |
|---------|---------|---------|
| 消費者透過「商品頁」直接加入 | 定期購（RegularOrder） | `OptionalTypeDef = ''` |
| 消費者透過「滿額加購 UI」加入 | 一般（Normal） | `OptionalTypeDef = 'CartReachPriceExtraPurchase'` |

---

## 識別欄位：`OptionalTypeDef`

### DB 欄位

```
資料表：ShoppingCart
欄位：ShoppingCart_OptionalTypeDef  (nvarchar)
      ShoppingCart_OptionalTypeId   (bigint)
```

### Cart API Enum 定義

```csharp
// Nine1.Cart.BL.BE/SalePage/Enums/SalePageOptionalTypeEnum.cs
public enum SalePageOptionalTypeEnum
{
    PointsPay,                   // 點加金
    RegularOrder,                // 定期購
    CartReachPriceExtraPurchase, // 購物車滿額加價購活動的加購商品 ← 關鍵值
    CartReachPieceExtraPurchase, // 購物車滿件加價購活動的加購商品
    SalepageBundle               // 組合商品
}
```

### `IsCartExtraPurchase` 屬性

Cart API 的 `SalePageEntityForCart` 與 `SalepageEntity` 都實作了此屬性：

```csharp
// Nine1.Cart.BL.BE/Processor/Cart/SalePageEntityForCart.cs
// Nine1.Cart.BL.BE/Processor/Cart/SalepageEntity.cs

/// <summary>是否滿額/滿件加價購活動的加購品</summary>
public bool IsCartExtraPurchase
{
    get => ArrangeIsCartExtraPurchase();
}

private bool ArrangeIsCartExtraPurchase()
{
    return OptionalTypeDef is nameof(SalePageOptionalTypeEnum.CartReachPriceExtraPurchase)
                           or nameof(SalePageOptionalTypeEnum.CartReachPieceExtraPurchase)
           && OptionalTypeId > 0;
}
```

**判斷條件**：
- `OptionalTypeDef == "CartReachPriceExtraPurchase"` 或 `"CartReachPieceExtraPurchase"`
- **AND** `OptionalTypeId > 0`（`OptionalTypeId` = `sku.specialPriceId`，加購條件特價 ID）

---

## 與「商品加價購」的差異

系統中有兩種加價購，識別方式不同：

| 類型 | 業務名稱 | 識別屬性 | 依據 |
|------|---------|---------|------|
| 商品加價購子品 | AddOnsSalepageExtraPurchase | `IsSubAddOns() == true` | `CartExtendInfoItemTypeDef == "Sub"` + RuleTypeDef |
| 購物車滿額加價購 | CartReachPriceExtraPurchase | `IsCartExtraPurchase == true` | `OptionalTypeDef == "CartReachPriceExtraPurchase"` |

`IsSubAddOns()` 是透過購物車的 ExtendInfo 結構判斷（商品組合關聯），`IsCartExtraPurchase` 是透過 `OptionalTypeDef` 字串判斷（加入來源）。

---

## 為何不用 `ShoppingCart_IsExtra`

DB 中存在 `ShoppingCart_IsExtra` 欄位（`bool`，註解「是否為加購」），但：

- 所有 INSERT 路徑皆寫死 `ShoppingCart_IsExtra = false`
- BL 層從未讀取此欄位
- **此欄位為廢棄預留欄位，不可信賴**
