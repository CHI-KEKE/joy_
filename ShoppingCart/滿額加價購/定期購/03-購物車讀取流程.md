# 03 — 購物車讀取流程

> 消費者進入購物車頁後，從 API 呼叫到前端渲染「交期標籤」的完整路徑。

---

## 流程圖（修改後正確版本）

```
[MWeb] 購物車頁初始化
  → 呼叫 Shopping API CartCreateAsync()
─────────────────────────────────────────────────────────────
[Shopping API] CartService.CartCreateAsync()
  執行 ProcessorTypeEnum.CartCreate
  → Shopping CartCreateProcessor.ExecuteAsync()
        ↓ POST api/carts/create
─────────────────────────────────────────────────────────────
[Cart API] GetCartProcessorLayers 依序執行

  ┌─────────────────────────────────────────────────────┐
  │ [1] CartCreateProcessor                             │
  │   CartRepository.GetCartAsync() 從 DB 讀取          │
  │   → SalePageEntityForCart                           │
  │       .OptionalTypeDef = 'CartReachPriceExtraPurchase'│
  │       .OptionalTypeId  = specialPriceId (> 0)       │
  │                                                     │
  │   IsCartExtraPurchase:                              │
  │     OptionalTypeDef == 'CartReachPriceExtraPurchase' │
  │     && OptionalTypeId > 0  →  true ✅               │
  │                                                     │
  │   GetShippingTypeDef()  ← 【本次改動點】             │
  │     原始值      = "RegularOrder"                    │
  │     IsSubAddOns()       = false                     │
  │     IsCartExtraPurchase = true → 降級               │
  │     回傳值      = "Normal" ✅                        │
  │                                                     │
  │   SalepageEntity.SaleProductShippingTypeDef = "Normal"│
  └─────────────────────────────────────────────────────┘
        ↓
  ┌─────────────────────────────────────────────────────┐
  │ [2] GetPayTypeIsAvaliableProcessor                  │
  │   判斷可用付款方式，不影響此商品                    │
  └─────────────────────────────────────────────────────┘
        ↓
  ┌─────────────────────────────────────────────────────┐
  │ [3] GetShoppingCartRegularSalePageProcessor         │
  │   hasRegularSalepage =                              │
  │     SalepageList.Any(x =>                           │
  │       x.SaleProductShippingTypeDef == "RegularOrder")│
  │   → 此商品已是 "Normal"，條件不成立                 │
  │   → ⏭️ 直接 return，不呼叫定期購外部 API            │
  │   → RegularOrderOptions        = null               │
  │   → SelectedRegularOrderSetting = null              │
  └─────────────────────────────────────────────────────┘
        ↓
  ┌─────────────────────────────────────────────────────┐
  │ [4] ArrangeUnallowedRegularOrderSalePageListProcessor│
  │   僅 IsApp == true 且商店開啟開關時作用              │
  │   Web 用戶完全不受影響                              │
  └─────────────────────────────────────────────────────┘
        ↓
  ┌─────────────────────────────────────────────────────┐
  │ [5] GetPurchaseExtraAndSetHiddenSalePageTokenProcessor│
  │   查 AmountThreshold PurchaseExtra 活動             │
  │   商品 Id 在活動清單內 && IsPrimary                  │
  │   → IsPurchaseExtra             = true              │
  │   → PurchaseExtraAmountThreshold = 活動門檻          │
  │   EncodeHiddenSalePageId(salePageId,                │
  │       isCartExtraPurchase: true)                    │
  │   → 使用 AddOnsSalePageToken 加密（專用 Token）      │
  └─────────────────────────────────────────────────────┘
        ↓
  後續：價格計算、物流計算、分期...
        ↓
  Cart API 回傳 CartEntity：
    SaleProductShippingTypeDef  = "Normal"
    RegularOrderOptions         = null
    SelectedRegularOrderSetting = null
    IsPurchaseExtra             = true
─────────────────────────────────────────────────────────────
[Shopping API] CartCreateProcessor
  context.Data = result.Data   ← 直接 pass-through

  後續 Shopping Processor：
    ArrangeHighlightTagProcessor.GetHighlightTagList()
      ShippingTypeDef → SaleProductShippingTypeEnum.Normal
      tagValue = "Normal"

  Shopping API 回傳 CartEntity 給 MWeb
─────────────────────────────────────────────────────────────
[MWeb] useProductData.tsx
  attributes.saleProductShippingTypeDef
      = salePage.SaleProductShippingTypeDef
      = "Normal"

[MWeb] productCard.tsx → getMaskMessage()
  getSaleProductShippingTypeName("Normal")
      → SALE_PRODUCT_SHIPPING_TYPE_NAMES["Normal"]
      → ""（空字串）

前端：不顯示任何交期標籤 ✅
      定期購選項 UI 不出現  ✅
      消費者可正常一次性結帳 ✅
```

## GetCartProcessorLayers 執行順序（完整）

```csharp
// ProcessorDefinitionCenter.cs
public static readonly List<ICollection<Type>> GetCartProcessorLayers = new()
{
    new List<Type> { typeof(CartCreateProcessor) },                                // [1]
    new List<Type> { typeof(GetPayTypeIsAvaliableProcessor) },                     // [2]
    new List<Type> { typeof(GetShoppingCartRegularSalePageProcessor) },            // [3]
    new List<Type> { typeof(ArrangeUnallowedRegularOrderSalePageListProcessor) },  // [4]
    new List<Type> { typeof(GetPurchaseExtraAndSetHiddenSalePageTokenProcessor) }, // [5]
    new List<Type> { typeof(GetShoppingCartSalePageGiftProcessor) },               // [6]
    new List<Type> { typeof(CalculateShoppingCartSalePageGiftQtyLimitProcessor) }, // [7]
    new List<Type> { typeof(GetShoppingCartInstallmentProcessor) },                // [8]
    new List<Type> { typeof(GetBookingPickupDateProcessor) },                      // [9]
    new List<Type> { typeof(CalculatePriceProcessor<CartEntity>) },                // [10]
    new List<Type> { typeof(ExecuteLogisticCalculateProcessor<CartEntity>) },      // [11]
    new List<Type> { typeof(CacheCartDataProcessor) }                              // [12]
};
```

> **重點**：`GetShippingTypeDef()` 在 [1] 執行，`SaleProductShippingTypeDef` 設為 `"Normal"` 後，[3] 自然不會進入定期購分支。順序設計正確，只缺 [1] 內的判斷邏輯。
