# 需求說明 — Story #596789

## 需求標題

[滿額加價購][加價購商品] 檢核加購商品不支援一般商品的購買資格限制設定

## 背景

系統有一個「購買資格設定」功能（OSM > 限購設定 > 購買資格設定），可讓品牌指定特定商品只允許特定會員購買（例如：僅限金卡會員）。

當商品 A 同時被：
1. 設定了「購買資格限制」（如：僅金卡會員可購買）
2. 加入某檔「滿額加價購」活動作為加購品

一般會員（未具金卡資格）在達成滿額門檻後，**理論上應可加購商品 A**，但現行邏輯會因「購買資格不符」而阻擋。

## 需求內容

當商品以「滿額加購品」身份進入購物車時，強制跳過「購買資格」的攔截機制，確保結帳流程順暢。

## Acceptance Criteria

### AC 1：加購清單標籤顯示

| 項目 | 說明 |
|------|------|
| 位置 | 商品頁、購物車頁的加購清單區域 |
| 場景 | 商品 A 設有購買資格限制，且出現在滿額加價購清單中 |
| 預期 | 在加購區域中，**不應顯示任何「購買資格限制」相關標籤** |

### AC 2：加購權限穿透

| 項目 | 說明 |
|------|------|
| 假設條件 | 一般會員（不具備購買商品 A 的一般資格）已達成滿額門檻 |
| 測試行動 | 於購物車或加購選單中點擊加購商品 A |
| 預期 | **系統允許加入購物車**，不彈出「資格不符」的阻擋訊息 |

### AC 3：一次性結帳一致性

| 項目 | 說明 |
|------|------|
| 場景 | 購物車同時含主商品與加購商品 A |
| 預期 | **結帳流程正常進行**，不因資格檢查阻斷下單 |

## 檢核標準

> 檢核活動主商品範圍有圈到一般商品 A，則加價購商品 A 在商品頁、購物車頁，
> 不應被視為購買資格限制設定商品（沒有出現任何相關標籤），應以加價購商品購買，且是一次性結帳。
