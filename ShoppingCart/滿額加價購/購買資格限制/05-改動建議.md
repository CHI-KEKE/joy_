## 修改後的完整方法（供參考）

```csharp
private void RemoveItems(CartContext cart)
{
    var unmatchItems = new List<SalepageEntity>();

    foreach (var group in cart.Data.SalepageGroupList)
    {
        var items = group.SalepageList
            .Where(s => s.PromotionFlags.Contains(
                            nameof(PromotionFlagDefEnum.RestrictedPurchasesNotMeet)) &&
                        s.IsCartExtraPurchase == false)  //// 滿額加購品跳過
            .ToList();

        //// 主商品若未達門檻，子商品需一起移進 UnmatchPurchasesQualificationSalePageList
        var majorItems = items.Where(x => x.IsPrimary && x.CartExtendInfos.Any()).ToList();

        foreach (var majorItem in majorItems)
        {
            var subCartIdList = majorItem.CartExtendInfos.SelectMany(x => x.RelatedItemCartIds).ToList();
            var cartIdListToAdd = subCartIdList.Except(items.Select(x => x.CartId));
            var subSalepageToAdd = cart.Data.SalepageGroupList
               .SelectMany(salepageGroup => salepageGroup.SalepageList)
               .Where(x => cartIdListToAdd.Contains(x.CartId));

            items.AddRange(subSalepageToAdd);
        }

        foreach (var item in items)
        {
            group.SalepageList.Remove(item);
        }

        unmatchItems.AddRange(items);
    }

    cart.Data.UnmatchPurchasesQualificationSalePageList = unmatchItems;
}
```

## 改動後各 AC 的解決方式

| AC | 解決方式 |
|----|---------|
| AC 1：加購清單不顯示資格標籤 | `ExtraPurchaseProductList.tsx` 已有 `isRestricted={false}`，Modal 已正確。購物車頁的加購品，改動後不再進入 `UnmatchPurchasesQualificationSalePageList`，前端不再標記 `isUnmatchPurchaseQualification: true` |
| AC 2：允許加購商品加入購物車 | 加購品不再被移出主清單，可正常操作 |
| AC 3：一次性結帳不阻擋 | `MergeRequestAndCartContextProcessor` 的 `UnmatchPurchaseQualificationSalePageOnly` 判斷不再觸發（加購品保留在主清單，購物車不為空） |
