# 購買資格限制（RestrictedPurchasesByMember）機制說明

## 功能定位

OSM 後台 > 限購設定 > 購買資格設定，允許品牌將特定商品設為「僅限特定會員購買」（例如：僅限金卡會員、僅限 VIP 等）。

## 資料流：Flag 如何被打上

```
Promotion Engine API
  └─ 計算購物車中每個商品的促銷條件
  └─ 若當前會員不符合商品 A 的購買資格
       → 在回傳的 sku.Flags 中加入 "RestrictedPurchasesNotMeet"

Cart Domain CalculatePromotionProcessor.cs（L1064）
  └─ salePage.PromotionFlags = promotionSku.Flags;
       → 把 Promotion Engine 回傳的 flags 存入 SalepageEntity
```

**關鍵**：`RestrictedPurchasesNotMeet` flag 由 Promotion Engine 決定，Cart Domain 只是接收並存入。

## Flag 在 Cart Domain 中的後續處理

```
LogisticCalculateProcessorLayers 執行順序：
  ...
  CalculatePromotionProcessor          ← Flag 寫入 SalepageEntity.PromotionFlags
  ArrangeUnmatchPurchasesQualificationProcessor  ← 根據 Flag 移動商品（問題點）
  ...
```

`ArrangeUnmatchPurchasesQualificationProcessor.RemoveItems()` 的現行邏輯：
```csharp
var items = group.SalepageList
    .Where(s => s.PromotionFlags.Contains(
        nameof(PromotionFlagDefEnum.RestrictedPurchasesNotMeet)))
    .ToList();
// ← 全部帶有此 Flag 的商品都被移出主清單，無論是否為加購品
```

## 移出後的影響

```
CartEntity.SalepageGroupList         ← 購物車主清單（商品已被移除）
CartEntity.UnmatchPurchasesQualificationSalePageList  ← 被移入此清單
```

## 前端的顯示行為

前端（MWeb CartClientApp）從 Cart API 收到後：

| 狀態 | 前端行為 |
|------|---------|
| 商品在 `UnmatchPurchasesQualificationSalePageList` | `isUnmatchPurchaseQualification: true` |
| 前端 `useProductCard.tsx` 處理 | `setIsDisabled(true)` + `setMaskType(MaskType.Unpurchasable)` |
| 購物車顯示 | 商品顯示「購買資格限制」標籤，無法操作 |

## 結帳時的攔截

**購物車建立結帳（CheckoutCreate）**：
```csharp
// MergeRequestAndCartContextProcessor.cs
if (checkoutEntity.SalepageGroupList.SelectMany(i => i.SalepageList).Any() == false &&
    checkoutEntity.UnmatchPurchasesQualificationSalePageList.Any())
{
    throw new CheckoutCreateException(
        CheckoutCreateExceptionTypeEnum.UnmatchPurchaseQualificationSalePageOnly);
}
```
→ 如果購物車只剩「購買資格不符」商品（含加購品），結帳會直接 throw。
