# 完整流程追蹤 — 從使用者觸發到清單渲染

## 流程總覽

```
使用者開啟購物車 或 進入加購活動頁面
        │
        ▼
① MWeb 前端（CartExtraPurchaseProvider.tsx）
        │  init() 觸發
        │  GET /V2/CartExtraPurchase/ExtraPurchaseList/{promotionId}?shopId={shopId}
        ▼
② MWeb Controller（Shopping API 層）
        │  接收請求，轉打 Cart Domain API
        ▼
③ Cart Domain：CartService.GetCartExtraPurchaseAsync()
        │  步驟1：呼叫 SPL 拿回所有活動+商品清單
        │  步驟2：客群過濾（MemberCollectionId 比對）
        │  步驟3：MapSplResponseToApiResponse（純 mapping）
        ▼
④ SPL：CartExtraPurchaseService.GetCartExtraPurchaseInfoAsync()
        │  ← 商品清單在此決定（詳見下方）
        ▼
⑤ MWeb 前端渲染（ExtraPurchaseProductList.tsx）
   唯一的客端過濾：排除「所有 SKU 庫存 = 0」的商品
```

## SPL 內部詳細流程

```
GetCartExtraPurchaseInfoAsync(shopId)
  └─ GetCartExtraPurchasePromotionsWithDetailsAsync()
       │
       ├─ 1. PromotionEngineRepository.GetPromotionEnginesByTypeAsync()
       │     條件：CartReachPriceExtraPurchase + 活動時間（含 2h 預取）
       │
       ├─ 2. GetPromotionEngineSpecialPriceDataByPromotionIdsAsync()
       │     取得特價表：ConditionId + SalePageId + SkuId + Price
       │
       ├─ 3. BuildSalePageDictionaryAsync()
       │     └─ SalepagesSearchByIdsAsync(includeInvisibleSalepage: true)
       │          → 回傳 GetSalepageListByIdsResponse（含完整時間欄位）
       │
       └─ 4. BuildPromotionEntitiesAsync()   ← ⚠️ 問題所在
             foreach salePage:
               if (salepage.IsSoldOut) continue;  // 只排除售完
               // ❌ 沒有 SellingStartDateTime 判斷
               conditionEntity.SalePageList.Add(salePageEntity)
  │
  └─ GetCartExtraPurchaseInfoAsync 最後過濾：
       活動時間有效（StartDateTime <= now <= EndDateTime）
```

## Cart Domain 的 Response 結構

Cart Domain 的 `MapSplResponseToApiResponse` 是**純 mapping**，不做任何過濾：

```csharp
// CartService.cs
private List<CartExtraPurchaseResponseEntity> MapSplResponseToApiResponse(...)
{
    return splResponseList.Select(spl => new CartExtraPurchaseResponseEntity
    {
        // 直接對應，無任何過濾邏輯
        SalePageList = condition.SalePageList.Select(salePage => new CartExtraPurchaseSalePageEntity
        {
            SalepageId = salePage.SalepageId,
            IsSoldOut   = salePage.IsSoldOut,
            // ← 無 IsComingSoon、無 SellingStartDateTime
        }).ToList()
    }).ToList();
}
```

## 前端的 Response 結構

`ICartExtraPurchaseSalePage`（MWeb TypeScript 介面）**完全沒有**時間欄位：

```typescript
interface ICartExtraPurchaseSalePage {
    salepageId: number;
    title: string;
    picUrl: string;
    isSoldOut: boolean;
    skuList: ICartExtraPurchaseSku[];
    // isComingSoon → 不存在
    // sellingStartDateTime → 不存在
}
```

前端沒有任何資料可以做時間過濾。
