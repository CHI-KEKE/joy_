```csharp
// After
foreach (var salePageItem in conditionSalePageItems)
{
    var salePageId = salePageItem.SalePageId;
    var salepage = salePageDic.ContainsKey(salePageId) ? salePageDic[salePageId] : null;
    if (salepage == null)
    {
        this._logger.LogWarning($"找不到商品頁資訊，商品頁序號: {salePageId}");
        continue;
    }

    //// 過濾售完商品
    if (salepage.IsSoldOut)
    {
        continue;
    }

    //// 過濾尚未開賣商品（Story #596792）
    //// 以 SellingStartDateTime 為準，忽略 EnableIsComingSoon 商店開關
    if (salepage.SellingStartDateTime > DateTime.Now)
    {
        continue;
    }

    var salePageEntity = await this.BuildSalePageEntityAsync(...);
    conditionEntity.SalePageList.Add(salePageEntity);
}
```

---

## 為什麼用 `SellingStartDateTime > now`

| 選項 | 是否符合 AC |
|------|------------|
| `if (salepage.IsComingSoon)` | ❌ 包含 `EnableIsComingSoon`，AC 說忽略此開關 |
| `if (salepage.SellingStartDateTime > DateTime.Now)` | ✅ 純時間判斷，符合 AC |

## 注意事項

1. **`SellingEndDateTime` 是否也要判斷？**  
   AC 只提到排除 `SellingStartDateTime > now` 的情況，`SellingEndDateTime` 的處理與一般商品下架邏輯相同，`SalepagesSearchByIdsAsync` 內部應已處理，不需額外加判斷。

2. **`IsClosed` 是否需要補充？**  
   AC 提到「商品頁開啟（`SalePage_IsClosed == false`）」，`SalepagesSearchByIdsAsync` 的商品查詢邏輯應已過濾關閉商品，建議確認 `SalepageService` 是否已處理，若否需補上 `if (salepage.StatusDef == "Closed") continue;`。
