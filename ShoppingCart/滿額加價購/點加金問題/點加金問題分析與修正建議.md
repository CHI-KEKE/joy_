# 滿額加價購 — 購物車 P1 加購區選購彈窗出現點加金付款選項問題

> User Story #597915

---

## 一、問題描述

使用者在購物車 P1 頁面下方的「滿額加價購區塊」，點擊加購品的「我要購買」按鈕後，彈出的規格選擇視窗（SKU Popup）中出現了「點加金」的付款方式選項（例如：5點 + NT\$5）。

**加購品不應支援點加金付款**，使用者應只能用加購價（固定金額）購買。

---

## 二、完整呼叫鏈

```
[使用者點擊「我要購買」]
  ↓
CartClientApp: salepageCard.tsx → handleButtonClick()
  ↓
CartClientApp: cartReachPriceExtraPurchaseBlock.tsx → handleAddProductToCart()
  → setSelectedSalepage(...)  將目標商品存入 CartContext
  ↓
CartClientApp: cart.provider.tsx
  → selectedSalepage 有值 → 自動開啟 ProductToCartModal
  ↓
CartClientApp: purchaseExtraBlock.tsx → 渲染 <ProductToCartModal ... />
  ↓
ProductToCartModal (ThemeCore 元件)
  → GET /webapi/Sku/GetSkuPopupInfo/{shopId}/{salepageId}?addOnsSalepage=true
  ↓
MWeb WebAPI: SkuController.GetSkuPopupInfo()
  → 轉呼叫 SPL
  ↓
SPL: SkuQueryService.GetSkuQueryAsync(id, addOnsSalepage=true)
  → 取得商品資料（含點加金 DisplayPointsPayPairsList）
  → 回傳給前端
  ↓
ProductToCartModal 把點加金選項顯示出來  ← 問題在這裡
```

---

## 三、根本原因

問題出在**前端**和**後端**各一個地方，兩者合起來才造成點加金出現。

---

### 3.1 前端問題（CartClientApp）

兩個不同路徑的加購品彈窗，對同一個 `ProductToCartModal` 元件（ThemeCore）的使用方式不一致：

#### 商品加價購（ClientApp）— 正確 ✅

檔案：`ClientApp/src/addOnsSalePage/components/AddOnsSalePageBlock.tsx`

```tsx
<ProductToCartModal
  ...
  isEnablePointPayOption={currentProductToCartModalType === ProductToCartModalType.MainSalePage}
  // 當是加購品時，等於 isEnablePointPayOption={false} → 不顯示點加金

  isAddOnsItem={currentProductToCartModalType !== ProductToCartModalType.MainSalePage}
  // 當是加購品時，等於 isAddOnsItem={true} → 標記為加購品情境
/>
```

明確傳入這兩個 prop，讓元件知道「這是加購品情境」，所以彈窗不會顯示點加金選項。

#### 購物車滿額加價購（CartClientApp）— 有問題 ❌

檔案：`CartClientApp/src/components/blocks/purchaseExtraBlock.tsx`

```tsx
<ProductToCartModal
  ...
  // ❌ 沒有 isEnablePointPayOption
  // ❌ 沒有 isAddOnsItem
/>
```

這兩個 prop 沒傳，`ProductToCartModal` 套用預設值（允許顯示點加金），導致點加金出現。

---

### 3.2 後端問題（SPL — salepage-listing）

檔案：`nine1.salepage.listing.web.api/src/BusinessLogic/Nine1.Salepage.Listing.BL.Services/Salepage/SkuQueryService.cs`

方法：`GetSkuQueryAsync(string id, bool addOnsSalepage, ...)`

問題在 Redis Cache 架構：

```csharp
// Cache key 只用 salepageId，沒有包含 addOnsSalepage
var cacheKeyEntity = this.GetCacheKey("GetSkuQueryAsync", new string[] { salepageId.ToString() });

var result = await this._dataCacheService.GetRedisCacheDataAsync<SkuQueryEntity>(
    ...
    async () =>
    {
        // 點加金資料在 cache closure 裡被組進去
        var pointsPayPair = await this._salepagePointsPayRepository.GetSalepagePointsPayListAsync(...);
        skuQuery.PointsPayList = this.ArrangePointsPayPair(pointsPayPair);
        skuQuery.DisplayPointsPayPairsList = this.ArrangeDisplayPointsPayPairsList(...);

        return skuQuery; // 含點加金的完整資料進 cache（快取 300 秒）
    }, ...);

// addOnsSalepage 參數在這裡完全沒有被使用
return result;
```

**結果**：不管呼叫時 `addOnsSalepage` 是 `true` 或 `false`，都拿到同一份含點加金的 cache，無法區分是否為加購品情境。

---

## 四、修正建議

### 4.1 前端修正（CartClientApp）

**檔案：** `CartClientApp/src/components/blocks/purchaseExtraBlock.tsx`

在 `<ProductToCartModal />` 補上兩個 prop：

```tsx
<ProductToCartModal
  decimal={decimalDigits}
  symbol={symbol}
  rate={rate}
  shopId={shopId}
  salePageId={selectedSalepage.salepageId}
  salePageCode={selectedSalepage.salepageCode}
  facebook={(window as any).FB}
  afterSuccess={...}
  afterClose={...}
  browsingMode={browsingMode}
  isEnableBackInStockFeature={isEnabledBackInStockAlert}
  skuQtyMode={SkuQtyMode.Accumulate}
  isPriceOnTop={isPriceOnTop}
  displayMode={...}
  specifyOptionalTypeDef={selectedSalepage.specifyOptionalTypeDef}
  enableSkuList={selectedSalepage.enableSkuList}
  isEnablePointPayOption={false}   // ✅ 新增：加購品不顯示點加金付款選項
  isAddOnsItem={true}              // ✅ 新增：標記此彈窗為加購品情境
/>
```

| Prop | 作用 |
|---|---|
| `isEnablePointPayOption={false}` | 告訴 `ProductToCartModal` 不要顯示「點加金」的付款方式選項 |
| `isAddOnsItem={true}` | 告訴 `ProductToCartModal` 這是加購品，讓元件內部行為對齊加購品的商業邏輯 |

---

### 4.2 後端修正（SPL）

**檔案：** `nine1.salepage.listing.web.api/src/BusinessLogic/Nine1.Salepage.Listing.BL.Services/Salepage/SkuQueryService.cs`

**修改位置：** `GetSkuQueryAsync` 方法，在 cache 取回後、`return result` 之前加入清空邏輯。

```csharp
var result = await this._dataCacheService.GetRedisCacheDataAsync<SkuQueryEntity>(...);

//// 加購品情境（addOnsSalepage=true）不應顯示點加金選項
//// 在 cache 取回後清空，不修改 cache 內容本身，避免影響一般商品頁的點加金顯示
if (addOnsSalepage)
{
    result.PointsPayList = new List<PointsPayEntity>();
    result.DisplayPointsPayPairsList = new List<SaleProductSKUPointsPayEntity>();
}

return result;
```

**為什麼這樣做，而不是修改 cache key：**

| 方案 | 優點 | 缺點 |
|---|---|---|
| **cache 取回後清空（建議）** | 不影響 cache 結構，不浪費 Redis 空間，改動最小 | 需注意 result 物件是否為 reference type（需確認不會污染 cache） |
| 把 `addOnsSalepage` 納入 cache key | 徹底隔離兩種情境 | 同一商品在 Redis 存兩份幾乎相同的資料，浪費空間，cache 命中率下降 |

> ⚠️ **注意**：若 `GetRedisCacheDataAsync` 回傳的是同一個 reference，直接清空欄位可能會同時污染 cache 內容。實作前需確認此方法是否回傳 **deep copy** 或原始 reference，若為 reference 則需先做 clone 再清空。

---

## 五、影響範圍

| 修改項目 | 影響範圍 |
|---|---|
| `purchaseExtraBlock.tsx` | 僅影響購物車 P1 加購區的彈窗，不影響其他任何頁面 |
| `SkuQueryService.GetSkuQueryAsync` | 僅在 `addOnsSalepage=true` 時清空點加金資料，一般商品頁 popup（`addOnsSalepage=false`）完全不受影響 |
