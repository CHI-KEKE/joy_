## 修改前後對照

### 修改前

```csharp
//// 尋覽主件商品清單並指定買就送贈品資訊清單
foreach (var salePage in salePageList)
{
    if (salePage.IsPrimary == false)
    {
        //// 若為子商品 (商品加價購、組合商品) , 移除買就送屬性
        continue;
    }

    var salePageGiftList = salePageGiftMappingList
        .Where(x => x.Key == salePage.Id)
        .Select(i => i.Value)
        .FirstOrDefault();

    if (salePageGiftList != null && salePageGiftList.Any())
    {
        salePage.SalepageGiftList = salePageGiftList.Select(x => new SalepageGiftEntity
        {
            ...
        }).ToList();
    }
}
```

### 修改後

```csharp
//// 尋覽主件商品清單並指定買就送贈品資訊清單
foreach (var salePage in salePageList)
{
    if (salePage.IsPrimary == false)
    {
        //// 若為子商品 (商品加價購、組合商品) , 移除買就送屬性
        continue;
    }

    //// 若為購物車滿額/滿件加價購活動的加購品，排除買就送贈品
    //// 加購品已享有優惠加購價格，不應同時觸發買就送贈品邏輯
    if (salePage.IsCartExtraPurchase)
    {
        continue;
    }

    var salePageGiftList = salePageGiftMappingList
        .Where(x => x.Key == salePage.Id)
        .Select(i => i.Value)
        .FirstOrDefault();

    if (salePageGiftList != null && salePageGiftList.Any())
    {
        salePage.SalepageGiftList = salePageGiftList.Select(x => new SalepageGiftEntity
        {
            ...
        }).ToList();
    }
}
```

## 為什麼只改這一處就夠

`SalepageGiftList` 只有一個寫入入口：`GetShoppingCartSalePageGiftProcessor`。  
修改後，滿額加購品的 `SalepageGiftList` 保持預設空 list，後續所有依賴此資料的環節自動正確：

| 環節 | 修改前 | 修改後 |
|------|-------|-------|
| `CalculateShoppingCartSalePageGiftQtyLimitProcessor` | 依贈品庫存錯誤調降加購品數量 | `SalepageGiftList` 空 → 自動跳過 ✅ |
| `OldCartMapperService` | 帶出贈品資料給 Shopping API | 帶出空 list ✅ |
| MWeb `productCard.tsx` | 顯示贈品標籤 | `!!salePageGiftList?.length` = false → 不渲染 ✅ |
| `DoPayProcessFlowProcessor`（結帳） | 分配贈品庫存、訂單含贈品 | 無贈品資料 → 不分配 ✅ |

---

## 為什麼不需要改 Shopping API / MWeb

| 層次 | 原因 |
|------|------|
| Shopping API | Pass-through，不處理贈品邏輯，Cart API 送空就傳空 |
| MWeb | `productCard.tsx` 已是條件渲染 `!!length`，資料為空自動不顯示 |

---

## 不受影響的情境

| 情境 | `IsCartExtraPurchase` | 結果 |
|------|----------------------|------|
| 商品以**一般方式**加入 | `false` | 買就送**正常觸發** ✅ |
| 同商品同時以一般方式加入（同一訂單） | `false` | 買就送**正常觸發** ✅ |
| 商品加價購**主品** | `false` | 買就送**正常觸發** ✅ |
| 商品加價購**子品** | `false`（已被 `IsPrimary==false` 排除） | 不受影響 ✅ |
| **滿額加購品** | **`true`** | 買就送**被排除** ✅ |
| **滿件加購品** | **`true`** | 買就送**被排除** ✅ |

| 商品加價購主品的買就送 | 行為**完全不受影響** |
