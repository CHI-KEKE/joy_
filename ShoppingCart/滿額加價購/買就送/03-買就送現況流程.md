# 03 — 買就送現況流程

> 說明買就送贈品從哪裡產生、怎麼流向前端、以及問題發生在哪個環節。

---

## 買就送資料來源

**資料表**：`SalePageGift`（主檔）、`SalePageGiftSlave`（明細）  
**後台入口**：OSM → 贈品管理 → 商品贈品(買就送)  
**查詢方式**：`csp_GetSalePageGiftList` Stored Procedure

這是「商品頁層級」的贈品設定，與促銷活動（滿額贈/滿件贈）是**完全不同的機制**。

| 類型 | 設定層級 | 觸發方式 |
|------|---------|---------|
| **買就送（本需求）** | 商品頁後台設定 | 購物車有該 SalePage 就觸發 |
| 滿額贈 / 滿件贈 | 促銷活動設定 | 達到金額/件數門檻才觸發 |

## 買就送完整流程（含 BUG 路徑）

```
[Cart API] GetShoppingCartSalePageGiftProcessor
  ─────────────────────────────────────────────
  購物車類型 == Express 或 PayPage？
    → YES：直接 return（不處理買就送）

  salePageList = 購物車所有商品
  salePageIdList = 所有商品的 SalePageId（含加購品 ID）

  一次呼叫 GetSalePageGiftListAsync(salePageIdList)
    → 查 SalePageGift 資料表
    → 回傳 Dictionary<SalePageId, List<贈品>>

  foreach salePage in salePageList:

    if (salePage.IsPrimary == false)
      → continue（排除 AddOns/Bundle 子品）✅

    ← ❌ 此處缺少 IsCartExtraPurchase 的排除
    ← CartReachPriceExtraPurchase 商品 IsPrimary = true，直接通過

    salePage.SalepageGiftList = [贈品 A, 贈品 B]  ← BUG：加購品被賦予贈品資料

  ─────────────────────────────────────────────
  多語系標題置換（SalepageGiftList）

[Cart API] CalculateShoppingCartSalePageGiftQtyLimitProcessor
  ─────────────────────────────────────────────
  foreach salePage:
    if SalepageGiftList.Any():
      ← BUG 副作用：加購品的贈品庫存 < 加購品 Qty
      → 強制調降加購品的 Qty 和 QtyLimit！  ← 影響加購品購買數量

[Cart API] OldCartMapperService
  ─────────────────────────────────────────────
  SalePageGiftList = salePage.SalepageGiftList.Select(→ ShoppingCartClientSalePageGiftEntity)
    ← BUG：把贈品資料帶回給 Shopping API

[Shopping API] → pass-through → [MWeb]

[MWeb] useProductData.tsx
  ─────────────────────────────────────────────
  salePageGiftList: getGiftProductData(salePage.SalePageGiftList)
    ← 無條件 map，不過濾 optionalTypeDef

[MWeb] productCard.tsx
  ─────────────────────────────────────────────
  {!!salePageGiftList?.length && <ProductGiftList data={salePageGiftList} />}
    ← BUG：加購品商品卡顯示「贈品」標籤

[結帳] DoPayProcessFlowProcessor
  ─────────────────────────────────────────────
  提取所有 SalepageGiftList → 驗證庫存 → 寫入 Redis → 訂單含贈品
    ← BUG：加購品贈品進入訂單，實際分配贈品庫存
```

---

## BUG 影響清單

| 影響點 | 說明 |
|-------|------|
| 購物車 UI | 加購品商品卡顯示贈品標籤和贈品預覽 |
| 購買數量 | 若贈品庫存少於加購品數量，加購品的可購數量被錯誤壓低 |
| 結帳試算 | 贈品清單包含因加購品觸發的贈品 |
| 訂單成立 | 贈品庫存被扣除、贈品被實際分配給消費者 |
