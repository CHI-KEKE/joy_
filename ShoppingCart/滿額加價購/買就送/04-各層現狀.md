## 狀態總覽

| 層次 | 項目 | 狀態 |
|------|------|------|
| Cart API | `IsCartExtraPurchase` 屬性定義 | ✅ 已就緒 |

## Cart API 詳細現狀

### ✅ IsCartExtraPurchase（已就緒）

```csharp
// SalePageEntityForCart.cs / SalepageEntity.cs
public bool IsCartExtraPurchase
{
    get => ArrangeIsCartExtraPurchase();
}

private bool ArrangeIsCartExtraPurchase()
{
    var result = false;
    result = OptionalTypeDef is nameof(SalePageOptionalTypeEnum.CartReachPriceExtraPurchase)
                             or nameof(SalePageOptionalTypeEnum.CartReachPieceExtraPurchase)
             && OptionalTypeId > 0;
    return result;
}
```

## Shopping API 詳細現狀

### ✅ Pass-through 設計，不需修改

Shopping API 的 `CartCreateProcessor` 直接轉發 Cart API 回應：

```csharp
context.Data = result.Data!;
```

`SalePageGiftList` 是 Cart API 填充的，Shopping API 只負責轉傳。`SalepageEntity.SalepageGiftList` 是 `List<SalepageGiftEntity>`，若 Cart API 送空 list，Shopping API 也只傳空 list。

## MWeb 詳細現狀

### ✅ 條件渲染，不需修改

**useProductData.tsx**：
```typescript
salePageGiftList: getGiftProductData(salePage.SalePageGiftList),
// 直接 map，沒有 optionalTypeDef 過濾
// 但若 SalePageGiftList 為空，map 結果也為空 []
```

**productCard.tsx**：
```typescript
{!!salePageGiftList?.length && <ProductGiftList data={salePageGiftList} />}
// salePageGiftList 為空 [] → !!0 = false → 不渲染 ✅
```

MWeb 的渲染邏輯本身是正確的：有資料才顯示。問題在 Cart API 把不該有的資料給了加購品。

## 各層資料流向

```
Cart API GetShoppingCartSalePageGiftProcessor
  現況：SalepageGiftList = [贈品] （加購品也有）
  修後：SalepageGiftList = []    （加購品為空）
              ↓
Cart API OldCartMapperService
  現況：SalePageGiftList = [贈品]
  修後：SalePageGiftList = []
              ↓
Shopping API（pass-through）
              ↓
MWeb productCard
  現況：顯示贈品標籤  ← BUG
  修後：不渲染       ← 正確
```
