# 02 — 識別機制

> 解釋為何 `IsPrimary` 無法排除滿額加購品，以及 `IsCartExtraPurchase` 為何是正確的識別手段。

---

## 三個識別屬性

### IsPrimary

```csharp
private bool ArrangeIsPrimary()
{
    var result = true;
    if (string.IsNullOrEmpty(CartExtendInfoItemTypeDef))
    {
        return result;  // 空字串 → 預設 true
    }
    result = CartExtendInfoItemTypeDef == nameof(CartExtendInfoItemTypeEnum.Major);
    return result;
}
```

**語意**：這個商品在購物車的 `CartExtendInfo` 父子結構中，是主品（Major）還是子品（Sub）？  
**用途**：排除「商品加價購的加購子品」和「組合商品的子品」

---

### IsSubAddOns

```csharp
public bool IsSubAddOns()
{
    var result = false;
    if (string.IsNullOrEmpty(CartExtendInfoItemTypeDef)) return result;

    result = CartExtendInfoItemGroup > 0
             && CartExtendInfoItemTypeDef == nameof(CartExtendInfoItemTypeEnum.Sub)
             && CartExtendInfos.Any(x => x.RuleTypeDef ==
                nameof(CartExtendInfoRuleTypeDefEnum.AddOnsSalepageExtraPurchase));
    return result;
}
```

**語意**：這個商品是否為「商品加價購」活動的子品？  
**用途**：定期購降級邏輯中排除商品加價購子品

---

### IsCartExtraPurchase

```csharp
private bool ArrangeIsCartExtraPurchase()
{
    var result = false;
    result = OptionalTypeDef is nameof(SalePageOptionalTypeEnum.CartReachPriceExtraPurchase)
                             or nameof(SalePageOptionalTypeEnum.CartReachPieceExtraPurchase)
             && OptionalTypeId > 0;
    return result;
}
```

**語意**：這個商品是否以「購物車滿額/滿件加價購」加購品身分加入？  
**條件**：`OptionalTypeDef` 符合加購類型 **AND** `OptionalTypeId > 0`（合法的特價 ID）

---

## 兩種加價購的結構差異

### 商品加價購（AddOnsSalepageExtraPurchase）

使用 **CartExtendInfo** 父子結構：

```
購物車 SalepageGroupList
 ├── 商品 A（主品）
 │     CartExtendInfoItemTypeDef = "Major"
 │     IsPrimary = true
 │     IsCartExtraPurchase = false
 │
 └── 商品 B（加購子品）
       CartExtendInfoItemTypeDef = "Sub"
       CartExtendInfoRuleTypeDef = "AddOnsSalepageExtraPurchase"
       IsPrimary = false  ← GetShoppingCartSalePageGiftProcessor 的 continue 排除這個
       IsSubAddOns() = true
       IsCartExtraPurchase = false
```

### 滿額加價購（CartReachPriceExtraPurchase）

使用 **OptionalTypeDef** 獨立標記：

```
購物車 SalepageGroupList
 ├── 商品 A（一般購買）
 │     CartExtendInfoItemTypeDef = ""
 │     OptionalTypeDef = ""
 │     IsPrimary = true
 │     IsCartExtraPurchase = false
 │
 └── 商品 A（滿額加購品）← 同一商品，不同購物車紀錄
       CartExtendInfoItemTypeDef = ""
       OptionalTypeDef = "CartReachPriceExtraPurchase"
       OptionalTypeId  = specialPriceId（> 0）
       IsPrimary = true  ⚠️  ← 和一般商品一樣！
       IsCartExtraPurchase = true  ✅ ← 唯一能區分的屬性
```

---

## 為什麼 IsPrimary 無法處理滿額加購品

| 加購品類型 | CartExtendInfoItemTypeDef | IsPrimary | 能被現有 `continue` 排除？ |
|-----------|--------------------------|-----------|--------------------------|
| 商品加購子品 | `"Sub"` | `false` | ✅ 能 |
| 組合商品子品 | `"Sub"` | `false` | ✅ 能 |
| **滿額加購品** | `""` 空字串 | **`true`** | ❌ **不能** |
| **滿件加購品** | `""` 空字串 | **`true`** | ❌ **不能** |

滿額/滿件加購品沒有使用 CartExtendInfo 機制，`CartExtendInfoItemTypeDef` 為空，`IsPrimary` 預設為 `true`，完全繞過現有排除條件。

---

## 正確的排除屬性

| 需排除的情境 | 應使用的屬性 |
|------------|------------|
| 商品加價購子品（AddOns Sub） | `IsPrimary == false` |
| 組合商品子品（Bundle Sub） | `IsPrimary == false` |
| **滿額加價購加購品** | **`IsCartExtraPurchase == true`** |
| **滿件加價購加購品** | **`IsCartExtraPurchase == true`**（同一屬性涵蓋） |

---

## OptionalTypeId > 0 的必要性

`IsCartExtraPurchase` 要求 `OptionalTypeId > 0`，這個值是 MWeb 傳入的 `sku.specialPriceId`。  
若為 0 代表不是透過合法加購條件加入，此時不應套用加購品邏輯，避免誤判。
