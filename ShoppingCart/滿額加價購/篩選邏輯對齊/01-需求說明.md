# 01 需求說明

## Story 背景

消費者在購物車 P1 達成滿額門檻後，系統會於下方加購區或跳轉至「加購品列表頁」供其選購。
若撈取邏輯不夠嚴謹，清單中可能會出現：
- 不適合單獨作為加購品的「組合商品」
- 已被商家關閉商品頁的「下架商品」
- 需額外執行分享行為才能購買的受限商品

## 功能路徑

| 入口 | 路徑 |
|------|------|
| 商品頁 → 加購列表 | `https://{shop}/V2/CartExtraPurchase/ExtraPurchaseList/{promotionId}?shopId={shopId}` |
| 購物車 → 加購列表 | 同上 URL + `&from=cart&cartUniqueKey={key}` |
| 購物車 P1 加購區 | 購物車頁面 P1 下方直接顯示的加購品卡片 |

## Acceptance Criteria

### AC1：驗證顯示位置與路徑
- **位置 A**：購物車頁 P1 下方的活動加購區
- **位置 B**：透過商品頁或購物車進入的「加購品列表頁」
- 預期：兩個位置的商品篩選邏輯一致

### AC2：驗證強制排除項目
| 類型 | 排除條件 |
|------|---------|
| 組合商品 | 商品類別為「組合商品」者不得出現 |
| FB 分享後購買商品 | 設定了「需分享至 FB 方可購買」的商品應排除 |
| 商品頁設定關閉 | 商品頁狀態為「關閉」時不得顯示 |

### AC3：驗證基礎狀態有效性
- 加購商品必須符合「活動有效、商品販售時間有效、商品頁開啟」之基本三角檢核
- **邏輯對齊**：滿額加價購的篩選邏輯需 100% 參照現有「商品加價購」過濾機制

## 對照基準：商品加價購 (CartAddOns)

商品加價購加購頁：`https://{shop}/V2/AddOnsSalePage/ExtraPurchaseList/{id}?shopId={shopId}`

對應 SPL Service：`SalepageExtraPurchaseService`

### CartAddOns 現行已排除項目
- `IsClosed`：商品頁設定關閉
- `SellingTime`：商品販售時間（開始/結束）
- `IsInvisible`：商品隱形（不公開）

### CartAddOns 尚未排除項目（⚠️ 需確認）
- `IsSalePageBundle`：組合商品（CartAddOns 也沒排除）
- `IsShareToBuy`：FB 分享後購買（CartAddOns 也沒排除）

> **注意**：Story AC 要求「對照商品加價購篩選邏輯」，但 CartAddOns 本身也未排除以上兩項。
> 這代表 AC 有超出對照基準的要求，需要與 PM/PD 澄清範圍。
