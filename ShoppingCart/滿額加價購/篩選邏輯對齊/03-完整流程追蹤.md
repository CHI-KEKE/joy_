# 03 完整流程追蹤

## 兩個入口的呼叫路徑

### 入口 A：加購品列表頁（商品頁進入 / 購物車進入）

```
使用者點擊「查看全部加購品」
      │
      ▼
React 前端（MobileWebMall MVC）
  GET /V2/CartExtraPurchase/ExtraPurchaseList/{promotionId}?shopId={shopId}
      │
      ▼
MWeb BLV2 - ExtraPurchaseService（待確認 Controller 名稱）
      │
      ▼
SPL HTTP Client
  GET /salepage/cart-extra-purchase/{shopId}/{promotionId}
      │
      ▼
★ SPL CartExtraPurchaseService.GetCartExtraPurchaseAsync()
      │
      ├─ 取得活動設定的加購品 SalePageId 清單
      │
      ├─ SalePageListingHttpClient.GetSalepageListByIdsAsync()
      │    └─ 回傳 GetSalepageListByIdsResponse（含 IsSalePageBundle、IsClosed 等欄位）
      │
      └─ BuildPromotionEntitiesAsync()   ← ★ 主要修改點
           ├─ 現有：過濾 IsSoldOut
           ├─ 現有：過濾 SellingTime（Story #596792 已修正）
           ├─ ❌ 缺少：過濾 IsSalePageBundle
           └─ ❌ 缺少：過濾 IsClosed
```

### 入口 B：購物車 P1 加購區

```
使用者進入購物車頁，P1 下方顯示加購品
      │
      ▼
React 前端（MobileWebMall MVC）
      │
      ▼
Cart Domain API
  GET /carts/cart-extra-purchase?cartUniqueKey={key}
      │
      ▼
★ Cart Domain - CartService.GetCartExtraPurchaseAsync()  [Line 834]
      │
      ├─ 呼叫 ISalePageListingHttpClient.GetCartExtraPurchaseAsync()
      │    └─ 等同於入口 A 的 SPL 呼叫
      │         └─ ★ SPL CartExtraPurchaseService（同一修改點）
      │
      └─ MemberCollectionMatchAsync()   ← 會員客群過濾（Cart Domain 層）
           └─ 回傳 List<SplCartExtraPurchaseResponseEntity>
```

## 結論：一處修改，兩個入口同步修正

| 入口 | 路徑 | 最終 Service |
|------|------|-------------|
| 加購品列表頁 | MWeb → SPL direct | SPL `CartExtraPurchaseService` |
| P1 加購區 | MWeb → Cart API → SPL | SPL `CartExtraPurchaseService`（同上）|

✅ **只需修改 SPL `CartExtraPurchaseService.BuildPromotionEntitiesAsync()`，兩個入口即同步修正。**
