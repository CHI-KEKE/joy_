# 05 — DI 註冊與設定

## 異動檔案

| 檔案 | 異動類型 |
|---|---|
| `AuditProcessorDefinitionCenter.cs` | 新增 Processor 到 OrderPlace 層 |

> `OrderAuditJobDIAttribute.cs` **不需要手動改**，它的 `foreach` 迴圈會自動掃描 `AuditProcessorDefinitionCenter` 的所有 Processor 並完成 DI 註冊。

---

## AuditProcessorDefinitionCenter 異動

**路徑：** `src/Nine1.Commerce.Console.BL/Services/AuditProcessor/AuditProcessorDefinitionCenter.cs`

在 `OrderPlaceAuditProcessorLayers` 中加入新 Processor：

```csharp
public static List<Type> OrderPlaceAuditProcessorLayers => new List<Type>
{
    typeof(OrderInfoProcessor),
    typeof(SalepageExtendInfoAuditProcessor),
    typeof(SalepageBundleAuditProcessor),
    typeof(OrderCouponDataAuditProcessor),
    typeof(SalepagePointsPayAuditProcessor),
    typeof(CartExtraPurchaseAuditProcessor),   // ← 新增
};
```

---

## 加入位置說明

放在最後一個是合理的，因為：
1. `OrderInfoProcessor` 最先跑，確認訂單基本資訊
2. `SalepageExtendInfoAuditProcessor` 處理商品加價購（舊）
3. `CartExtraPurchaseAuditProcessor` 處理滿額加價購（新），相似功能放在相鄰位置

---

## 確認 INine1PromotionClient 已在 DI 中註冊

搜尋現有 DI 設定確認 `INine1PromotionClient` / `Nine1PromotionClient` 是否已被 `OrderAuditJobDIAttribute` 或其他地方註冊。

若未註冊，需在 `OrderAuditJobDIAttribute.cs` 補上：
```csharp
services.AddScoped<INine1PromotionClient, Nine1PromotionClient>();
```
