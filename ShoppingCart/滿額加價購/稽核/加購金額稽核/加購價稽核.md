# 加購品價格稽核

## 概述

針對訂單中的「滿額加價購（CartReachPriceExtraPurchase）」加購品，驗證其結帳價格是否與 Promotion API 活動設定的加購價一致。

- **Processor**：`CartExtraPurchaseAuditProcessor`
- **稽核來源**：購物車快照（PayProcessContext.ShoppingCartV2）
- **參考資料**：Promotion API

---

## 跳過條件（IsSkip）

符合以下任一條件時，此稽核項目不執行：

| 條件 | 原因 |
|---|---|
| `IsForCache == false` | 非 Cache 路徑，購物車快照不存在，無比對依據 |
| 購物車快照中無 `IsCartExtraPurchase == true` 的商品 | 本次訂單不包含滿額加價購加購品 |

---

## 稽核流程

### Step 1：收集加購品

從 `ShoppingCartV2.SalePageGroupList` 展開所有 `SalePage`，篩選 `IsCartExtraPurchase == true` 的商品，同時收集不重複的 `SalePageId` 清單供後續 API 查詢使用。

---

### Step 2：查詢在期活動清單

呼叫 Promotion API：`POST /api/promotion-rules/ongoing-list`

**Request：**
```json
{
  "Type": ["CartReachPriceExtraPurchase"],
  "StartDate": "orderDateTime",
  "EndDate": "DateTime.Now（稽核當下）"
}
```

**DB 查詢語意：**
```
StartDateTime <= EndDate（稽核當下）
AND EndDateTime >= StartDate（下單時間）
```

**業務語意：** 在下單時尚未結束、且在稽核前已開始的活動。

**若回傳空清單：**  
→ 有加購品但找不到在期活動（商家刪除或提前結束），判定為 **Fail**。

---

### Step 3：查詢活動加購品特殊價格

對每個在期活動呼叫 Promotion API：`POST /api/promotion-price-lists/get-promotion-price`

**Request：**
```json
{
  "PromotionEngineId": 123,
  "SalePageIds": [456, 789]
}
```

**說明：**
- 僅回傳 `ValidFlag = true` 的資料
- 不屬於該活動的 `SalePageId` 會被靜默過濾（不視為錯誤）
- 彙總所有在期活動的結果至 `allSpecialPrices`

---

### Step 4：逐一比對加購品價格（ValidateExtraPurchasePrice）

**匹配鍵（三鍵）：**
```
SalePageId + SaleProductSKUId + SpecialPriceId
```

| 情境 | 處理方式 |
|---|---|
| 查無對應特殊價格（`matched == null`） | Warning Log + **跳過**（加購品可能已從活動移除，無法判斷） |
| 價格相符 | 通過 |
| 價格不符，但 `matched.UpdatedDateTime >= orderDateTime` | Info Log + **跳過**（加購價在訂單後才更新，視為商家合法調整） |
| 價格不符，且 `matched.UpdatedDateTime < orderDateTime` | 加入失敗訊息 → **Fail** |

---

## 稽核結果

- **全部通過**：輸出 Info Log「稽核通過」
- **任一 Fail**：呼叫 `AuditFailLog`，輸出 `MetricsLogEntity`（Type = `CartExtraPurchaseAuditProcessor`），觸發 Slack 告警

**Fail 訊息格式：**
```
加購品價格不符 SalePageId={X} SkuId={Y} CachePrice={Z} ActivityPrice={W}
```
多筆失敗以 ` | ` 分隔。

---

## 關鍵欄位

| 欄位 | 來源 | 說明 |
|---|---|---|
| `SalePage.IsCartExtraPurchase` | 購物車快照 | 識別加購品 |
| `SalePage.SpecialPriceId` | 購物車快照 | 對應活動設定的特殊價格 ID |
| `SalePage.Price` | 購物車快照 | 使用者結帳時看到的加購價（稽核對象） |
| `SalePage.SaleProductSKUId` | 購物車快照 | 商品 SKU，用於三鍵匹配 |
| `SpecialPrice.TypeValue` | Promotion API | 活動設定的加購價（參考基準） |
| `SpecialPrice.UpdatedDateTime` | Promotion API | 加購價最後更新時間，用於判斷是否在訂單後才異動 |

---

## 下單時間取得邏輯

```csharp
var orderDateTime = context.ShoppingCartV2.ShoppingDateTime ?? context.EventOrderDateTime;
```

優先使用購物車產生時間（`ShoppingDateTime`），若無則 fallback 至事件攜帶的訂單時間（`EventOrderDateTime`）。
