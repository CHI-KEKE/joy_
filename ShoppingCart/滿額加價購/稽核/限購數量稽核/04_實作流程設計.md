# 限購數量稽核 — 實作流程設計

## 整體流程（擴充現有 ExecuteAsync）

```
Step 1（現有）: 從購物車快照收集加購品 extraPurchaseItems

Step 2（現有）: GetOngoingPromotionsAsync → ongoingPromotions

Step 3（現有 + 擴充）: foreach promotion in ongoingPromotions
  3-1（現有）: GetPromotionSpecialPricesAsync → specialPrices
  3-2（新增）: GetPromotionDetailAsync       → promotionDetail
  3-3（新增）: 合併 specialPrices + promotionDetail.ExtraPurchaseRuleList
               → 建立 ExtraPurchaseQtyLimitInfoEntity list

Step 4（現有）: 逐一比對加購品價格（ValidateExtraPurchasePrice）

Step 5（新增）: 以 (PromotionId, ConditionId) 分組，稽核限購數量
               （ValidateExtraPurchaseQtyLimit）
```

---

## 中間 Entity 設計（ExtraPurchaseQtyLimitInfoEntity）

在 Step 3-3 迴圈內，將兩個 API 的資料合併成此 entity list：

```csharp
public class ExtraPurchaseQtyLimitInfoEntity
{
    public int    PromotionId   { get; set; }  // 用於分組，避免跨活動混用
    public int    ConditionId   { get; set; }  // 門檻識別
    public long   SpecialPriceId { get; set; } // 匹配購物車加購品的鍵
    public long   SkuId          { get; set; } // 匹配購物車加購品的鍵
    public decimal ReachPrice    { get; set; } // Cyclable 計算用
    public int    LimitQty       { get; set; } // -1 = 無限制
    public bool   IsCyclable     { get; set; } // 是否循環
}
```

組建方式（迴圈內）：

```
foreach sp in specialPrices:
    condition = promotionDetail.ExtraPurchaseRuleList
                  .FirstOrDefault(r => r.ConditionId == sp.ConditionId)
    if condition == null: skip（資料不完整）

    add ExtraPurchaseQtyLimitInfoEntity:
      { PromotionId   = promotion.Id,
        ConditionId   = sp.ConditionId,
        SpecialPriceId = sp.Id,
        SkuId          = sp.SaleProductSKUId,
        ReachPrice     = condition.ReachPrice,
        LimitQty       = condition.LimitQty,
        IsCyclable     = promotionDetail.IsCyclable }
```

## Step 5 稽核邏輯（ValidateExtraPurchaseQtyLimit）

```
// cartTotalPrice = 購物車所有非加購品的 Price × Qty 加總
var cartTotalPrice = extraPurchaseItems 以外的 SalePage Price * Qty 加總

foreach group in qtyLimitInfoList.GroupBy(x => (x.PromotionId, x.ConditionId)):

    var limitQty   = group.First().LimitQty
    var reachPrice = group.First().ReachPrice
    var isCyclable = group.First().IsCyclable

    // -1 代表無限制
    if limitQty == -1: continue

    // 找出購物車中屬於此 (PromotionId, ConditionId) 的加購品
    var matchedItems = extraPurchaseItems.Where(
        item => group.Any(g => g.SpecialPriceId == item.SpecialPriceId
                            && g.SkuId == item.SaleProductSKUId))

    if matchedItems.IsEmpty: continue

    var actualQty = matchedItems.Sum(Qty)

    int allowedQty;
    if isCyclable:
        var cycles = floor(cartTotalPrice / reachPrice)
        allowedQty = cycles * limitQty
    else:
        allowedQty = limitQty

    if actualQty > allowedQty:
        → Fail（記錄 PromotionId, ConditionId, actualQty, allowedQty）
```

---

## 判斷表

| 情境 | allowedQty 計算 | 處理 |
|---|---|---|
| `LimitQty == -1` | — | 跳過，無限制 |
| 找不到對應 ConditionId | — | Warning + 跳過，資料不完整 |
| `IsCyclable = false` | `LimitQty` | 直接比對 |
| `IsCyclable = true` | `floor(cartTotal / ReachPrice) × LimitQty` | 循環計算後比對 |
| `actualQty > allowedQty` | — | Fail |
| `actualQty <= allowedQty` | — | 通過 |
