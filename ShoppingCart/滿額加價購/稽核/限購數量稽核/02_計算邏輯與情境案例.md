# 限購數量稽核 — 計算邏輯與情境案例

## 基本計算公式

```
// 不循環
allowedQty = LimitQty

// 循環（IsCyclable = true）
cycles     = floor(cartTotalPrice / ReachPrice)
allowedQty = cycles * LimitQty

// 稽核判斷
actualQty = 該 (PromotionId, ConditionId) 下所有加購品 Qty 加總
if actualQty > allowedQty → Fail
```

---

## Case 1：單階層、不循環、未超限 ✅

```
設定：ReachPrice=100, LimitQty=2, IsCyclable=false
cartTotalPrice = 150
加購品 9001 Qty = 2

allowedQty = 2（不循環，直接取 LimitQty）
actualQty  = 2
結果：通過
```

---

## Case 2：單階層、不循環、超限 ❌

```
設定：ReachPrice=100, LimitQty=2, IsCyclable=false
cartTotalPrice = 350（花再多也不影響 allowedQty）
加購品 9001 Qty = 3

allowedQty = 2
actualQty  = 3
結果：Fail（不循環時消費金額不影響上限）
```

---

## Case 3：單階層、循環、未超限 ✅

```
設定：ReachPrice=100, LimitQty=2, IsCyclable=true
cartTotalPrice = 250
加購品 9001 Qty = 4

cycles     = floor(250 / 100) = 2
allowedQty = 2 × 2 = 4
actualQty  = 4
結果：通過
```

---

## Case 4：單階層、循環、超限 ❌

```
設定：ReachPrice=100, LimitQty=2, IsCyclable=true
cartTotalPrice = 250
加購品 9001 Qty = 5

cycles     = floor(250 / 100) = 2
allowedQty = 2 × 2 = 4
actualQty  = 5
結果：Fail
```

---

## Case 5：多階層、不循環、各門檻獨立稽核 ✅

```
設定：
  ConditionId=1: ReachPrice=100, LimitQty=2
  ConditionId=2: ReachPrice=200, LimitQty=3
  IsCyclable=false
cartTotalPrice = 250（兩個門檻都達到）

加購品 9001（ConditionId=1）Qty = 2
加購品 9002（ConditionId=2）Qty = 3

ConditionId=1: allowedQty=2, actualQty=2 → 通過
ConditionId=2: allowedQty=3, actualQty=3 → 通過

兩個門檻獨立計算、互不影響。
```

---

## Case 6：多階層，其中一個門檻超限 ❌

```
設定同 Case 5

加購品 9001（ConditionId=1）Qty = 3  ← 超過
加購品 9002（ConditionId=2）Qty = 2

ConditionId=1: allowedQty=2, actualQty=3 → Fail ❌
ConditionId=2: allowedQty=3, actualQty=2 → 通過 ✅

只要任一門檻超限即整體 Fail。
```

---

## Case 7：多階層、循環，各門檻用各自 ReachPrice 計算 ✅

```
設定：
  ConditionId=1: ReachPrice=100, LimitQty=2
  ConditionId=2: ReachPrice=200, LimitQty=1
  IsCyclable=true
cartTotalPrice = 400

加購品 9001（ConditionId=1）Qty = 8
加購品 9002（ConditionId=2）Qty = 2

ConditionId=1: cycles=floor(400/100)=4, allowedQty=4×2=8, actualQty=8 → 通過
ConditionId=2: cycles=floor(400/200)=2, allowedQty=2×1=2, actualQty=2 → 通過

每個門檻用「自己的 ReachPrice」計算循環次數，互不影響。
```

---

## Case 8：LimitQty = -1，無限制 ✅

```
設定：LimitQty=-1
加購品 Qty = 999

-1 代表商家未設限，直接跳過此門檻稽核，不判斷。
```

---

## Case 9：同門檻包含多種加購商品，加總稽核 ❌

```
設定：ConditionId=1, LimitQty=3, IsCyclable=false

加購品 9001（ConditionId=1）Qty = 2
加購品 9003（ConditionId=1）Qty = 2  ← 同一門檻的不同商品

actualQty = 2 + 2 = 4（同門檻所有加購品 Qty 加總）
allowedQty = 3
結果：Fail（不能對每個商品個別比，要加總）
```

> ⚠️ 這是最容易實作錯誤的 Case：必須「以 (PromotionId, ConditionId) 分組後加總 Qty」，不能「逐一比對每個加購品的 Qty 對上 LimitQty」。
