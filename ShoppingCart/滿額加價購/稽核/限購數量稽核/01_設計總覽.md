# 滿額加價購 — 限購數量稽核 設計總覽

## 稽核目的

驗證購物車快照中，每個滿額加價購活動下，各門檻（ConditionId）的加購品**實際購買總量**是否超過活動設定的限購上限（LimitQty）。

---

## 資料來源

| 資料 | 取得方式 | 欄位 |
|---|---|---|
| 在期活動清單 | `GetOngoingPromotionsAsync`（現有） | `PromotionId` |
| 加購品特殊價格 | `GetPromotionSpecialPricesAsync`（現有） | `SpecialPriceId`, `SalePageId`, `SkuId`, `ConditionId` |
| 活動詳細設定 | `GetPromotionDetailAsync`（新增） | `IsCyclable`, `ExtraPurchaseRuleList[].ConditionId/ReachPrice/LimitQty` |
| 加購品實際購買資訊 | 購物車快照 `SalePageGroupList`（現有） | `SpecialPriceId`, `SaleProductSKUId`, `Qty` |

---

## 核心設計決策

### 1. 匹配鍵：`SpecialPriceId + SkuId`

用 `SpecialPriceId` 作為主要匹配鍵，因為它天然代表「某活動 × 某門檻 × 某商品」的唯一組合，可正確應對多活動、多門檻包含相同商品的情境。

### 2. 分組鍵：`(PromotionId, ConditionId)`

LimitQty 是**門檻層級**的限制，代表「此活動此門檻下，所有加購品加起來最多能買幾件」。  
必須以 `(PromotionId, ConditionId)` 分組後加總 Qty，而不是對每個加購品個別比對。

### 3. 允許數量計算方式

```
if LimitQty == -1:
    → 無限制，跳過

if IsCyclable == false:
    allowedQty = LimitQty

if IsCyclable == true:
    cycles = floor(cartTotalPrice / ReachPrice)
    allowedQty = cycles * LimitQty
```

### 4. cartTotalPrice 計算範圍

以購物車快照中**所有非加購品**（`IsCartExtraPurchase = false`）的 `Price × Qty` 加總，與 Promotion Engine 計算行為一致。
