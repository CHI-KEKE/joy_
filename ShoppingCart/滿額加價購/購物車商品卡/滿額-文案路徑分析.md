# 滿額加價購 文案節點路徑分析

> 範圍：`CartReachPriceExtraPurchase`（滿額加價購）  
> 資料流方向：Promotion Frontend API → Cart → Shopping

---

## 涉及 API 端點總覽

| 呼叫方 | 端點 | 用途 |
|---|---|---|
| Cart | `POST /api/cart-calculate` | 計算折扣、產生 MaxReachCondition / PricePairList |
| Cart | `POST /api/promotion-rules/list-by-ids` | 取得活動名稱（Name） |
| Cart | `POST /api/salepage-collections/salepage-collection-match` | 確認商品被哪個活動圈選 |

---

## 文案一：`CartExtraPurchase`（加購品商品卡標籤）

**期望文案**：`{活動名稱}累積滿額 $500`  
**DisplayType**：`CartExtraPurchase`  
**觸發時機**：加購品已達門檻（`IsMatch = true`），顯示在加購品商品卡

### 節點路徑

```
【Promotion Frontend API】
  DB (SpecialPrice 表)
    └─ GetCartExtraPurchaseSpecialPriceListAsync()
         → Dictionary<ConditionId, List<SpecialPriceId>>

  ParsePromotionEngineRuleObject()
    └─ CartExtraPurchaseConditionEntity {
           ConditionId,
           ReachPrice,        ← 門檻金額
           SpecialPriceIdList  ← 哪些加購品屬於此條件
       }

  POST /api/cart-calculate
  CartReachPriceExtraPurchase.MatchedProcess()
    └─ 建立 CartExtraPurchasePricePair {
           SkuId,
           SpecialPriceId,
           ConditionId,
           ConditionPrice = reachedConditionDict[conditionId].ReachPrice  ← N值 ($500)
       }
    └─ PromotionRecord { CartExtraPurchasePricePairs }

【Cart】CalculatePromotionProcessor
    └─ promotionInfo.CartExtraPurchasePricePairList
           = promotionRecords.First().CartExtraPurchasePricePairs
    └─ promotionInfo.Name = promotion.Name
           （來自 /api/promotion-rules/list-by-ids）
    └─ promotionInfo.IsMatch = true

  Cart Response → Shopping
    └─ CartPromotionInfoEntity {
           Name: "夏日活動",
           IsMatch: true,
           CartExtraPurchasePricePairList: [
               { SkuId, SpecialPriceId, ConditionPrice: 500 }
           ]
       }

【Shopping】GetSalepagePromotionDisplayTextProcessor
    └─ cartExtraPurchasePromotionList（IsMatch = true）
    └─ ArrangeCartExtraPurchasePromotionDisplayTextAsync()
    └─ GetSalepageDiscountDisplayAsync(DisplayType = CartExtraPurchase,
                                       skuId, optionalTypeId)

  CartReachPriceExtraPurchaseDisplayService.GetPromotionHintText(CartExtraPurchase)
    └─ reachRule = PricePairList.FirstOrDefault(
           x.SkuId == skuId && x.SpecialPriceId == optionalTypeId)
    └─ condition = ToCurrency(reachRule.ConditionPrice)  → "$500"
    └─ return string.Format("{0}{1} {2}",
           promotionInfo.Name,                    // "夏日活動"
           PromotionBaseDisplay.CumulativeSpendThreshold, // "累積滿額"
           condition)                             // "$500"
    → 輸出：「夏日活動累積滿額 $500」
```

## 文案二：`MatchedPromotion`（主商品已符合活動）

**期望文案**：`{活動名稱}滿額 $500`  
**DisplayType**：`MatchedPromotion`  
**DisplayTypeDescription**：`已符合`  
**觸發時機**：主商品（圈選商品）已達門檻（`IsMatch = true`）

### 節點路徑

```
【Promotion Frontend API】
  POST /api/cart-calculate
  CartReachPriceExtraPurchase.MatchedProcess()
    └─ reachedConditions = conditions.Where(c => totalPrice >= c.ReachPrice)
    └─ maxReachCondition = reachedConditions.MaxBy(c => c.ReachPrice)
    └─ CartExtraPurchaseMaxReachCondition {
           ConditionId,
           ReachPrice: 500,   ← N值（達到的最高門檻）
           ReachPiece: 0
       }
    └─ PromotionRecord { CartExtraPurchaseMaxReachCondition }

【Cart】CalculatePromotionProcessor
    └─ promotionInfo.CartExtraPurchaseMaxReachCondition
           = promotionRecords.First().CartExtraPurchaseMaxReachCondition
    └─ promotionInfo.IsMatch = true
    └─ promotionInfo.Name = "夏日活動"

  Cart Response → Shopping
    └─ CartPromotionInfoEntity {
           Name: "夏日活動",
           IsMatch: true,
           CartExtraPurchaseMaxReachCondition: { ReachPrice: 500 }
       }

【Shopping】GetSalepagePromotionDisplayTextProcessor
    └─ matchedPromotions = PromotionList.Where(x.IsMatch == true)
    └─ GetSalepageDiscountDisplayAsync(DisplayType = MatchedPromotion)

  CartReachPriceExtraPurchaseDisplayService.GetPromotionHintText(MatchedPromotion)
    └─ reachPrice = MaxReachCondition.ReachPrice  → 500
    └─ condition = ToCurrency(500)  → "$500"
    └─ return string.Format("{0}{1} {2}",
           promotionInfo.Name,               // "夏日活動"
           PromotionBaseDisplay.ReachPrice,  // "滿額"
           condition)                        // "$500"
    → 輸出：「夏日活動滿額 $500」

  DisplayTypeDescription：
    └─ PromotionBaseDisplay.MatchedPromotion → "已符合"
```

## 文案三：`MismatchedPromotion`（主商品尚未符合活動）

**觸發時機**：主商品未達門檻（`IsMatch = false`）、活動未排除  
**顯示商品卡**：主商品卡  

> 主商品卡同時有兩個欄位：
> - `DisplayText`：`享{活動名稱}`
> - `DisplayTypeDescription`：`再湊 $200`（格式串 + 差額金額）
>
> 這兩個欄位由 **`GetSalepageDiscountDisplayAsync`** 同一次呼叫內分別產生。

### 節點路徑

```
【Promotion Frontend API】
  POST /api/cart-calculate
  CartReachPriceExtraPurchase.MismatchedProcess()
    └─ hintState.LackOfPrice = (門檻金額 - 已購金額)  → 200
    └─ PromotionInstruction { State.LackOfPrice }

【Cart】CalculatePromotionProcessor
    └─ promotionInfo.IsMatch = false
    └─ promotionInfo.LackOfPrice = hintState.LackOfPrice  → 200
    └─ promotionInfo.Name = "夏日活動"

  Cart Response → Shopping
    └─ CartPromotionInfoEntity {
           Name: "夏日活動",
           IsMatch: false,
           LackOfPrice: 200
       }

【Shopping】GetSalepagePromotionDisplayTextProcessor
    └─ mismatchedPromotions = PromotionList.Where(
           x.IsMatch == false && x.IsAppOnly == false &&
           x.IsExcluded == false && x.LackOfRegion != true)
    └─ GetSalepageDiscountDisplayAsync(DisplayType = MismatchedPromotion)
         ↓ 同一方法內同時產生兩個欄位

  ① DisplayText
  CartReachPriceExtraPurchaseDisplayService.GetPromotionHintText(MismatchedPromotion)
    └─ return string.Format("{0}{1}",
           PromotionBaseDisplay.Enjoy,  // "享"
           promotionInfo.Name)         // "夏日活動"
    → 「享夏日活動」

  ② DisplayTypeDescription
  PromotionBaseDisplayService.GetDisplayTypeDescription(MismatchedPromotion)
    └─ CartReachPriceExtraPurchase ∈ _pricePromotionList
    └─ displayDescription = PromotionBaseDisplay.MismatchedPromotionByPrice
         → 格式串："再湊 {0}"
    └─ GetLackOfCondition(promotionInfo)
         → ToCurrency(promotionInfo.LackOfPrice) → "$200"
    └─ string.Format("再湊 {0}", "$200")
    → 「再湊 $200」
```

## 文案四：「再湊」 系列（加購品顯示但主商品未達門檻）

此路徑由 `UnreachCartExtraPurchaseDisplayTextProcessor` 負責，與前三者獨立。  

**觸發時機說明**：  
加購品**不需要被使用者加入購物車**。只要：  
1. 購物車有屬於該加價購活動圈選的主商品  
2. 主商品金額未達門檻  
3. 該活動的加購品出現在 SalepageList（透過快取或本次直接帶入）  

→ Promotion Engine 在 `/api/cart-calculate` 計算時，會對這些加購品貼上 `CartExtraPurchaseThresholdNotMeet`（定義於 `CartReachPriceExtraPurchase.TagsWhenMismatched`）  
→ Cart 的 `ArrangeCartExtraPurchaseUnreachThresholdProcessor` 將有此 flag 的加購品從 `SalepageList` 移出，放入 `UnreachAmountCartExtraPurchaseSalepageList`  
→ Shopping 的 `UnreachCartExtraPurchaseDisplayTextProcessor` 為這些商品產生「再湊」文案並顯示

### 4-A：金額未達門檻（`UnreachAmountCartExtraPurchaseSalepageList`）

**期望文案**：`再湊 $200 享加價購優惠`  
**DisplayType**：`MismatchedPromotion`  
**欄位**：`DisplayText`（沒有 DisplayTypeDescription）

```
【Promotion Engine（/api/cart-calculate）】
  CartReachPriceExtraPurchase（Mismatched 狀態）
    └─ hintState.LackOfPrice = 最近門檻 - 已購金額  → 200
    └─ hintState.PurchasedItemPrice = 已購金額       → 300
    └─ TagsWhenMismatched.Add(CartExtraPurchaseThresholdNotMeet)
    → Engine 將此 flag 貼在「本活動圈選的加購品」商品上

【Cart】CalculatePromotionProcessor (line 667, 686)
    └─ promotionInfo.LackOfPrice        = hintState.LackOfPrice        → 200
    └─ promotionInfo.PurchasedItemPrice = hintState.PurchasedItemPrice → 300
    └─ promotionInfo.CartExtraPurchases = 各條件的門檻清單 {
           ConditionId, ReachPrice, SpecialPriceIdList
       }
    └─ 有 CartExtraPurchaseThresholdNotMeet flag 的加購品由
       ArrangeCartExtraPurchaseUnreachThresholdProcessor 移出
    └─ 放入 UnreachAmountCartExtraPurchaseSalepageList

  salepage.PromotionFlags 包含：
      "{PromotionEngineId}:CartExtraPurchaseSpecialPriceId:{SpecialPrice_Id}"
  → Shopping 藉此解析出對應的 PromotionEngineId

【Shopping】UnreachCartExtraPurchaseDisplayTextProcessor
  Step 1: BuildCartPromotionEngineMapping()
    └─ 解析 PromotionFlags → mapping[salepage.CartId] = promotionEngineId

  Step 2: 逐筆加購品
    └─ promotionInfo = PromotionInfoList.FirstOrDefault(
           x.Id == promotionEngineId &&
           x.PurchasedItemPrice > 0 &&
           x.LackOfPrice > 0)                    ← 確認還沒達標
    └─ totalAmount = promotionInfo.PurchasedItemPrice → 300
    └─ thresholdCondition = CartExtraPurchases.FirstOrDefault(
           x.SpecialPriceIdList.Contains(salepage.OptionalTypeId))
    └─ threshold = thresholdCondition.ReachPrice  → 500（此加購品對應的條件門檻）
    └─ lackAmount = Math.Abs(threshold - totalAmount) → 200

  翻譯 key：UnreachAmountThresholdHint = "再湊 ${0} 享加價購優惠"
    → 輸出：「再湊 $200 享加價購優惠」
```

> **為什麼不直接用 `promotionInfo.LackOfPrice`？**  
> `LackOfPrice` 是 Promotion Engine 回傳的「到達最近門檻的差額」，代表整體活動層級。  
> 但當活動有多個門檻條件（如：滿 $300 可加購A、滿 $500 可加購B），  
> 加購品 A 的差額 vs 加購品 B 的差額會不同。  
> 因此 Shopping 必須根據每個加購品的 `OptionalTypeId` → 查出對應的 `CartExtraPurchases[].ReachPrice` → 個別計算差額，才能顯示正確金額。

### 4-B：活動條件變更或過期（`ExpiredPromotionSalepageList`）

**期望文案**：`本商品因活動條件變更，目前無法加購`  
**DisplayType**：`Disabled`

```
  UnreachCartExtraPurchaseDisplayTextProcessor
    └─ 活動已過期 或 查不到活動 → 顯示預設文案
    └─ 翻譯 key：PromotionConditionChangedOrExpired
         → 「本商品因活動條件變更，目前無法加購」
```

### 4-C：APP 限定（`ExpiredPromotionSalepageList` + `IsAppOnly`）

**期望文案**：`本商品為 App 限定商品，請至 App 購買`  
**DisplayType**：`Disabled`

```
  UnreachCartExtraPurchaseDisplayTextProcessor
    └─ promotionInfo.IsMatch == false &&
       (lackSalesChannel & AppMask) != 0
    └─ 翻譯 key：PromotionConditionAppOnly
         → 「本商品為 App 限定商品，請至 App 購買」
```

## 各文案欄位彙整

| 商品卡 | DisplayTypeDescription | DisplayText | 來源 Processor |
|---|---|---|---|
| 主商品（未達門檻） | `再湊 $200`（`MismatchedPromotionByPrice` + `LackOfPrice`） | `享夏日活動` | `GetSalepagePromotionDisplayTextProcessor` |
| 主商品（已達門檻） | `已符合` | `夏日活動滿額 $500` | `GetSalepagePromotionDisplayTextProcessor` |
| 加購品（已達門檻，正常區） | `滿額加價購` | `夏日活動累積滿額 $500` | `GetSalepagePromotionDisplayTextProcessor` |
| 加購品（未達門檻，特殊區） | 無 | `再湊 $200 享加價購優惠`（完整句在 DisplayText） | `UnreachCartExtraPurchaseDisplayTextProcessor` |
| 加購品（活動失效/APP限定） | 無 | `本商品因活動條件變更，目前無法加購` | `UnreachCartExtraPurchaseDisplayTextProcessor` |

> **注意**：主商品卡的「再湊 $200」在 `DisplayTypeDescription`；  
> 未達門檻加購品卡的「再湊 $200 享加價購優惠」整句都在 `DisplayText`，由不同 Processor 產生。

## 再湊文案差異比較

| | 主商品卡「再湊 $200」 | 加購品卡「再湊 $200 享加價購優惠」 |
|---|---|---|
| **出現時機** | 主商品 `IsMatch = false` | 加購品在 `UnreachAmountCartExtraPurchaseSalepageList` |
| **欄位** | `DisplayTypeDescription` | `DisplayText` |
| **Processor** | `GetSalepagePromotionDisplayTextProcessor` | `UnreachCartExtraPurchaseDisplayTextProcessor` |
| **差額計算** | 直接用 `promotionInfo.LackOfPrice`（Engine 算好） | 自行計算 `CartExtraPurchases[i].ReachPrice - PurchasedItemPrice` |
| **為何差異** | 單一活動整體差額，Engine 已算好 | 多門檻時不同加購品差額不同，需 per-condition 查 OptionalTypeId |
| **格式串** | `"再湊 {0}"` → `"再湊 $200"` | `"再湊 ${0} 享加價購優惠"` → `"再湊 $200 享加價購優惠"` |
| **搭配文案** | 同時有 `DisplayText = "享夏日活動"` | 只有 DisplayText，無 DisplayTypeDescription |

### 差額金額來源追溯

```
主商品卡：
  Promotion Engine hintState.LackOfPrice
    → Cart CalculatePromotionProcessor (line 667) promotionInfo.LackOfPrice
      → Shopping CartReachPriceExtraPurchaseDisplayService.GetLackOfCondition()
           → _currencyService.ToCurrency(promotionInfo.LackOfPrice)
           → "$200"

加購品卡：
  Promotion Engine hintState.PurchasedItemPrice
    → Cart CalculatePromotionProcessor (line 686) promotionInfo.PurchasedItemPrice → 300
  Promotion Engine CartExtraPurchaseConditions[].ReachPrice
    → Cart CalculatePromotionProcessor (line 646) promotionInfo.CartExtraPurchases[].ReachPrice → 500
      → Shopping UnreachCartExtraPurchaseDisplayTextProcessor
           → Math.Abs(500 - 300) = 200
           → string.Format("再湊 ${0} 享加價購優惠", 200)
           → "再湊 $200 享加價購優惠"
```

## 活動名稱（`Name`）來源

```
Cart → POST /api/promotion-rules/list-by-ids
  → GetPromotionEngineResponseEntity.Name
  → CalculatePromotionProcessor: promotionInfo.Name = promotion.Name
  → Shopping 直接讀取 promotionInfo.Name
```

`Name` 由 `/api/promotion-rules/list-by-ids` 取得，與 `/api/cart-calculate` 是兩隻獨立的 API，在 Cart 的 `CalculatePromotionProcessor` 組合後傳給 Shopping。
