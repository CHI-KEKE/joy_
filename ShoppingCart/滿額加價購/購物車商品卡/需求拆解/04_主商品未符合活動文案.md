# 需求④：主商品「未符合活動」文案

> **對應 AC**：活動名稱顯示邏輯與 Figma「商品卡顯示定義 spec」一致  
> **DisplayType**：`MismatchedPromotion`  
> **影響類型**：`CartReachPriceExtraPurchase`（滿額加價購）、`CartReachPieceExtraPurchase`（滿件加價購）

---

## 文案組成說明

| 欄位 | 說明 |
|------|------|
| `DisplayText` | 活動主文案（來自各活動 DisplayService 的 `GetPromotionHintText`） |
| `DisplayTypeDescription` | 狀態描述（再湊 $XXX 或 再湊 N 件，來自 `GetDisplayTypeDescription`） |

---

## 滿額加價購（`CartReachPriceExtraPurchase`）

### DisplayText
**來源**：`CartReachPriceExtraPurchaseDisplayService.GetPromotionHintText(MismatchedPromotion)`

```csharp
private const string MismatchedPromotionRuleFormat = "{0}{1}";

// MismatchedPromotion 分支
return string.Format(MismatchedPromotionRuleFormat,
    PromotionBaseDisplay.Enjoy,         // {0} "享"  ← ✅ zh-TW.json enjoy 已存在
    promotionInfo.Name);                // {1} 活動名稱
// 輸出："享{活動名稱}"
```

### DisplayTypeDescription
**來源**：`PromotionBaseDisplayService.GetDisplayTypeDescription(MismatchedPromotion)` for `CartReachPriceExtraPurchase`

```csharp
// CartReachPriceExtraPurchase 在 _pricePromotionList 中
// → displayDescription = PromotionBaseDisplay.MismatchedPromotionByPrice = "再湊 {0}"
// → GetLackOfCondition() = _currencyService.ToCurrency(promotionInfo.LackOfPrice)

displayTypeDescriptionInfo.DisplayTypeDescription =
    string.Format(displayDescription, this.GetLackOfCondition(promotionInfo));
// 輸出："再湊 $500"（✅ mismatched_promotion_by_price 已存在）
```

## 滿件加價購（`CartReachPieceExtraPurchase`）

### DisplayText
> ❌ **`CartReachPieceExtraPurchaseDisplayService` 不存在**，`GetPromotionHintText` 無法被呼叫，`DisplayText` 不輸出

### DisplayTypeDescription
**來源**：`PromotionBaseDisplayService.GetDisplayTypeDescription(MismatchedPromotion)` for `CartReachPieceExtraPurchase`

```csharp
// CartReachPieceExtraPurchase 不在 _pricePromotionList（金額清單）
// CartReachPieceExtraPurchase 不在 _piecePromotionList（件數清單）
// → displayDescription 為空字串 ""
// → 不進入 format，直接回傳空的 DisplayTypeDescriptionEntity

return displayTypeDescriptionInfo;  // DisplayTypeDescription = ""
```

> ❌ **`CartReachPieceExtraPurchase` 未被加入 `_piecePromotionList`，導致描述文案完全不輸出**

## 需調整項目

### 1. 新增 `CartReachPieceExtraPurchaseDisplayService`（詳見需求③）

`GetPromotionHintText(MismatchedPromotion)` 需回傳：
```csharp
// 輸出："享{活動名稱}"
return string.Format(MismatchedPromotionRuleFormat, PromotionBaseDisplay.Enjoy, promotionInfo.Name);
```

### 2. 將 `CartReachPieceExtraPurchase` 加入 `_piecePromotionList`

**檔案**：`PromotionBaseDisplayService.cs`

```csharp
private readonly Lazy<List<PromotionEngineTypeDefEnum>> _piecePromotionList
    = new Lazy<List<PromotionEngineTypeDefEnum>>(() =>
{
    return new List<PromotionEngineTypeDefEnum>()
    {
        // ... 現有類型 ...
        PromotionEngineTypeDefEnum.DiscountReachPieceWithFreeGift,
        PromotionEngineTypeDefEnum.MultiBuyLowestPriceFree,
        //// 購物車滿件加價購                         ← 新增
        PromotionEngineTypeDefEnum.CartReachPieceExtraPurchase,
    };
});
```

加入後，`GetDisplayTypeDescription(MismatchedPromotion)` 會使用：
```csharp
displayDescription = PromotionBaseDisplay.MismatchedPromotionByPiece;
// → "再湊 {0} 件"
// GetLackOfCondition() 需回傳差距件數（由 CartReachPieceExtraPurchaseDisplayService 實作）
```

### 3. `GetLackOfCondition` 實作（在新 DisplayService 中）

```csharp
public override string GetLackOfCondition(CartPromotionInfoEntity promotionInfo)
{
    // 回傳缺少件數
    return promotionInfo.LackPieceCount.ToString();
}
```

> 確認 `CartPromotionInfoEntity.LackPieceCount` 由 Cart 計算層正確填入。
