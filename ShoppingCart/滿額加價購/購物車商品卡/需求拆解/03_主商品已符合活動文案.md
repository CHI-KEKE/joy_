## `CartExtraPurchaseMaxReachCondition` 語意說明

加價購活動可設定**多個門檻層級**，例如：

```
Condition 1: 滿 2 件 → 可加購商品 A（$99）
Condition 2: 滿 5 件 → 可加購商品 A（$79）＋ 商品 B
Condition 3: 滿 10 件 → 可加購商品 A（$59）＋ 商品 B ＋ 商品 C
```

若使用者購買 7 件，則 Condition 1 ✅、Condition 2 ✅、Condition 3 ❌。

| 屬性 | 含義 |
|---|---|
| `CartExtraPurchasePricePairList` | 所有已達成門檻的加購品清單（Condition 1+2 的品項） |
| `CartExtraPurchaseMaxReachCondition` | **最高已達成門檻**（此例為 Condition 2，`ReachPiece = 5`） |

MatchedPromotion 文案顯示的 N 即來自 `MaxReachCondition.ReachPiece`（或 `.ReachPrice`）。

## 資料流確認（滿件加價購）

```
Promotion Engine Rule（CartReachPieceExtraPurchase.MatchedProcess）
  → CartExtraPurchaseMaxReachCondition = new CartExtraPurchaseCondition {
        ConditionId = maxReachCondition.ConditionId,
        ReachPrice  = maxReachCondition.ReachPrice,   // 0（件數版不用）
        ReachPiece  = maxReachCondition.ReachPiece    // ← N 的來源
    }
  → PromotionRecord (Engine 層)
  → PromotionRecord (Cart 層)
  → promotionInfo.CartExtraPurchaseMaxReachCondition (Cart BL)
  → CartReachPieceExtraPurchaseDisplayService (Shopping)
```

**`CartExtraPurchasePricePair.ConditionPiece` 的填值**：  
參照滿額版 `CartReachPriceExtraPurchase` 在 `MatchedProcess` 建立 `pricePairs` 時設定 `ConditionPrice = reachedConditionDict[pair.ConditionId].ReachPrice`，滿件版（`CartReachPieceExtraPurchase`，待 NuGet 2.25.1 發版）應對應設定 `ConditionPiece = reachedConditionDict[pair.ConditionId].ReachPiece`。

> ⚠️ **`CartReachPieceExtraPurchase` Rule class 尚未存在於 NuGet 套件（v2.25.1 待發版）。**  
> Shopping 這端的 DisplayService 可先實作，待 NuGet 發版後資料自然流通。

## 滿件加價購（`CartReachPieceExtraPurchase`）

### DisplayText
> ❌ **`CartReachPieceExtraPurchaseDisplayService` 尚未存在 → T3 實作目標**

設計方案（對照滿額版 B 版格式）：

```csharp
private const string PromotionRuleFormat = "{0}{1} {2}件";

// MatchedPromotion
var reachPiece = promotionInfo.CartExtraPurchaseMaxReachCondition.ReachPiece;
return string.Format(PromotionRuleFormat,
    promotionInfo.Name,                          // {0} 活動名稱
    PromotionBaseDisplay.ReachPiece,             // {1} "滿件"  ✅ T1 已新增
    reachPiece.ToString());                      // {2} "5"
// 輸出："{活動名稱}滿件 5件"

// CartExtraPurchase
var conditionPiece = reachRule.ConditionPiece;
return string.Format(PromotionRuleFormat,
    promotionInfo.Name,
    PromotionBaseDisplay.CumulativePieceThreshold, // "累積滿件"  ✅ T1 已新增
    conditionPiece.ToString());
// 輸出："{活動名稱}累積滿件 5件"

// MismatchedPromotion
return string.Format(MismatchedPromotionRuleFormat,
    PromotionBaseDisplay.Enjoy, promotionInfo.Name);
// 輸出："享{活動名稱}"
```
