# 需求②：活動依開始時間新到舊排序

> **對應 AC**：同一類型內的活動，依活動開始時間新到舊排序（新活動在前）  
> **影響類型**：`CartReachPriceExtraPurchase`（滿額加價購）、`CartReachPieceExtraPurchase`（滿件加價購）

---

## 現狀分析

### 排序機制：`CartPromotionInfoComparer`

```csharp
// 檔案：src/BusinessLogic/Nine1.Shopping.BL.BE/Promotion/CartPromotionInfoComparer.cs

int IComparer<CartPromotionInfoEntity>.Compare(CartPromotionInfoEntity? x, CartPromotionInfoEntity? y)
{
    // 第一排序：活動類型（依 _promotionTypeSorts 陣列 index）
    if (xType.Value.Equals(yType) == false)
    {
        return PromotionDiscountTypeCompare(nameof(xType), nameof(yType));
    }

    // 第二排序：同一類型內，開始時間由新到舊（y 在前 → 新的在前）
    if (y.StartDateTime.CompareTo(x.StartDateTime) != 0)
    {
        return y.StartDateTime.CompareTo(x.StartDateTime);
    }

    // 第三排序：時間相同時，ID 由大到小
    return y.Id.CompareTo(x.Id);
}
```

### 滿額/滿件加價購在類型排序中的位置

```csharp
private readonly string[] _promotionTypeSorts = new[]
{
    // ... 其他類型 ...
    nameof(PromotionEngineTypeDefEnum.CartReachPriceExtraPurchase),  // index 22（最後第二）
    nameof(PromotionEngineTypeDefEnum.CartReachPieceExtraPurchase),  // index 23（最後）
};
```

→ 滿額加價購排在滿件加價購之前，兩者都排在所有其他活動類型後面。

### `LitePromotionInfoComparer`（未勾選商品用）

同樣邏輯，包含 `CartReachPriceExtraPurchase` 和 `CartReachPieceExtraPurchase`，排序一致。
