# 需求①：不同類型的區塊排序

> **對應 AC**：商品卡依序顯示以下活動區塊（有資料才顯示，無資料則隱藏該區塊）  
> **規格順序**：買就送 → 單品特價 → 已符合活動（折價券／優惠碼）滿件加價購 → 未符合活動 → APP 獨享活動 → 無法參與活動 → 不適用折價券

---

## 關鍵架構發現：買就送的資料來源與 DiscountDisplayList 完全分離

### 買就送 ≠ 促銷活動，是商品層級設定

| 欄位 | 說明 |
|------|------|
| `SalepageEntity.SalepageGiftList` | **買就送**贈品清單，由 Cart service 直接填充 |
| `SalepageEntity.DiscountDisplayList` | 所有活動文案（單品特價、加價購、滿額折扣…），由 Shopping Processor pipeline 填充 |

**買就送（`SalepageGiftList`）不通過任何 Shopping Processor，Shopping 不對它做排序。**  
前端根據 `SalepageGiftList` 是否有資料，自行決定是否渲染「買就送」區塊。  
→ 對於買就送的顯示位置，**後端無需調整，屬於前端工作**。

---

## 現狀分析（DiscountDisplayList 的寫入順序）

| 步驟 | 內容 | DisplayType |
|------|------|-------------|
| 1 | 加購品活動標籤（`ArrangeCartExtraPurchasePromotionDisplayTextAsync`） | `CartExtraPurchase` |
| 2 | **已符合活動**（單品特價排第一，其餘含滿額/滿件加價購） | `MatchedPromotion` |
| 3 | **未符合活動** | `MismatchedPromotion` |
| 4 | **APP 獨享活動** | `AppOnly` |
| 5 | **無法參與活動** | `PromotionUnavailable` |
| 6 | **不適用折價券 / 不適用點數**（並行 Processor 補入） | `Disabled` |

### MatchedPromotion 內部排序
```csharp
// GetSalepagePromotionDisplayTextProcessor.cs 行 438-441
// 唯一的排序：把單品特價（DiscountByMemberWithPrice）強制排到最前
matchedPromotions = matchedPromotions
    .OrderByDescending(x => x.Type == PromotionEngineTypeDefEnum.DiscountByMemberWithPrice)
    .ToList();
// → 確定：單品特價排第一
// → 其他類型（含滿額/滿件加價購）：來自 HashSet，迭代順序不保證
```

## 問題點

| 區塊 | 規格期望 | 現狀 | 差異 | 負責層 |
|------|---------|------|------|--------|
| 買就送 | 第 1 區，獨立顯示 | **`SalepageGiftList` 獨立欄位**，前端直接使用 | N/A（前端工作） | **前端** |

| 無法參與活動 | 第 6 區 | `PromotionUnavailable`，位置正確 | ✅ | — |

## 待確認事項

### 關於單品特價的區分方式

前端是否根據 `PromotionEngineType`（活動類型欄位）自行區分「單品特價」（`DiscountByMemberWithPrice`）與其他 `MatchedPromotion` 活動？

- **若前端自行分拆（較可能）**：後端只需確保 `DiscountByMemberWithPrice` 在 `MatchedPromotion` 中排第一（現狀已符合）
- **若前端依 `DisplayType` 分區**：需在後端新增獨立 `DisplayType` 值（`SingleItemSpecialPrice`）

## 需調整項目（後端）

目前 `DiscountDisplayList` 的區塊順序 **已符合規格**（CartExtraPurchase → Matched → Mismatched → AppOnly → Unavailable → Disabled）。

若前端需要後端額外區分「單品特價」：

```csharp
// DiscountDisplayTypeEnum.cs 新增（如需要）
SingleItemSpecialPrice,   // 單品特價（DiscountByMemberWithPrice）
```
