# 需求⑤：不可折抵點數文案

> **對應 AC**：  
> - 滿件加價購活動後台設定「不能使用點數」時，**活動範圍品**商品卡顯示「不適用點數折抵」提示  
> - 滿件加價購活動後台設定「不能使用點數」時，**加購品**商品卡顯示「不適用點數折抵」提示  
> - 後台**未設定**時，兩者皆不顯示  
> **Processor**：`GetLoyaltyPointExcludedDisplayTextProcessor`

---

## 現有機制

### 文案輸出邏輯

```csharp
// 檔案：GetLoyaltyPointExcludedDisplayTextProcessor.cs
const string loyaltyPointExcludedFlag = "LoyaltyPointExcluded";

foreach (var salePage in salepageList)
{
    if (salePage.PromotionFlags.Any(x => x.Contains(loyaltyPointExcludedFlag)) == false)
        continue;

    salePage.DiscountDisplayList.Add(new SalepageDiscountDisplayEntity
    {
        DiscountSource = SourceTypeEnum.LoyaltyPoint.ToString(),
        DisplayText = GetLoyaltyPoints.LoyaltyPointExecluded,  // "不適用點數折抵"
        DisplayType = DiscountDisplayTypeEnum.Disabled
    });
}
```

### 翻譯文案

| Key | zh-TW 值 | 狀態 |
|-----|---------|------|
| `loyalty_point_execluded` | `不適用點數折抵` | ✅ 已存在 |

### 機制說明

此 Processor 採用 **flag-based** 判斷：
- `PromotionFlags` 包含 `LoyaltyPointExcluded` → 顯示文案
- `PromotionFlags` 不包含 → 不顯示
- Shopping 層**不需感知**活動類型，只要 Cart 計算層打入 flag 即可

---

## 可能的結果與對應處理

### 情況 A：Cart 計算層已正確打 flag
→ Shopping 層 **`GetLoyaltyPointExcludedDisplayTextProcessor` 不需任何修改**  
→ 功能已完整，直接驗收即可

### 情況 B：Cart 計算層未打 flag（或只打範圍品，未打加購品）
→ 需在 `nine1.cart` 專案補充 flag 設定邏輯

### 情況 C：需在 Shopping 層補充特殊判斷
→ 若 Cart 計算層結構上無法支援 flag，考慮在 `GetLoyaltyPointExcludedDisplayTextProcessor` 新增：

```csharp

        });
    }
```
