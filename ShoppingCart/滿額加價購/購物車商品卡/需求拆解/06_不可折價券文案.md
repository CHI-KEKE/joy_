# 需求⑥：不可折價券文案

> **對應 AC**：  
> - 滿件加價購活動不能使用折價券，所以**活動範圍品**商品卡顯示「不適用折價券」提示  
> - 滿件加價購活動不能使用折價券，所以**加購品**商品卡顯示「不適用折價券」提示  
> **Processor**：`GetSalepageDiscountDisplayTextProcessor`

---

## 現有機制

### 文案值

| Key | zh-TW 值 | 狀態 |
|-----|---------|------|
| `not_fit_coupons` | `不適用折價券` | ✅ 已存在 |

### 觸發邏輯（現有）

```csharp
// 檔案：GetSalepageDiscountDisplayTextProcessor.cs
// 處理 SalepageGroupList 中的一般商品

// 舊版加價購加購品（IsPurchaseExtra）
if (salePage.IsPurchaseExtra)
{
    text = GetSalepageDiscountDisplayText.NotFitCoupons;  // "不適用折價券"
}
```

## 問題分析

### 觸發條件缺口

| 商品類型 | 觸發條件 | 顯示文案 | 狀態 |
|---------|---------|---------|------|
| 舊版加價購加購品（`IsPurchaseExtra`） | ✅ 已有判斷 | ✅ `不適用折價券` | ✅ 已實作 |
| **新版購物車加價購加購品**（`IsCartExtraPurchase`） | ❌ 無判斷 | ❌ 不輸出 | ❌ 缺少 |
| **新版購物車加價購活動範圍品** | ❌ 無判斷 | ❌ 不輸出 | ❌ 缺少 |

### 現有加購品排除邏輯的問題

```csharp
// 商品加價購加購品（舊版 AddOns）跳過
if (salePage.CartExtendInfoItemTypeDef == nameof(CartExtendInfoItemTypeEnum.Sub) &&
    salePage.CartExtendInfos.Any(y => y.RuleTypeDef == nameof(CartExtendInfoRuleTypeDefEnum.AddOnsSalepageExtraPurchase)))
{
    continue;  // ← 舊版加購品跳過（因為不走折價券邏輯）
}
```

新版購物車加購品（`IsCartExtraPurchase`）**不在**此 continue 條件中，會進入主流程但沒有 `not_fit_coupons` 判斷。

### 範圍品的識別方式

範圍品（活動圈選商品，非加購品）目前沒有像 `IsCartExtraPurchase` 這樣的屬性標記，需透過 `PromotionFlags` 判斷是否為某筆加價購活動的圈選商品。

可能的識別方式：
```csharp
// 方式 A：PromotionFlags 包含加價購活動旗標
salePage.PromotionFlags.Any(x => x.Contains(nameof(PromotionFlagEnum.CartExtraPurchaseSpecialPriceId)))

// 方式 B：PromotionInfoList 中有已符合的加價購活動圈選到此商品
context.Data.PromotionInfoList
    .Any(x => (x.Type == PromotionEngineTypeDefEnum.CartReachPriceExtraPurchase ||
               x.Type == PromotionEngineTypeDefEnum.CartReachPieceExtraPurchase)
           && x.SalePageIds.Contains(salePage.SalePageId))
```

> 需確認哪種方式在現有資料結構中最可行。

## 需調整項目

### 修改 `GetSalepageDiscountDisplayTextProcessor.cs`

在 `IsPurchaseExtra` 判斷旁邊補充新版加購品邏輯：

```csharp
// 舊版加價購加購品
if (salePage.IsPurchaseExtra)
{
    text = GetSalepageDiscountDisplayText.NotFitCoupons;
}

// 新版購物車滿額/滿件加價購加購品          ← 新增
if (salePage.IsCartExtraPurchase)
{
    text = GetSalepageDiscountDisplayText.NotFitCoupons;
}

// 新版購物車滿額/滿件加價購活動範圍品       ← 新增（識別方式待確認）
// 此處判斷需確認範圍品的識別 flag 或屬性
// 方案待定：
// if (salePage.IsCartExtraPurchaseRangeProduct)
// {
//     text = GetSalepageDiscountDisplayText.NotFitCoupons;
// }
```

### 注意事項

1. `text` 覆蓋規則：加價購的折價券判斷放在折價券相關邏輯**之後**，確保最終覆蓋為「不適用折價券」
2. 折價券不適用文案與點數不適用文案同為 `Disabled` 類型，兩者可同時存在於 `DiscountDisplayList`
3. 範圍品識別方式須確認（與 PM/後端確認 `PromotionFlags` 設計）
