## 情境四-A：加購品商品卡（未達門檻，特殊清單）

**預期文案**：`DisplayText：再湊 N件 享加價購優惠`（無 DisplayTypeDescription）

```
【PromotionFrontendAPI — Engine】
  CartReachPieceExtraPurchase.MismatchedProcess()
    └─ hintState.LackPieceCount    = (最近門檻) - (已購件數)   → N
    └─ hintState.PurchasedItemPiece = 已購件數                 → M  (⚠️ 待 NuGet 2.25.1)
    └─ TagsWhenMismatched → 在本活動圈選的「加購品」上貼標：
         CartExtraPurchaseThresholdNotMeet
    └─ PromotionInstruction {
           State { LackPieceCount: N, PurchasedItemPiece: M }
       }

【Cart — CalculatePromotionProcessor】
  line 638: isCartExtraPurchaseType = true
  line 646: promotionInfo.CartExtraPurchases
              = promotion.CartExtraPurchaseConditions  (含每個條件的 ReachPiece)
  line 669: promotionInfo.LackPieceCount    = hintState.LackPieceCount    → N
  line 687: promotionInfo.PurchasedItemPiece = hintState.PurchasedItemPiece  → M
            [T-Bug2 新增；NuGet 2.25.1 前 M = 0]
  line 583: promotionInfo.Type = CartReachPieceExtraPurchase
  promotionInfo.IsMatch = false
    → Cart Response → Shopping

【Cart — ArrangeCartExtraPurchaseUnreachThresholdProcessor】
  加購品的 PromotionFlags 包含 "CartExtraPurchaseThresholdNotMeet"
    → 從 SalepageGroupList 移至 UnreachAmountCartExtraPurchaseSalepageList
  加購品的 PromotionFlags 也包含：
    "{PromotionEngineId}:CartExtraPurchaseSpecialPriceId:{SpecialPrice_Id}"

【Shopping — UnreachCartExtraPurchaseDisplayTextProcessor】
  BuildCartPromotionEngineMapping():
    salepage.PromotionFlags 解析 → mapping[cartId] = promotionEngineId

  foreach UnreachAmountCartExtraPurchaseSalepageList:
    promotionEngineId = mapping[salepage.CartId]

    // 滿額 filter 失敗（LackOfPrice == 0，PurchasedItemPrice == 0）
    amountPromotionInfo = null → 進入 else 分支

    // 滿件 filter [T-Bug2 修復後]
    piecePromotionInfo = PromotionInfoList.FirstOrDefault(
      x.Id == promotionEngineId &&
      x.Type == CartReachPieceExtraPurchase &&
      (x.PurchasedItemPiece > 0 || x.LackPieceCount > 0))  ← LackPieceCount > 0 即通過

    thresholdCondition = CartExtraPurchases
                           .FirstOrDefault(SpecialPriceIdList.Contains(optionalTypeId))

    lackPiece = PurchasedItemPiece > 0
        ? Math.Abs(thresholdCondition.ReachPiece - PurchasedItemPiece)  ← NuGet 2.25.1 後精確
        : LackPieceCount                                                  ← 現階段 fallback

    DisplayText = Format("再湊 {0}件 享加價購優惠", lackPiece)
                = "再湊 2件 享加價購優惠"
```

### T-Bug2 修復說明

**修復前根本原因**：

| 問題 | 說明 |
|---|---|
| Filter 失效 | `LackOfPrice: > 0` 對滿件永遠是 `false` → `promotionInfo = null` → 整筆 `continue` → 完全無文案 🚨 |
| 無「已購件數」欄位 | `PurchasedItemPiece` 不存在於 Shopping / Cart `CartPromotionInfoEntity` |
| 差額計算錯誤 | `ReachPrice - PurchasedItemPrice` 對滿件無意義 |
| 格式串錯誤 | `"再湊 ${0} 享加價購優惠"` 含 `$` 符號，件數不能用 |

**修復後新增檔案 / 修改**：

| 層 | 檔案 | 修改內容 |
|---|---|---|
| Cart BE | `Nine1.Cart.BL.BE\Promotion\Responses\HintInfo.cs` | 新增 `PurchasedItemPiece` 屬性 |
| Cart BE | `Nine1.Cart.BL.BE\Promotion\CartPromotionInfoEntity.cs` | 新增 `PurchasedItemPiece` 屬性 |
| Cart Service | `CalculatePromotionProcessor.cs` line 687 | `promotionInfo.PurchasedItemPiece = hintState.PurchasedItemPiece` |
| Shopping BE | `Nine1.Shopping.BL.BE\Promotion\CartPromotionInfoEntity.cs` | 新增 `PurchasedItemPiece` 屬性 |
| Shopping Service | `UnreachCartExtraPurchaseDisplayTextProcessor.cs` lines 81-145 | 拆分滿額/滿件 if/else，新增滿件 branch |
| Shopping Translation CS | `AutoGenerated\Backend.Service.CartProcessor.UnreachCartExtraPurchaseDisplayTextProcessor.cs` | 新增 `UnreachPieceThresholdHint` property |
| Shopping i18n（7 locale） | `i18n\Nine1.Shopping\backend.service.cart_processor.unreach_cart_extra_purchase_display_text_processor\` | 新增 `unreach_piece_threshold_hint` key |

### `lackPiece` fallback 精確度

| 活動結構 | `lackPiece` 來源 | 正確性 |
|---|---|---|
| 單一門檻（所有加購品同條件） | `LackPieceCount` | ✅ 精確 |
| 多門檻，此加購品屬最低門檻條件 | `LackPieceCount` | ✅ 精確 |
| 多門檻，此加購品屬更高門檻條件 | `LackPieceCount`（低估） | ⚠️ 近似 |
| NuGet 2.25.1 後（任何結構） | `ReachPiece - PurchasedItemPiece` | ✅ 精確 |

**狀態**：✅ **T-Bug2 修復後有文案**（多門檻精確計算待 NuGet 2.25.1）
