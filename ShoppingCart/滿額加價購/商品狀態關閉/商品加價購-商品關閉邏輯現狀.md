# 商品加價購 — 商品頁設定關閉（IsClosed）邏輯現狀

## 概述

說明商品加價購（CartAddOns）在顯示加購品清單時，如何排除「商品頁設定關閉」的商品，
以及整條資料流路徑。

---

## 完整呼叫路徑

```
前端使用者點「查看加購品」
  ↓
MVC Controller（AddOnsSalePageController.ExtraPurchaseList）
  └─ 只 return View()，不含任何業務邏輯
  ↓
前端 React 直接打 SPL
  GET ${FtsApiDomain}/salepage-listing/api/salepage/add-ons/{shopId}/{salePageId}
  ↓
SalepageExtraPurchaseController.GetAddOnsSalepagesAsync()
  ↓
SalepageExtraPurchaseService.GetAddOnsSalepagesAsync()
  │
  ├─ Step 1：Salepage Search Match → 撈命中的 PromotionId 清單
  ├─ Step 2：促購前台 → 取得全店適用的加價購活動 Id
  ├─ Step 3：合併所有 PromotionId
  ├─ Step 4：促購前台 → 取得活動詳細資訊
  ├─ Step 5：過濾 Exclude 的 CollectionId
  ├─ Step 6：篩選出 ON 檔活動
  ├─ Step 7：MongoDB price_rule → 取得加購品 SalepageId 清單
  │
  ├─ Step 8：GetSalepageSearchDataAsync()  ← ★ IsClosed 排除在這裡
  │     ↓
  │   ISalepageSearchService.SearchForPresentationAsync()
  │     ↓
  │   SPS 回傳 SalepageListingResponseDataSalepageModel
  │     ↓
  │   foreach salepageInfo：
  │     if (IsClosed == true)      → continue（排除）
  │     if (SellingTime 無效)      → continue（排除）
  │     if (IsInvisible)           → continue（排除）
  │
  ├─ Step 9：依活動設定排序
  └─ Step 10：組合 GetAddOnsSalepagesResponse 回傳
        └─ AddOnsSalepages（List）：每筆含 SalepageId、Title、Price、IsSoldOut...
  ↓
前端收到加購品清單，渲染頁面
```

## IsClosed 排除的程式碼位置

**檔案**：
```
C:\91APP\Salepage\salepage-listing\nine1.salepage.listing.web.api\src\
  BusinessLogic\Nine1.Salepage.Listing.BL.Services\Salepage\SalepageExtraPurchaseService.cs
```

**位置**：`GetSalepageSearchDataAsync()`，約 L390–399

```csharp
foreach (var salepageInfo in salepageInfos)
{
    //// 商品頁狀態為關閉、非開賣時間，即不顯示於加購品列表
    if (salepageInfo == null
        || salepageInfo.IsClosed == true              // ← IsClosed 排除
        || salepageInfo.SellingStartDateTime > DateTime.Now
        || salepageInfo.SellingEndDateTime < DateTime.Now)
    {
        continue;
    }
    // ... IsInvisible 檢查
}
```

## 欄位來源追蹤

| 層級 | 型別 / 檔案 | 欄位 |
|------|------------|------|
| 資料庫 | `SalePage` table | `SalePage_IsClosed` (bool) |
| SPS response | `SalepageListingResponseDataSalepageModel` | `IsClosed` (bool, JSON: `"isClosed"`) |
| 排除判斷 | `salepageInfo` (同上型別) | `salepageInfo.IsClosed == true` |

`IsClosed` 是**直接欄位（非計算屬性）**，由 SPS 從 DB 讀取後透過 JSON 反序列化傳入。

---

## 前端收到的 Response

回傳型別：`GetAddOnsSalepagesResponse`

前端收到的 `AddOnsSalepages` 清單中**不含 `IsClosed` 欄位**，
排除邏輯在 SPL 內部完成，前端只拿到已過濾的乾淨清單。

```csharp
public class AddOnsSalepageExtraPurchaseEntity
{
    public long SalepageId { get; set; }
    public string Title { get; set; }
    public decimal MinAddOnsPrice { get; set; }
    public bool IsSoldOut { get; set; }
    public List<AddOnsSalepageSkuEntity> SkuList { get; set; }
    // ... 其他欄位，無 IsClosed
}
```

## 架構特點

- **前端直接打 SPL**：MVC Controller 只負責 return View()，業務邏輯全在 SPL
- **排除在 SPL 內部完成**：前端不需感知 IsClosed，拿到的就是有效商品
- **同樣的欄位可用於滿額加價購**：`CartExtraPurchaseService` 使用的 response model 含有相同的 `IsClosed` 欄位，可直接比照加上相同排除邏輯

---

## 對 Story #582440 的意涵

滿額加價購的 `CartExtraPurchaseService.BuildPromotionEntitiesAsync()` 使用相同的
`SalepageListingResponseDataSalepageModel`，**`IsClosed` 欄位存在且可直接使用**。

只需在 `BuildPromotionEntitiesAsync()` 的過濾鏈中新增：

```csharp
.Where(x => x.IsClosed == false)    // 排除商品頁設定關閉的商品
```

即可對齊商品加價購的現行邏輯。
