# 商品加價購 / 滿額加價購 — 組合商品（IsSalePageBundle）邏輯現狀

## 概述

說明商品加價購與滿額加價購在顯示加購品清單時，
對於「組合商品」的處理現狀，以及兩者的差異。

---

## 組合商品的識別方式

組合商品透過 `SalePageModes` 欄位識別，當 `SalePageModes` 字串中包含 `"C"` 時即為組合商品。

```csharp
// GetSalepageListByIdsResponse.cs
public bool IsSalePageBundle
{
    get { return this.SalePageModes.ContainMode(SalepageModesEnum.C.ToString()); }
}

// SalepageModesEnum.cs
/// <summary>組合商品</summary>
C,

// ModeExtensions.cs
public static bool ContainMode(this string str, string c)
{
    if (str.IsNullOrWhiteSpace()) return false;
    return str.Contains(c);
}
```

**範例**：
- `SalePageModes = "AC"` → 包含 `'C'` → `IsSalePageBundle = true`
- `SalePageModes = "A"` → 不含 `'C'` → `IsSalePageBundle = false`

## 滿額加價購（CartExtraPurchaseService）現狀

### 使用的 Response Model
`GetSalepageListByIdsResponse`

### 欄位狀況
| 欄位 | 存在？ | 說明 |

| `SalePageModes` | ✅ 存在 | 原始模式字串 |

### 排除邏輯
```csharp
// BuildPromotionEntitiesAsync()，L639 附近
// 目前只排除售完商品：
if (salepage.IsSoldOut)
{
    continue;
}
// ❌ IsSalePageBundle 雖然已算好且為 true，但完全沒有拿來做排除
```

### 結論
> 判斷欄位有，且組合商品進來時 `IsSalePageBundle = true` 已算好，
> 但**沒有拿來用作排除邏輯**。
> 組合商品可能出現在滿額加價購的加購品清單中。
