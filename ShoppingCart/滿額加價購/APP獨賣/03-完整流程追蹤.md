# 03 — 完整流程追蹤（APP 獨賣）

> 本文件只討論**商品頁維度**：`SalePage_IsAPPOnly = true` 的商品作為加購品時，各階段的實際行為。

---

## 情境：商品 A（`SalePage_IsAPPOnly = true`）同時是滿額加購的加購品

### 階段一：加購選單顯示

```
消費者在 Web 端達成滿額門檻
    │
    ▼
[MWeb 前端] cartReachPriceExtraPurchaseBlock.tsx
    • 活動層級 isAppOnly == false（加購活動本身非 APP-only）
    • → 顯示加購選單入口 ✅
    │
    ▼
[Cart Domain API] GET /V2/CartExtraPurchase/ExtraPurchaseList/{id}
    │
    ▼
[SPL] CartExtraPurchaseService.BuildPromotionEntitiesAsync()
    • 呼叫 SalepagesSearchByIdsAsync()
      → 回傳 GetSalepageListByIdsResponse（無 IsAPPOnly 欄位）
    • BuildSalePageEntityAsync() 過濾條件：
      if (salepage.IsSoldOut) continue;  ← 唯一過濾，SalePage_IsAPPOnly 不在此
    • → 商品 A 正常出現在加購清單 ✅
    │
    ▼
[MWeb 前端] cartExtraPurchase.utility.ts
    • convertCartExtraPurchaseSalePageToEntity()
    • isAPPOnly: DEFAULT_IS_APP_ONLY = false  ← 硬碼，與 SalePage_IsAPPOnly 無關
    • → 加購 Modal 中正常顯示商品 A，無任何限制標籤 ✅
```

---

### 階段二：加入購物車

```
消費者選擇商品 A → 點確認加購
    │
    ▼
[MWeb 前端] CartExtraPurchaseProvider.tsx
    • 呼叫 Cart Domain API，帶入 optionalTypeDef = "CartReachPriceExtraPurchase"
    • Cart Domain 接收請求
    │
    ▼
[Cart Domain] 加購品加入 SalepageGroupList
    • IsCartExtraPurchase = true（computed property，由 OptionalTypeDef 判斷）
    • SalePage_IsAPPOnly：Cart Domain 完全不讀此欄位
    • → 商品 A 成功加入購物車 ✅
```

---

### 階段三：購物車計算

```
購物車重新載入（GET /cart）
    │
    ▼
[Cart Domain] CalculatePromotionProcessor.cs
    • GetPromotionRequestSkuListAsync()：商品 A 帶 CartExtraPurchase flag 送進 PE
    • PE 計算：SalePage_IsAPPOnly 不傳入 PE，PE 不知道此欄位
    • PE 回傳：商品 A 的加購活動命中，加購價格生效
    • → 商品 A 正常計算，無 APP 通路限制 ✅
    │
    ▼
[MWeb BLV2] CalculateShoppingCartPromotionEngineDiscountProcessor.cs
    • 商品 A 的 promotion 清單中無 IsAPPOnly = true（因為 SalePage_IsAPPOnly 根本沒有流到此）
    • → 不產生「APP 獨享」標籤 ✅
```

---

### 階段四：結帳

```
消費者送出結帳
    │
    ▼
[Cart Domain] 所有 Checkout Processors
    • 無任何 Processor 讀取 SalePage_IsAPPOnly
    • 無任何 Processor 因 APP 獨賣而阻擋結帳
    • → 商品 A 以加購價格完成結帳 ✅
```

---

## 各階段狀態總覽

| 階段 | 層級 | 商品 A 的狀態 | 是否有問題 |
|------|------|-------------|-----------|
| 加購選單顯示 | SPL | 不過濾，正常顯示 | ✅ |
| 加購選單顯示 | MWeb 前端 | isAPPOnly 硬碼 false | ✅ |
| 加入購物車 | Cart Domain | 正常加入，不讀 SalePage_IsAPPOnly | ✅ |
| 購物車計算 | Cart Domain PE | PE 不知道 SalePage_IsAPPOnly，正常計算 | ✅ |
| 購物車顯示 | MWeb 前端 | 不顯示「APP 獨享」標籤 | ✅ |
| 結帳 | Cart Domain | 無阻擋，以加購價格結帳 | ✅ |

**結論：整個流程中，`SalePage_IsAPPOnly` 在每一個環節都被自然繞過，不產生任何影響。**
