
# SCMAPIV2 多語系

## 架構說明

SCMAPIV2 的翻譯系統與其他專案相同，資料來源是 S3，但 StorageProvider 依環境不同：

| 環境 | StorageProvider | 說明 |
|------|----------------|------|
| Dev（本機） | `FileStorageProvider` | 以本機 `C:\Files` 為快取路徑 |
| QA / Prod | `RedisStorageProvider` | 以 Redis 為快取，資料來自 S3 |

**快取層級**：In-Memory Cache（60秒）⇒ File / Redis Cache ⇒ S3 Fetch ⇒ 直接回傳 Key 值

---

## 翻譯檔路徑

**Repo 來源**（新增 Key 要改這裡）：
```
C:\91APP\SCMAPIV2\nineyi.scm.apiv2\CrossLayer\Translations\i18n\NineYi.Scm.ApiV2\{module}\{locale}.json
```

**本機 File Cache**（`FileStorageProvider` 讀取位置）：
```
C:\Files\i18n\NineYi.Scm.ApiV2\{module}\{locale}.json
```

**S3**（QA / Prod 讀取位置）：
```
https://s3-ap-northeast-1.amazonaws.com/91dev-ap-northeast-1-translation-service/NineYi.Scm.ApiV2/{locale}/{module}.json
```

---

## 新增 Key 的完整流程

1. 在 `i18n\NineYi.Scm.ApiV2\{module}\zh-TW.json` 與 `en-US.json` 加入新 Key
2. 更新 `CrossLayer\Translations\AutoGenerated\` 對應的 `.cs` 檔案（新增 static string property）
3. 在 Validator / Service 中使用該 Key
4. 同步本機 File Cache（見下方）
5. 透過 CI/CD pipeline 將 JSON 上傳 S3（QA / Prod 才會生效）

---

## 問題：新增 Key 後顯示 Key 值而非翻譯

### 原因

TranslationClient **不直接讀 Repo 的 i18n JSON**，而是讀 File Cache（本機）或 Redis（QA）。

- **本機**：`C:\Files\i18n\NineYi.Scm.ApiV2\` 若不存在或版本舊，就 Fetch S3
- **S3 沒有新 Key** → Fetch 回來也沒有 → 回傳 Key 值本身

### 本機 Fix

把 Repo 的 i18n 複製到本機 File Cache 路徑：

```powershell
Copy-Item -Path "C:\91APP\SCMAPIV2\nineyi.scm.apiv2\CrossLayer\Translations\i18n\NineYi.Scm.ApiV2\*" `
          -Destination "C:\Files\i18n\NineYi.Scm.ApiV2\" `
          -Recurse -Force
```

複製後**重啟 IIS / IIS Express**，讓 TranslationClient 重新載入快取。

### QA Fix

需將更新後的 JSON 上傳到 S3，再讓 Redis Cache 失效。
通常透過 **Jenkins / CI 部署 pipeline** 中的 translation deploy step 執行。

## Debug

SCMAPIV2 沒有公開的 `/translations/debug` API，可直接確認：

1. 查看 `C:\Files\i18n\NineYi.Scm.ApiV2\{module}\zh-TW.json` 是否有該 Key
2. 直接 curl S3：
   ```
   https://s3-ap-northeast-1.amazonaws.com/91dev-ap-northeast-1-translation-service/NineYi.Scm.ApiV2/{module}/zh-TW.json
   ```
3. 若本機 File Cache 有但回傳 Key 值 → 重啟 IIS，讓 In-Memory Cache 過期（60秒）
