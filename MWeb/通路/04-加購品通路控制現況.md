# 加購品通路控制現況分析

## 設計意圖（業務期望）

> 加購品清單**不應受商品層級的 `IsAPPOnly` 影響**，
> 應只依據**活動本身的通路設定**（`IsWeb`、`IsiOSApp`、`IsAndroidApp`）決定是否顯示。

---

## 各層通路控制現況

### 情境 A：活動詳細頁 → 加購品清單入口

```
useInitPromotionEngineDetail.ts L211
  setIsShowCartExtraPurchaseBlock(
      promotionType === CartReachPriceExtraPurchase   ← 只看活動類型
  )
```

| 檢查項目 | 現況 |
|----------|------|
| 商品的 `IsAPPOnly` | ✅ **無影響**（結構上不存在於加購品資料） |
| 活動的通路設定（`IsWeb` 等） | ❌ **未判斷**，純看活動類型 |

---

### 情境 B：購物車加購品區塊

購物車商品資料（`shopping/models/models.d.ts`）：

```typescript
interface IShoppingCartClientSalePageEntity {
    IsPurchaseExtra: boolean;          // 是否為加購品
    IsAppOnlyPromotion: boolean;       // 活動是否 APP 獨享 ← 有帶回！
    // ... 其他欄位
}
```

| 檢查項目 | 現況 |
|----------|------|
| 商品的 `IsAPPOnly` | ✅ **無影響**（不在加購品資料結構中） |
| `IsAppOnlyPromotion`（活動通路） | ⚠️ **有帶回但從未被讀取使用** |

## 整體現況矩陣

| 層級 | 機制 | 商品 `IsAPPOnly` | 活動通路設定 |
|------|------|:---:|:---:|
| 活動列表撈取 | `IsWebVisible / IsiOSAppVisible` | — | ✅ |
| 活動詳細頁菜籃 | `IsAPPOnlyPromotion` → 下載APP按鈕 | — | ✅ |
| **加購品清單入口** | 只看活動類型 | ✅ 無影響 | ❌ |
| **購物車加購品區塊** | `IsAppOnlyPromotion` 有帶但未使用 | ✅ 無影響 | ❌ |

## 現行參考實作（活動詳細頁的做法）

`basketAction.tsx` 是目前**唯一有根據 `IsAPPOnlyPromotion` 控制行為**的地方：

```tsx
// basketAction.tsx
const isShowAppDownload = (): boolean => {
    return props.isAPPOnlyPromotion && isMatchedUserScopes;
};

// isAPPOnlyPromotion = true → 顯示「下載 APP」按鈕（取代加入購物車）
```

如需補齊加購品的通路判斷，可參考此邏輯。

## 潛在問題場景

假設某活動通路設定為 **APP Only**（`IsWeb=false, IsAndroidApp=true, IsiOSApp=true`）：

| 入口 | 現況行為 | 期望行為 |
|------|----------|----------|
| 活動列表頁 | Web 看不到此活動 ✅ | — |
| 活動詳細頁（直連） | 顯示下載 APP 按鈕 ✅ | — |
| 活動詳細頁加購品入口 | **顯示「查看加購品」連結** ⚠️ | 不應顯示 |
| 加購品清單頁（直連） | **直接進入，無阻擋** ⚠️ | 應導向下載 APP 或提示 |
| 購物車加購品區塊 | **正常顯示** ⚠️ | 不應顯示 |
