# 加購品通路控制 — 修補建議

## 目標

> 加購品清單應依照**活動本身的通路設定**決定是否對 Web 消費者顯示，
> **不受商品層級 `IsAPPOnly` 影響**（此部分現況已符合，無需改動）。

---

## 需要補齊的兩個入口

### 入口 1：活動詳細頁的加購品區塊連結

**檔案：** `useInitPromotionEngineDetail.ts`

**現況：**
```typescript
// L211 — 只判斷活動類型
setIsShowCartExtraPurchaseBlock(
    serverPromotionDetail.PromotionEngineType.toString() ===
        PromotionEngineTypeEnum[PromotionEngineTypeEnum.CartReachPriceExtraPurchase]
);
```

**建議修改方向：**
```typescript
// 加上通路判斷：活動類型符合 AND 非 APP Only 活動
setIsShowCartExtraPurchaseBlock(
    serverPromotionDetail.PromotionEngineType.toString() ===
        PromotionEngineTypeEnum[PromotionEngineTypeEnum.CartReachPriceExtraPurchase]
    && !serverPromotionDetail.IsAPPOnlyPromotion   // ← 補上此條件
);
```

> `IsAPPOnlyPromotion` 已存在於 `promotionDetail` 物件，
> 由後端 `PromotionEngineService.cs` 的 `_promotionAppHelper.IsAPPOnlyPromotion()` 計算後帶回。

---

### 入口 2：購物車加購品區塊

**檔案：** `shopping/models/models.d.ts` 的 `IShoppingCartClientSalePageEntity`

**現況：**
- `IsAppOnlyPromotion: boolean` 欄位**已存在**於購物車商品資料
- 但**沒有任何 component 讀取使用**此欄位

**建議修改方向：**
在購物車中顯示加購品區塊的 component 加入判斷：
```typescript
// 購物車加購品觸發區塊
if (salePage.IsPurchaseExtra && salePage.IsReachPurchaseExtraAmountThreshold) {
    // 現況：直接顯示加購品按鈕
    // 建議：加上通路判斷
    const shouldShow = !salePage.IsAppOnlyPromotion || isApp;
    if (shouldShow) {
        // 顯示加購品觸發按鈕
    }
}
```

## 後端是否需要異動

| 層 | 現況 | 是否需要改 |
|----|------|-----------|
| FTS `salepage/cart-extra-purchase` | 無通路欄位回傳 | ❓ 視 FTS 是否已在服務端過濾 |
| `PromotionEngineDetailPageService.cs` | `IsAPPPromotion` 已帶出 | ✅ 無需改動 |
| `CartExtraPurchaseController.cs` | 無任何通路驗證 | ⚠️ 可考慮加後端守門 |

## 參考實作：活動詳細頁的既有做法

`basketAction.tsx` 的 `isShowAppDownload()` 是目前系統中唯一依據活動通路設定控制 UI 的地方：

```tsx
// basketAction.tsx
const isShowAppDownload = (): boolean => {
    return (
        props.isAPPOnlyPromotion &&
        ((props.isLogin && props.isMatchedUserScopes && props.crmShopMemberCardName !== '') ||
            (props.crmShopMemberCardName === '' && props.isMatchedUserScopes))
    );
};
// 結果：isAPPOnlyPromotion = true → 按鈕改為「下載 APP」
```

加購品入口可比照此邏輯，**將 `IsAPPOnlyPromotion` 作為顯示條件的一部分**。

## 修改影響範圍評估

| 修改點 | 影響 | 風險 |
|--------|------|------|
| `useInitPromotionEngineDetail.ts` | 活動詳細頁加購品連結隱藏邏輯 | 低，只影響顯示判斷 |
| 購物車加購品觸發 component | 購物車加購品按鈕顯示邏輯 | 中，需確認欄位準確性 |
| `CartExtraPurchaseController.cs` | 後端守門（可選） | 低，加 redirect 或 404 |

---

## 不需要改動的部分

| 項目 | 原因 |
|------|------|
| 商品的 `IsAPPOnly` 對加購品的影響 | `ICartExtraPurchaseSalePage` 結構中無此欄位，天然排除 |
| FTS 回傳的加購品清單內容 | 業務上不應依商品 `IsAPPOnly` 過濾加購品 |
| `ExtraPurchaseProductList.tsx` 的 UI 渲染 | 渲染邏輯無需加 APP Only 判斷 |
