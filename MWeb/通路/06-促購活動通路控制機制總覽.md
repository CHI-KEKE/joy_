# 促購活動通路控制機制總覽

## 後端通路過濾架構

### 通路識別

```csharp
// UserClientTrackSourceTypeEnum
Web            → IsWebVisible
iOSApp         → IsiOSAppVisible
AndroidApp     → IsAndroidAppVisible
LocationWizard → IsLocationWizardVisible
APP            → IsiOSAppVisible && IsAndroidAppVisible
```

### 活動是否 APP 獨享的計算（Entity 層）

```csharp
// PromotionEngineEntity.cs
public bool IsAppOnly
{
    get
    {
        // 線上通路：IsWeb=false + iOS=true + Android=true
        return this.IsWeb == false && (this.IsAndroidApp && this.IsiOSApp);
    }
}
```

## 各活動類型通路控制現況

| 活動類型 | 後端列表過濾 | 前端詳細頁 | 前端加購/贈品 |
|----------|:---:|:---:|:---:|
| 一般折扣活動 | ✅ `IsWebVisible` | ✅ APP Only 標籤 + 下載按鈕 | — |
| 滿額/滿件贈品 | ✅ `IsWebVisible` | ✅ APP Only 標籤 + 下載按鈕 | — |
| **滿額加價購** | ✅ `IsWebVisible` | ✅ APP Only 標籤 + 下載按鈕 | ❌ 加購品入口無通路判斷 |
| 點數兌換活動 | ✅ `IsWebVisible` | ✅ APP Only 判斷 | — |

## `IsAPPOnlyPromotion` 的完整流程

```
DB: PromotionEngine.IsWeb / IsAndroidApp / IsiOSApp
  ↓
後端 PromotionAppHelper.IsAPPOnlyPromotion()
  └─ IsWeb == false && IsAndroidApp && IsiOSApp
  ↓
PromotionEngineService.cs
  ├─ GetPromotionList → IsVisiblePromotion() → 活動列表過濾
  ├─ FillPromotionLabelList → IsAPPOnlyPromotion = true → 標籤
  └─ GetSalePagePromotionList → IsAPPOnlyPromotion 帶入商品活動資訊
  ↓
前端 promotionDetail.IsAPPOnlyPromotion
  ├─ promotionCard.tsx → APP Only 標籤顯示
  ├─ basketAction.tsx → 下載 APP 按鈕（取代加入購物車）
  └─ basketDiscountMessage.tsx → 提示文案
```

## 購物車層的通路欄位

購物車商品 Entity（`IShoppingCartClientSalePageEntity`）：

```typescript
IsAppOnlyPromotion: boolean  // 活動是否 APP 獨享（已有，但未被購物車 UI 使用）
IsPurchaseExtra: boolean     // 是否為加購品
IsReachPurchaseExtraAmountThreshold: boolean  // 是否達加購門檻
```

## 現行通路判斷有效範圍

```
消費者進入活動頁面的路徑：

  [首頁/活動列表] → IsVisiblePromotion() 過濾  ← ✅ 通路判斷有效
       ↓
  [活動詳細頁] → IsAPPOnlyPromotion 控制菜籃   ← ✅ 通路判斷有效
       ↓
  [加購品清單頁] → 只看活動類型               ← ❌ 通路判斷缺失

  [購物車 P1] → 無通路過濾                    ← ❌ 通路判斷缺失
       ↓
  [購物車加購品觸發] → IsAppOnlyPromotion 未使用
```
